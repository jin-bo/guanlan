## 1. 目标与范围

**P3.1 = 给 entity/concept 页加可选 `aliases` frontmatter，让别名进入 `[[wikilink]]` 解析命名空间。**
一次投入同时解两个 backlog 问题：

- **消假断链**：`[[大语言模型]]` 与 `[[LLM]]` 都解析到声明这些别名的 `concepts/LLM.md`，不再误报断链（broken-link 笔记 §6-2）。
- **补 CJK 同义召回**：别名串纳入 query 的 2-gram 召回面与 ingest 去重纪律，缓解中文「同义不同名」漏召回（CJK 笔记第 1 条）。

**全程零-LLM、确定性、可幂等重建。** 别名是人写的同义词声明，解析/校验全是算术（正则 + 集合运算）。

### 纳入 P3.1

| 改动 | 文件 | 性质 |
|------|------|------|
| 别名解析归口（`alias_index` / `link_resolution_index` / 扩展 `link_target_stems`） | `guanlan/pages.py` | 零 LLM |
| 别名 frontmatter 校验（类型 + 全局唯一） | `guanlan/check.py` | 零 LLM，写门禁 |
| `[[别名]]` 边归到拥有页节点（不建幽灵节点） | `guanlan/graph.py` | 零 LLM |
| Web 站内 `[[别名]]` 联链 | `guanlan/web/render.py` | 零 LLM |
| 自愈 prompt 补一句别名约束 | `guanlan/gate.py` | 文案 |
| 别名约定（声明 / 去重 / 2-gram 召回面） | `skills/guanlan-wiki/{SKILL.md,references/conventions.md}` | 约定，零代码 |

`lint` 无需改动：它复用 `build_graph` 的 broken 边，graph 别名解析后断链/缺失实体 finding 自动减少。

## 2. 数据约定

entity / concept 页 frontmatter 可选追加 `aliases`：

```yaml
---
title: '大语言模型'
type: concept
tags: []
aliases: ['LLM', '大模型', 'large language model']   # 可选；缺省合法
sources: []
last_updated: 2026-06-05
---
```

- **可选**：缺 `aliases` 键完全合法（绝大多数页面不需要）。
- **类型**：非空字符串列表（同 `tags`/`sources` 的 YAML 单引号纪律）。
- **键名对齐 Obsidian** `aliases`，便于用户直接用 Obsidian 打开同一目录（CJK 笔记第 1 条）。
- **归一口径**：别名按 `pages.link_stem` 归一（小写、剥 `|`/`#`/`.md`/路径末段）后入解析集——与 `[[…]]` 查找口径**完全对称**，故 `[[LLM]]` / `[[llm]]` 都命中声明 `LLM` 别名的页（**\[决策P3.1-3]**）。

## 3. 解析归口（`pages.py`，单一可信源）

沿用 P3 决策P3-2/P3-6「四命令同一套解析口径、归口 `pages.py`」。新增两个派生器，**不改** `page_stem_index`（仍是 stem→path 的底座）：

```
page_stem_index(wiki)        # {stem(小写): 路径}  —— 全页面（含 config），不变
alias_index(wiki)            # {别名(归一): 拥有页 stem(小写)} —— 仅 content 页，容错读取
link_target_stems(wiki)      # 解析键集 = stems ∪ aliases  —— 扩展：现含别名
link_resolution_index(wiki)  # {键(stem|别名): 拥有页路径}  —— Web 站内导航用，stem 优先
```

- **`alias_index`**（**\[决策P3.1-1/2]**）：只扫 `iter_pages`（content 页；config 页不声明别名）；用**容错档** `load_page`（坏/缺 frontmatter 跳过、绝不抛，与 health/lint/graph 一致）；同名别名按 `iter_pages` 稳定排序**先到先得**（`setdefault`），保证幂等——真冲突由 `check` 报错（§4）。
- **`link_target_stems` 扩展为 stems ∪ aliases**：这是保持核心不变式 **`graph.broken ≡ check.wikilink.broken`** 的关键（**\[决策P3.1-5]**）——`check` 与 `graph` 仍共用这一个解析键集，别名一并生效，两边断链判定不分叉。
- **`link_resolution_index`**（**\[决策P3.1-6]**）：stem→path 之上叠加 别名→拥有页 path；**页面 stem 优先于别名**（撞名时不被别名遮蔽，且 `check` 已就该撞名报错）。供 Web 渲染解析 `[[别名]]` 为站内导航。

## 4. 命名空间唯一（`check.py`，写门禁）

别名与所有页面 stem 共享一个解析命名空间——歧义会破坏确定性解析，故 `check` 确定性校验（零 LLM、纯算术）。三类违规：

| kind | 触发 | 阻断? |
|------|------|-------|
| `frontmatter.bad_type` | `aliases` 存在但非「非空字符串列表」 | 是（既有 kind） |
| `aliases.collides_stem` | 归一别名与**某现有页 stem** 同名 | 是 |
| `aliases.duplicate` | 同一归一别名在库内被声明 ≥2 次（跨页或同页） | 是 |

- **都阻断写门禁**（**\[决策P3.1-4]**）：与断链「警告非阻断」相反——断链是 Karpathy 模式的**生长信号**，别名撞名/重复是**确定性危害**（解析歧义），应即时修复。落地上：这些 kind **不在** `gate._WARNING_KINDS`，经 `partition_violations` 自然归阻断，并入有界自愈（`gate.REPAIR_PROMPT` 补一句别名约束）。
- **不校验「撞 title」**（**\[决策P3.1-4] 受控偏离 CJK 笔记**）：`title` 不是 wikilink 解析键（解析按 stem/别名），别名撞 title 无解析后果；为保持命名空间最小、避免把不参与解析的字段拉进唯一性约束，**不报**撞 title。若日后 title 也要可链，再单列。
- 校验只针对 content 页（`iter_pages`）的**严格档**解析结果，与既有 frontmatter 校验同一遍循环，不二次扫盘。

## 5. graph / web 联链

- **`graph.build_graph`**（**\[决策P3.1-5]**）：`[[…]]` 经 `link_stem` 归一后分类新增一档——
  ① 命中节点 stem → resolved（id=该 stem）；
  ② **命中别名 → resolved，边归到「拥有页 stem」节点**（不建别名幽灵节点，邻接/孤儿口径不被别名稀释）；
  ③ 命中 config stem → 丢弃；④ 谁都不中 → broken。
  因 `link_target_stems` 已含别名，别名目标永远在「②」被先行解析，绝不落到 broken——故 `graph.broken ≡ check.wikilink.broken` 不变式继续成立。
- **`web/render.py`**（**\[决策P3.1-6]**）：解析表由 `page_stem_index` 换成 `link_resolution_index`，`[[别名]]` 渲染为站内锚链。行内 `code` 兜底（决策P4-3）**不**因此放宽——它的「整段忠实等于某现有页」判据比对的是**页面真名/路径**，别名不满足，故 `` `别名` `` 不会被兜底成链，仍须显式 `[[别名]]`，行为收敛。

## 6. 决策记录

- **\[决策P3.1-1]** 别名进入 `[[wikilink]]` 解析命名空间，归口 `pages.py`（`alias_index`）；与 stem 同口径、小写不敏感。`[[别名]]` 解析到声明页 → 消假断链。
- **\[决策P3.1-2]** 仅 content 页（`iter_pages`）可声明 `aliases`；config 页不扫。容错档读取，坏 frontmatter 跳过；同名别名稳定排序先到先得（幂等），真冲突交 `check`。
- **\[决策P3.1-3]** 别名按 `link_stem` 归一存入解析集，与 `[[…]]` 查找口径完全对称。
- **\[决策P3.1-4]** 命名空间唯一由 `check` 确定性校验，三类违规（`bad_type`/`aliases.collides_stem`/`aliases.duplicate`）**均阻断**写门禁（确定性危害，非生长信号）；**不**校验「撞 title」（title 非解析键，受控偏离 CJK 笔记）。
- **\[决策P3.1-5]** 解析键集统一扩为 stems ∪ aliases（`link_target_stems`），`check` 与 `graph` 共用，保持 `graph.broken ≡ check.wikilink.broken`；`graph` 增显式别名分支把 `[[别名]]` 边归到拥有页 stem 节点，不建幽灵节点。
- **\[决策P3.1-6]** Web 站内联链经 `link_resolution_index`（stem→path ∪ 别名→拥有页 path，stem 优先）；行内 code 兜底口径不放宽，别名仅经显式 `[[别名]]` 成链。
- **\[决策P3.1-7]** 召回与去重为 skill/约定层职责（零代码）：entity/concept 可声明 `aliases`；ingest 建页前查别名去重（把既有「补一行常用别名」升级为结构化声明）；2-gram 召回面纳入别名串。

## 7. 测试要点

- `pages`：`alias_index` 归一/容错/先到先得；`link_target_stems` 含别名；`link_resolution_index` stem 优先且别名落到拥有页 path。
- `check`：合法别名通过且 `[[别名]]` 不再 `wikilink.broken`；`aliases` 类型错 → `bad_type`；别名撞 stem → `collides_stem`；别名重复 → `duplicate`；缺 `aliases` 键仍合法。
- `graph`：`[[别名]]` → resolved 边指向拥有页节点、不在 `broken`、不新增节点；`broken ≡ check` 回归。
- `lint`：声明别名后对应 `broken_link`/`missing_entity` finding 消失。
- `web`（有 `web` extra 时）：`[[别名]]` 渲染为 `a.wikilink[data-page=拥有页]`；`` `别名` `` 行内 code 不成链。
