# 多格式转换

`guanlan convert` 补的是一个 CLI 缺口:`ingest` 只接受 `.md`,而很多资料是 PDF/DOCX/PPTX/XLSX/HTML/图片。`convert` 把它们经 **`pdf-to-markdown` 技能**(MinerU → marker → pypdf 分层回退)转成 `raw/<slug>.md` 源,再走原样的 `.md` ingest。

> 它**不是新的主流程**,也**不替代** Web 端的「上传→解析→人审→晋级」(那条更强)。它补的是 CLI 用户「PDF/DOCX → raw → ingest」的官方入口。

## 用法

```bash
guanlan -C my-wiki convert 报告.pdf                 # 转换并落 raw/报告.md
guanlan -C my-wiki convert 报告.pdf --dry-run        # 只打到 stdout,raw/ 零写(人审预览)
guanlan -C my-wiki convert 报告.pdf --ingest         # 转换成功后串联 ingest
guanlan -C my-wiki convert 报告.pdf --name notes     # 覆盖目标 slug → raw/notes.md
guanlan -C my-wiki convert a.pdf --backend marker    # 指定后端
```

| 参数 | 默认 | 说明 |
|---|---|---|
| `src` | — | 待转换的文件(PDF/DOCX/PPTX/XLSX/HTML/图片…) |
| `--name` | 原件 stem | 覆盖目标 slug |
| `--origin` | 转换前原始 `src` 路径 | 显式出处(写入 `origin` provenance) |
| `--overwrite` | 关 | 同名 `raw/` 已存在时显式覆盖(默认不覆盖) |
| `--dry-run` | 关 | 只把结果打到 stdout,`raw/` 零写 |
| `--ingest` | 关 | 转换成功后串联 `ingest raw/<slug>.md` |
| `--backend` | `auto` | 转换后端:`auto`/`mineru`/`marker`/`python`(透传技能 `convert.py`) |

## 行为契约

- **两步默认,不自动 ingest**:`convert` 只做「转换 + 落源」(确定性宿主写 `raw/`,无 Agentao、无门禁、无快照、无 `log.md`)。建页是另一步 `guanlan ingest`(或加 `--ingest` 串联)。
- **脚本零 LLM**:观澜不带 LLM 客户端/密钥,只外调转换器。marker 是否用 Gemini 由**用户环境**决定(无 `--model`、无 env 擦除、无模型透传)。
- **图片随源落盘**(P5.2.1):转换器抽出的图片落到 `raw/images/<slug>/<slug>-N.ext`,并重写 markdown 图片引用指向新相对路径,使 `raw/<slug>.md` **自洽**。容量三道闸:单图 20 MiB / 累计 200 MiB / 张数 500,超限报错(非静默丢图)。`--dry-run` 连图一并零落盘。
- **无新运行依赖、无新 extra**:复用技能的分层后端,任一可用即可;全不可用时优雅降级报错(退出码 `1`)。

> 真正跑一次转换需至少装好一个后端(MinerU / marker / pypdf)。MCP **不做 convert**(写 `raw/` 与只读姿态冲突)。
