# 维护:体检与图谱

知识库长期保鲜靠一组**零 LLM 的确定性维护工具**。`health` / `lint` 是 **advisory(建议性,默认不阻断)**;`graph` / `reindex` 是写派生物/修索引;`heal` 是唯一需 LLM 的物化写操作(走门禁)。

---

## `guanlan health`

文件级结构体检:**桩页(stub)+ index↔磁盘同步**。

```bash
guanlan -C my-wiki health
guanlan -C my-wiki health --json
guanlan -C my-wiki health --strict     # 有建议则以退出码 6 失败(供 CI/nightly)
```

| 参数 | 说明 |
|---|---|
| `--json` | 输出 JSON 契约 |
| `--strict` | 有建议时退出码 `6`(`EXIT_LINT_FINDINGS`),用于 CI/nightly 把建议升级为失败 |

默认 advisory:即使有建议也退出 `0`。`index_missing_page`(磁盘有页未收录)由 [`reindex`](#guanlan-reindex) 修。

---

## `guanlan lint`

图感知结构 lint:**孤儿页 / 断链 / 缺失实体**,外加图谱拓扑的 advisory 发现(hub 节点 / 跨社区细链 / 孤立社区 / 割边 / 割点)。

```bash
guanlan -C my-wiki lint
guanlan -C my-wiki lint --json
guanlan -C my-wiki lint --strict
```

| 参数 | 说明 |
|---|---|
| `--json` | 输出 JSON 契约 |
| `--strict` | 有发现则退出码 `6` |

默认 advisory。`missing_entity`(高频断链)由 [`heal`](#guanlan-heal) 物化成页。

---

## `guanlan graph`

确定性建图:`[[wikilink]]` → `graph/graph.json` + 自包含零-JS `graph/graph.html`(含社区、拓扑提示、割边/割点)。

```bash
guanlan -C my-wiki graph
guanlan -C my-wiki graph --json-only     # 只写 graph.json,跳过 graph.html
```

| 参数 | 说明 |
|---|---|
| `--json-only` | 只写 `graph.json`,跳过 `graph.html` |

图谱是**可幂等重建的派生物**,从不成为事实来源。LLM 推断边被刻意排除(保持可重建性)。

---

## `guanlan reindex`

索引回填:把磁盘已存在但未收录的内容页**登记进 `index.md`**。**零 LLM**,修 `health.index_missing_page`。

```bash
guanlan -C my-wiki reindex
guanlan -C my-wiki reindex --dry-run     # 只打印 worklist,不写盘
guanlan -C my-wiki reindex --prune       # 额外删除指向不存在文件的悬空行
```

| 参数 | 说明 |
|---|---|
| `--dry-run` | 只打印 worklist,纯读不写盘 |
| `--prune` | 额外删除 index 里指向不存在文件的悬空行(`index_dangling`),默认关 |
| `--json` | 输出 JSON 契约 |

默认应用(写 `wiki/index.md`)。一行摘要留空给后续 `ingest`/人填(决策 P3.4-1)。只写 `index.md`,不碰 `raw/`、不过门禁、不写 `log.md`。

---

## `guanlan heal`

缺失实体物化:把高频断链(被多页 `[[...]]` 引用却无页)**按需 LLM 建成 entity 页**。**走 P2 写门禁。需配置模型。**

```bash
guanlan -C my-wiki heal --dry-run                 # 仅打印 worklist(纯读、零 LLM)
guanlan -C my-wiki heal                            # 物化(默认上限/阈值)
guanlan -C my-wiki heal --limit 3 --min-refs 2
```

| 参数 | 说明 |
|---|---|
| `--limit` | 本批最多物化几个(按引用频次降序;须 ≥ 1) |
| `--min-refs` | 入选阈值:被 ≥ 此数页引用才物化(对齐 lint;须 ≥ 1) |
| `--dry-run` | 仅打印 worklist,纯读、零 LLM、不触 Agentao |
| `--model` | 覆盖 Agentao 模型 |
| `--json` | 输出 worklist/receipt 的结构化 JSON |

`--dry-run` 先看清要建哪些再决定。物化是受治理写入(子进程 + `raw/` 快照,同 ingest)。

## 退出码

| 码 | 名 | 含义 |
|---|---|---|
| `0` | `EXIT_OK` | 成功(advisory 命令即使有建议也是 0,除非 `--strict`) |
| `1` | `EXIT_USAGE` | 用法/环境错误(如缺可选依赖、参数非法、IO 失败) |
| `3` | `EXIT_CHECK_FAILED` | `check` 校验失败(写门禁把关项) |
| `4` | `EXIT_RAW_MUTATED` | 写操作期间 `raw/` 被改动(快照不一致) |
| `5` | `EXIT_AGENT_ERROR` | Agentao 运行时/Agent 出错 |
| `6` | `EXIT_LINT_FINDINGS` | `health`/`lint` 在 `--strict` 下有建议/发现 |

**advisory-not-gate**:`health`/`lint` 默认退 `0`,只在 `--strict` 才用 `6` 失败——日常用看报告、CI/nightly 用 `--strict` 当门禁。
