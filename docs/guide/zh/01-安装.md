# 安装

## 环境要求

- **Python 3.10+**。
- 核心命令（`init` / `check` / `health` / `lint` / `graph` / `reindex` / `search`）**零 LLM、可离线运行**，不需要任何模型或网络。
- `ingest` / `query` 以及 Web 端问答需要**配置一个模型**（经 Agentao 运行时）。
- `convert` 从观澜这一侧看也是零 LLM，但它会**外调** `pdf-to-markdown` 技能的转换后端（MinerU / marker / pypdf），真正跑一次转换需至少装好其中一个后端。

## 安装核心包

```bash
pip install guanlan-wiki
```

> PyPI 发布名是 `guanlan-wiki`（裸名 `guanlan` 已被一个无关项目占用）；安装后**命令行与导入名仍是 `guanlan`**。

验证：

```bash
guanlan --version     # 应打印当前版本（如 0.1.8.dev0）
guanlan --help
```

## 可选宿主（叠加层，按需安装）

观澜的核心是 CLI；Web 与 MCP 宿主是**可选叠加层**，不装也能用全部 CLI 命令。

### Web 宿主

```bash
pip install 'guanlan-wiki[web]'
```

带入可选依赖：`fastapi` / `uvicorn` / `markdown` / `python-multipart` / `anyio`。装完才有 `guanlan web`，详见 [Web 宿主](05-web-宿主.md)。

### MCP 宿主

```bash
pip install 'guanlan-wiki[mcp]'
```

带入官方 `mcp` SDK（`mcp>=1.27,<2`）与 `anyio`。装完才有 `guanlan mcp`，详见 [MCP 宿主](06-mcp-宿主.md)。

> 两个 extra 都是「**没装就优雅降级**」：未安装对应依赖时,运行 `guanlan web` / `guanlan mcp` 会给出明确的「请 `pip install 'guanlan-wiki[...]'`」提示,而非崩溃。

## 从源码开发安装

仓库用 [`uv`](https://github.com/astral-sh/uv) 管理依赖:

```bash
git clone <repo-url>
cd guanlan
uv run guanlan --help     # 直接在项目环境里跑 CLI
uv run pytest             # 跑测试
```

开发模式下,仓库根**自身就是一个样例 wiki**,维护引擎 `skills/guanlan-wiki/` 会命中 Agentao 的 repo-root 技能发现路径(`<工作目录>/skills/`),**免安装**。详见仓库根 [`CLAUDE.md`](../../../CLAUDE.md) 的「Two run modes」。
