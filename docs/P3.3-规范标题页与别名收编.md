## 1. 目标与范围

P3.2 把高频断链物化成页，但只教了**最稳的一条**：文件名 == 归一目标键（`大模型` → `大模型.md`），
`[[大模型]]` 经同一 `link_stem` 天然解析。这条路正确但**留了两个洞**：

1. **命名不规范沉淀**：`大模型`、`地钱`、中英缩写等**口语/俗称**会被原样固化成页名，库越长越乱；更规范的
   学名/全称（`大语言模型`、`地下钱庄`）反而进不了 stem。core 其实**已允许**建规范标题页并在 `aliases`
   收编原键（`test_heal_canonical_title_with_aliases_resolved` 已证：stem≠target + `aliases:[target]` →
   回执 `resolved`、新页不计越界），但 **v1 prompt/skill 不引导**。
2. **与 ingest 的同名页去重缺闭环**：heal 新页**不登记 `index.md`**（`HEAL_PROMPT` 只追加 `log.md`），于是
   ① `health` 报 `index_missing_page`（磁盘有、index 无）失步；② query 的 2-gram `index.md` 召回看不见它；
   ③ **后续 ingest 查重（扫 `index.md`/`aliases`）看不见它 → 可能为同一实体另建重复页**。

**P3.3 = 把规范命名从「允许」升为「引导」，并补上 heal↔ingest 的同名页去重闭环。** 三件事：

- **引导规范标题**：口语/缩写目标，若引用上下文给出更规范全称，建规范 stem + `aliases` 收编原键（§2/§4）。
- **收编既有页变体**：目标其实是某**已有页**的变体时，不再「跳过」也不新建重复页，而是向该已有页 `aliases`
  **追加**收编（受控放宽写集审计，§3/§5）。
- **登记 index.md**：heal 新页/更新一律登记 `index.md`（含别名注记），让去重闭环、query 召回、health 同步成立（§4）。

**全程零新退出码、零 `gate.py` 改动、零新解析口径。** core 的 worklist 聚合（`lint.missing_entities`）、写门禁
（`run_guarded_write_result`）、别名解析（`link_resolution_index`，决策P3.1-6）、重算图回执（`heal._build_receipts`
本就 alias-aware）**全部原样复用**。唯一 core 改动是 **`heal._writeset` / `_snapshot_wiki`**（§3/§4），其余是
prompt/skill/conventions 的引导文案。

### 纳入 P3.3

| 改动 | 文件 | 性质 |
|------|------|------|
| `HEAL_PROMPT`：从「直接作文件名」补三态引导（规范标题 / 收编既有页 / 直接用目标名），收尾登记 `index.md` | `guanlan/heal.py` | 文案（仍薄，步骤在 skill） |

## 2. 规范标题判据（何时改名，判断驱动、默认保守）

heal 对每个目标在**新建**时三选一，**判据全在 skill（intelligence 不入包）**，wrapper prompt 只列出三态存在：

| 模式 | 触发 | 落地 |
|------|------|------|
| **A 直接用目标名**（默认） | 目标已是合用的页名，或拿不准更规范形式 | 建 `entities/<目标>.md`（同 P3.2 最稳路径） |
| **B 规范标题 + 收编** | 目标**口语/俗称/缩写**，且**引用上下文里出现了更规范的全称/学名** | 建 `entities/<规范名>.md`，`aliases:['<目标>', …]` 收编原键 |
| **C 收编到既有页**（§3） | 目标其实是某**已有页**（`entities/` 或 `concepts/`）的变体（读 `index.md`/引用页可认出） | 向该已有页 `aliases` **追加** `<目标>`，**不新建** |

- **默认保守（**\[决策P3.3-1]**）**：无明确更规范形式 → 走 A。**不为「显得规范」强行改名**——改名后若 stem≠target
  又漏收编，断链不消、回执 `still_broken`（§5 风险），还可能撞 `check` 唯一性。「拿不准就用目标名」是硬默认。
- **改名必收编（**\[决策P3.3-2]**）**：选 B（**新建**规范页）时 `aliases` **至少含原目标归一键**，且**可顺带收编上下文
  同义变体**（新页是 added、不走 §3 窄缝，仅受 `check` 唯一性约束）；但选 C（**改既有页**）时**只准追加本批目标键**——
  §3 条件 3 的严格口径（新增别名须全部 ∈ 本批 target），**不得 piggyback 无关别名**。每个别名须**全局唯一**（不撞任何页
  stem、不重复声明）——由 `check`（`aliases.collides_stem` / `aliases.duplicate`，决策P3.1-4）**阻断兜底**，违规则有界自愈
  或 `EXIT_CHECK_FAILED` 留盘。skill 不必自校验唯一性。
- **解析零新口径（**\[决策P3.3-3]**）**：A/B/C 三态的「是否真修好」**统一**由 §5 重算 `link_resolution_index`
  （stem ∪ alias，stem 优先，决策P3.1-6）判定——`heal._build_receipts` 本就查它，**回执逻辑一字不改**：
  目标命中某页（经 stem 或别名）即 `resolved`，否则 `still_broken`。B/C 的收编因此与 A 的同名落点同样可验证。

## 3. 收编既有页变体：受控放宽写集审计（核心 core 改动）

模式 C 是 P3.3 唯一触碰确定性边界的地方。P3.2-11 的写集审计口径是「**新建页不算越界、改/删已有页算有害写**」——
但「目标是已有页变体」的正解恰恰是**改已有页**（向其 `aliases` 追加一行）。v1 因此只能「跳过」，把断链留到
后续 ingest 或人工。P3.3 **受控放宽**这条线（**\[决策P3.3-5]**）：

> 对一处「**修改已有内容页**」的写入，**当且仅当下列全部成立**时**不**计入 `unexpected_writes`。
> **判据纯按写后 final-state、不追溯文件操作历史**（删后重建 / 整页覆写只要终态满足下列即视同安全；
> 快照只有 before/after 内容，本就无从、也无需证明 agent 中途怎么写）：
> 1. 该页是已有 content 页（entity/concept），before/after **均存在**（`diff_raw` 归为 `modified`，非 `removed`），
>    且**两侧都是普通文件**——任一侧为 symlink/special 一律不豁免、仍 `unexpected`（解析字段只对普通文件计算，
>    否则「替换 symlink 指向另一文件」可能在终态伪装成别名增长被误豁免，Finding 3）；
> 2. **终态只体现 `aliases` **尾部纯追加**（按原始字符串**列表前缀**判，非集合）**：before 的 `aliases` **缺失或为合法
>    字符串列表**、after 为**合法字符串列表**，且 **before 原始列表是 after 的前缀**（`after[:len(before)] == before`，即原有
>    别名**原序、原重复次数、一个不删**地保留、新别名只在尾部追加）；**正文 body 逐字节不变**，其余 frontmatter 字段
>    （`title`/`type`/`tags`/`sources`）不变（`last_updated` 允许 bump）。**按集合判会漏**：`['LLM','LLM']→['LLM','大模型']`
>    或「重排既有别名后再追加」按 set/⊇ 都像 superset，实则删了原值 / 动了原序（Finding 1）；故须**前缀**校验并校验
>    before/after 形态合法（P2 增量门禁不阻断历史 aliases 坏账，helper 不能假设 before 合法）。**重排即不豁免**（计
>    `unexpected`，无害但下轮自然收敛）；
> 3. **新增别名全部收编本批目标**：新增别名归一键集**非空且全部** ∈ 本批 worklist `target` 集——「至少一个命中」
>    挡不住 piggyback（同一次可夹带无关别名仍被豁免），故要求**全部**命中，杜绝借收编之名给已有页乱加别名；
> 4. 该写入**未引入新的 alias 唯一性阻断违规**——P2 是**增量门禁**（`gate.py:258`/`:328`：只追究不在写前 `baseline`
>    里的新违规），故 `gate_ok=True` 仅表示本轮没新增 `aliases.collides_stem`/`aliases.duplicate`（新加的本批目标键若
>    撞名即新违规、会阻断），**不**保证全库历史无 alias 欠债。本相位**不改 P2 增量口径、不额外跑全量 check**。
>
> 任一条不成立 → 仍计 `unexpected_writes`（含：动了 body、改了别的 frontmatter 字段、删页、删/重排原有 alias
> 字符串（破坏前缀）、别名字段形态非法、任一侧 symlink、新增别名有任一不属本批目标）。

- **仍守住的线**：heal **永不改已有页正文、永不删页、永不碰 `raw/`**。放宽只开「**向已有页纯追加本批目标别名**」
  这一道窄缝——它是**消假断链**（决策P3.1 的本职），不是内容篡改。删页 / 改正文 / 改其余 frontmatter 一律照旧
  `unexpected`（终态不止别名增长即越界，与「覆写还是原地改」无关）。
- **判定需 before 内容（本相位的真实 core 成本）**：要判「仅别名增长」须有写前的解析字段，但**不能因此丢掉原始字节
  保真度**——`changed_paths` 仍须是「本轮 wiki/ 写集**全量**」（§5/§6）。故 `_snapshot_wiki` 每页值**并列两层**，各司其职：
  - **原始指纹**（普通文件 sha256 / 符号链接目标，同 P3.2 `heal.py:148`）：`diff_raw` 的增/删/改分类与 `changed_paths`
    **只看它**——任何字节变化（含别名重排、改大小写、归一集不变的纯文本改动）都照常进 `changed_paths`，口径不退化；
  - **解析字段**（**仅对普通文件计算**，symlink/special 不算——见条件 1）`(body 指纹,
    frontmatter-除-{aliases,last_updated} 的规范化指纹, aliases **原始字符串列表 + 形态合法性标记**)`：**仅供**
    `_is_safe_alias_collection` 判豁免，不参与 diff 分类。存**有序原始字符串列表**（非集合、非仅归一集）才能做**前缀**校验
    （原有别名原序原次保留、只尾部追加）与 before/after 形态合法（P2 增量门禁不阻断历史 aliases 坏账，helper 不能假设 before 合法）。
  helper 据解析字段判 **两侧均普通文件 ∧ body 指纹不变 ∧ 其余 frontmatter 指纹不变 ∧ before/after alias 形态合法 ∧
  before 原始列表是 after 的前缀（`after[:len(before)]==before`）**，全真才把该 `modified` 项从 `unexpected` 移除——**但它仍留在 `changed_paths` 里**（确实写过）。**两个陷阱**：① 只存解析字段 → agent 把
  `aliases` 重排/改大小写而归一集不变时原始字节已变却被判「无改动」、`changed_paths` 漏报（故原始指纹不可省，Finding 1）；
  ② 只存原始指纹 → 无法证明 `title`/`tags`/`sources` 未被悄改（故解析字段不可省）。`_writeset` 对**被修改**的内容页加
  一条分支做此确定性 diff——纯算术、零 LLM、不解析 agent 文本。这是 P3.3 唯一触及确定性边界的实现改动，需与现有
  symlink / `log.md` append-only 处理（`heal.py:276`）共存。
- **「过 check」必须落到 `_writeset` 签名（条件 4 的接口落点）**：条件 3（命中本批目标）与条件 4（过门禁）单靠 snapshot
  helper 落不了——现 `_writeset(before, after)`（`heal.py:273`）既不接 `work_targets` 也不接 `gate.ok`。故 `_writeset`
  扩为 **`_writeset(before, after, work_targets, gate_ok)`**：**先按 P3.2 原口径算越界**（改/删/覆盖已有页全算
  `unexpected`），**再仅当 `gate_ok`** 时把通过 `_is_safe_alias_collection(before[p], after[p], work_targets)` 的收编项
  从 `unexpected` 移除。门禁未过 → 不豁免、改动留盘待修（与 `_build_receipts(gate_ok=…)` 的 still_broken 口径一致，
  `heal.py:281`）。调用处相应改为传本批 `target` 集与 `write.gate.ok`。
- **与回执解耦**：放宽只影响 `unexpected_writes` 这一**批次级**事实；目标 `resolved`/`still_broken` 仍**独立**由
  §5 重算图判定（决策P3.3-3）。收编成功 → `link_resolution_index[目标]` 指向被收编页 → `resolved`，且
  `created_path=None`（非本轮新建，仅更新）——`_build_receipts` 现有 `resolved_to in added` 逻辑天然得出。

## 4. 登记 index.md：补全去重 / 召回 / 同步（写集允许面 += index.md）

heal 新建/更新页**应登记 `index.md`**（**\[决策P3.3-4]**，**prompt/skill 约定，非确定性硬门禁**），否则它不是一张
「物化完整」的页：

- `health._check_index_sync` 对「磁盘有、index 无」报 `health.index_missing_page`——不登记 = 每轮 heal 制造失步建议；
- query 的 2-gram **`index.md` 召回**看不见它；
- **后续 ingest 查重**（SKILL ingest 步 3：扫 `index.md` 与现有页 `aliases`）看不见它 → 同名页去重失效（§5）。

**为何是约定而非硬契约**：heal 不为「漏登记」当轮判失败。漏登记**不是有害写**（不毁旧数据），塞进 `unexpected_writes`
属范畴错误；单列新字段又违反本相位「零新字段、`--json` 契约不变」（§6）；降级回执也不对（链接确已 `resolved`）。
`index.md` 是 config/派生物、可幂等重建，遗漏由 **`health.index_missing_page` 下轮兜底发现**（`index_md_links`，`pages.py`），
与「`index.md` 与 ingest 同信任级」一致。故登记由 prompt/skill 驱动、health 把关，**不**进 heal 的确定性写集裁决。

故把 **`wiki/index.md` 纳入 heal 的写集允许面**：它与 `wiki/log.md` 并列为 heal 合法触碰的两个 **config catalog**，
`_writeset` 对其修改**不计** `unexpected_writes`。与 `log.md` 的差别：`log.md` 是严格 append-only（另由 append-only
校验把关，`heal.py:271`），而 `index.md` 是**分区目录**、需向对应分区**插行**，故**不**套 append-only、允许常规编辑——
这与 ingest 在全量写门禁下自由维护 `index.md` 同一信任级（`index.md` 是 config / 派生物 / 可幂等重建，P2 门禁本就不护它）。

- **登记内容**：新页在 `## Entities`（或对应分区）追加 `- [<标题>](entities/<stem>.md) — <一句话>`；**模式 B/C 的别名
  在句末注记**（conventions §别名：`… — 一句话（别名：大模型/LLM）`），让别名词命中 2-gram 召回、并给 ingest 查重留线索。
- **收尾顺序**：每个目标处理完，prompt/skill 要求 **建页/收编 → 登记/更新 index.md → 追加 log.md**。三者都在
  允许面内（新页 added、index.md/ log.md catalog 豁免、收编走 §3 窄缝），不触 `unexpected_writes`。

## 5. 与 ingest 的同名页去重双向闭环（**\[决策P3.3-6]**）

P3.3 的落点是让**同一实体在 heal 与 ingest 两条写路径下收敛到唯一页**，不产生 `大模型.md`/`大语言模型.md`/`LLM.md`
三页并存：

- **正向（heal → 未来 ingest）**：heal 建规范页 `大语言模型.md`、`aliases:['大模型']`、并在 `index.md` 注记别名。
  后续 ingest 处理涉及该实体的资料时，其**查重**（扫 `index.md` 注记 + 现有页 `aliases`）命中此页 → **更新而非新建**；
  若资料引入新变体（`LLM`），ingest 把它追加进该页 `aliases`（ingest 走全量写门禁，可改已有页）。→ 无重复页。
- **反向（既有 ingest 页 → heal）**：heal 读 `index.md` + 引用页时认出目标是某既有页变体 → 走**模式 C**（§3 收编），
  **不新建**重复页。这把 P3.2 v1 的「疑似已有页别名 → 跳过」升级为「**收编到既有页**」——断链当轮即消，不再等后续 ingest。
- **收口**：两路径都以「**唯一 stem + 别名命名空间全局唯一**」为不变式，由 `check`（决策P3.1-4）确定性兜底——
  撞名/重复别名阻断写门禁，从机制上禁止两页声明同一别名。

> 这条闭环把 P3.1 的「ingest 查重纪律」（决策P3.1-7）与 P3.2 的「heal 物化」正式咬合：**别名是两条写路径
> 共享的去重锚**，谁先建页、谁来收编变体都收敛到同一页。

## 6. 不变量与回执（沿用，不改）

- **检测确定性 / 生成有门禁 / 默认非自动 / 按需且有界**：P3.2 四立场不变。heal 仍是显式按需、不入 ingest、不自动跑。
- **回执纯由图判定**：§5 的 `resolved`/`still_broken` 全程**不解析 agent 输出**（决策P3.2-12）。A/B/C/跳过在图上
  的表现统一——目标解析了即 `resolved`，没解析（漏收编/真跳过/上下文不足）即 `still_broken`，下轮再纳入 worklist。
- **`--json` 契约不变**：沿用 P3.2 的 `{worklist, postponed, receipts, unexpected_writes, changed_paths, exit_code}`
  全部六字段、不增不减（`changed_paths` = 本轮 wiki/ 写集全量，见 `heal._result_dict`，P3.2 §6）；放宽只让本该为 0 的
  `unexpected_writes`（合法收编）不再误报。CLI 无新参数、退出码仍写路径四值（决策P3.2-5）。
- **幂等**：A/B/C 都让目标退出 worklist（解析后不再 broken），重跑 → 空 worklist → 短路 `EXIT_OK`。已有页被收编后
  其别名集稳定，重跑不重复追加（worklist 里该目标已消失，根本不会再被处理）。

## 9. 风险

- **漏收编致空补**：选 B 改名却漏把原键写进 `aliases` → `[[大模型]]` 仍断、回执 `still_broken`。缓解：默认走 A
  （决策P3.3-1）、conventions 把「改名必收编」立为硬线、§5 回执确定性识别并下轮重试。
- **收编窄缝被误用**：放宽（决策P3.3-5）虽四条件夹紧，仍给了 heal 触碰已有页的能力。若实测出现「借收编之名改坏
  已有页」，可收紧为「别名追加须 ≤ 本批目标数」或回退到「仅跳过」。**body 逐字节不变 + 收编本批目标**是主防线。
- **index.md 信任放宽**：把 `index.md` 纳入允许面后，buggy agent 可改乱目录（P2 门禁本就不护 index）。缓解：index 是
  config/派生物，`health` 下轮报失步、可幂等重建；与 ingest 同信任级，非 P3.3 新引入的风险面。
- **真·非实体反复重试**：收编（C）已消解「变体-of-已有页」一大类；残留的**真·非实体 / 永远上下文不足**目标仍每轮
  重入 worklist、重付 LLM——同 P3.2 已知折中，根治见 §10 持久化跳过。缓解仍是 `--limit`/`--min-refs`/`--dry-run`。
- **批量改名质量**：一次 guarded write 内既建新页又收编既有页，目标多则上下文杂、易选错规范名。`--limit` 是唯一闸门，
  默认保守（≤10），报告始终显示推迟数。
