# P3.10 页型↔目录一致性体检（schema 漂移 advisory，零 LLM）

> 半阶段增强，不是新里程碑。给 `health` **加两类确定性 advisory**——页 frontmatter `type` 与所在目录不符、
> 内容页落在四规范目录之外——把"盘面页型组织漂没漂"做成零-LLM 体检项。源自反向评审
> （[`backlog/notes/gbrain-反向评审结论.md`](backlog/notes/gbrain-反向评审结论.md) §4）：借 gbrain `schema detect`
> 的**零-LLM detect 半**，**弃**其 LLM-suggest + mutate/migrate/retype 回填（过重、绑 DB、改内容越薄壳线）。

## 0. 关键前提：SCHEMA.md 是自由文本、不被机器解析

走查确认：`SCHEMA.md` 在库根、**从不被 Python 读取解析**（仅 `paths.py` 做存在性校验、`init` 当模板生成）。
它是**人/LLM 面的自由 markdown 配置**（`## 启用的页面类型` 表是给人看的、非 API 契约）。观澜的页型分类
**硬编码**在 `check._VALID_TYPES`（`source/entity/concept/synthesis`，`check.py:41`）+ `reindex._DIR_TO_SECTION`
（四目录，`reindex.py:33`），`page_type`（`pages.py:170`）只从 frontmatter `type` 取、**不看目录**。

> **据此定调（决策P3.10-1）**：**不解析 SCHEMA.md**（保其自由文本、零耦合；解析那张表会引入脆弱的格式依赖）。
> 漂移信号取**确定性可得**的两路——「frontmatter `type` ↔ 目录」一致性 + 「游离目录」——对照**硬编码规范
> 分类法归口**，而非对照 SCHEMA.md 散文。SCHEMA.md 仍是人面声明，本体检**提醒你让盘面与它保持自洽**、但不读它。

（注：因页型硬编码 + `check` 阻断非法 `type`，**正常写门禁下**观澜盘面不应出现"未声明的页型"——绕过门禁或
历史坏 frontmatter 仍可能留下非法 `type`，但那由 `check` 报、不归本体检；gbrain 那种 drift 在观澜退化为下面
两类。"SCHEMA.md 驱动自定义页型"是更大的口子，明确**出本阶段范围**，见 §5。）

## 1. health 所需的最小公共常量（不做跨模块重构）

新检查只需两样确定性常量：**目录→页型映射** `DIR_TO_TYPE` 与**合法页型集** `VALID_TYPES`。
本阶段只把这两个常量放进 `pages.py`（审计命令的共享原语本就在此），供 `health` 取用：

```python
# pages.py：health 体检所需的目录↔页型常量
DIR_TO_TYPE = {
    "sources":   "source",
    "entities":  "entity",
    "concepts":  "concept",
    "syntheses": "synthesis",
}
VALID_TYPES = frozenset(DIR_TO_TYPE.values())
```

- **不把这当成"分类法大一统"重构**（决策P3.10-3）：P3.10 的价值是下面两个确定性 finding，不是顺手收编
  「目录 ↔ 页型 ↔ index 分区」三元关系。`reindex._DIR_TO_SECTION` 维持原样、**不动**。
- 为不凭空多出"新的一处分叉"（`pages.VALID_TYPES` vs `check._VALID_TYPES`），`check` **可**把
  `_VALID_TYPES = VALID_TYPES` 改引同一常量（一行、行为字节不变、原测试全绿）——但这是**机会性清理、非
  P3.10 必做项**，不做也不阻塞本阶段（实现时由测试护住行为即可）。
- 若日后真要把三元关系彻底收一处，那是独立的零-behavior-change 重构候选，与本体检增强解耦——本文档
  **不**把"三处永不分叉"立为硬目标。

## 2. 两类新 finding（落进 `health`，非新命令）

沿 `health`「是否内部自洽」语义 + advisory 退出（默认 `EXIT_OK`，`--strict` + findings → `EXIT_LINT_FINDINGS(6)`），
在 `run_health`（`health.py:93`）的 per-page 循环里加两检（与 `health.stub_page`、`health.index_*` 并列）：

### 2.1 `health.type_dir_mismatch`（逐页）

页 frontmatter `type` 与所在目录约定的 type 不符：

```
wiki/entities/Foo.md  type=source   → "页在 entities/ 目录但 type=source，与目录约定 type=entity 不符"
```

- 取目录 = `path.relative_to(wiki).parts[0]`，查 `DIR_TO_TYPE` 得"目录约定 type"。
- **直接读 `meta.get("type")`，不走 `page_type(meta)`**：后者对非法字符串**原样返回**、只在缺失/非字符串时才回退
  `unknown`（`pages.py:175`），拿来比对会把非法 `type` 误报成 mismatch。先 `isinstance(type_, str) and type_ in
  VALID_TYPES` 判合法，**仅合法**才与目录约定 type 比对，不符才报。
- 非法 / 非字符串 / 缺失的 `type` 一律**跳过**、交 `check` 阻断：缺失是 `check.frontmatter.missing_key`、
  非法/非字符串是 `check.frontmatter.bad_type`（`check.py:69-72`），`health` 不重复其职责（容错档 `load_page`
  拿 meta，`meta=None` 亦跳过，决策P3.10-6）。

### 2.2 `health.uncharted_page`（逐页）

内容页落在四规范目录之外（含直接躺 `wiki/` 根的非 config 页、或 `wiki/misc/…` 这类杂目录）：

```
wiki/misc/Bar.md  → "页不在 sources/entities/concepts/syntheses 任一目录，无法按约定归类"
```

- 目录 ∉ `DIR_TO_TYPE`（或 `parts` 不足两段）→ 报。
- 这正是 `reindex._section_for`（`reindex.py:95`）"落四目录外 → 跳过不猜"那批页——`reindex` 静默跳过，`health` 把它们**显式列出**，让人决定归位还是扩约定。

## 3. 决策（沿用仓库 `决策PX-N` 体例）

- **决策P3.10-1（不解析 SCHEMA.md）**：见 §0。保自由文本、零耦合；信号取目录/type 确定性比对。
- **决策P3.10-2（落 `health` 不开新命令）**：两类 finding 进 `health`，复用其 `HealthReport`/JSON 契约/退出语义。比 gbrain 独立 `schema detect` 命令**更省面**（不引新命令/退出码/SSE）。
- **决策P3.10-3（只放最小公共常量、不做分类法大一统）**：见 §1，仅把 health 所需的 `DIR_TO_TYPE`/`VALID_TYPES` 放进 `pages.py`；`reindex._DIR_TO_SECTION` 不动，`check._VALID_TYPES` 改引为**机会性清理、非必做**。P3.10 的价值是两个 finding，不在收编全局分类法。
- **决策P3.10-4（mismatch 是 advisory 非 blocking）**：页 `type` 合法只是**放错目录**——内容有效、非契约违规，故 advisory（`health`），区别于 `check.frontmatter.bad_type`（`type` 非法 → 阻断）。
- **决策P3.10-5（uncharted 是 advisory 非 error）**：Karpathy 模式允许 wiki 演进出新目录；`health` 提示、不阻断（`reindex` 也只跳过不报错）。避免把"库在长新结构"误判成坏库。
- **决策P3.10-6（坏/缺 frontmatter 不重复报）**：`meta=None`、`type` 缺失（check `missing_key`）/非法（check `bad_type`）一律交 `check`；`health` 用容错档 `load_page`、跳过这些页，只管"`type` 合法但放错目录"。
- **决策P3.10-7（无新退出码）**：riding `health` 既有 `EXIT_LINT_FINDINGS`。

## 4. 测试要点（仿 `test_health.py` 风格）

- type 与目录相符（四目录各一正例）→ 无 `type_dir_mismatch`。
- `entities/X.md` 配 `type=source` → 一条 `type_dir_mismatch`，detail 含 `type=source` 与约定 `entity`。
- `wiki/misc/Y.md`、`wiki/Z.md`（根、非 config）→ `uncharted_page`；四目录内页**不**报 uncharted。
- 缺 `type`（`check` 报 `missing_key`）/ 坏 `type`（`check` 报 `bad_type`）→ `health` **不**出 `type_dir_mismatch`（不重复）。
- `--strict`：有上述 finding → 退出 6；默认 → 0。
- JSON 契约：finding 含 `{page, kind, detail}`，`pages_checked` 计内容页。
- 若 `check` 改引 `pages.VALID_TYPES`（机会性）：原 `check` 测试全绿（行为字节不变）；不改则 `check`/`reindex` 本就不动。

## 5. 边界（明确不做）

- **不解析 SCHEMA.md、不支持 SCHEMA.md 驱动的自定义页型/目录**（决策P3.10-1）——那要 `check` 读 SCHEMA、动校验数据流，是更大的口子（候选 E 轨"可配置分类法"，非本阶段）。
- 不自动**移动/改名/retype** 页（gbrain mutate/migrate/retype 回填明确不借）——只报、不动盘。
- 不改 `check`/`reindex` **行为**（纯加法 `health` 检查；`pages.py` 仅新增常量；`check` 若改引为零-behavior-change 一行、`reindex` 不动）。
- 不碰 `raw/`、不起 LLM、不引依赖。
