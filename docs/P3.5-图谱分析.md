## 3. 行为细则

### 3.1 确定性社区检测（手写 Louvain）

标准 Louvain（模块度优化）本身有"节点遍历序随机 + 平局随机"两处不确定，会让 `graph.json` 不再字节稳定。
P3.5 用**确定性变体**（决策P3.5-2）：

1. 无向化（`undirected_adjacency`），每条唯一邻接对权重 1；
2. 每轮**按 node_id 升序**遍历（`g.nodes` 已 sorted）；
3. 把节点移入"模块度增益最大"的邻居社区；**增益相等时取代表节点 id 最小的社区**（确定性 tie-break）；
4. 无节点再移动则本层收敛；聚合超节点后重复，直到模块度不再提升；
5. **规范化重编号**：按"社区最小成员 id 升序"给社区排序，赋 `0..k-1`——故社区号与遍历无关、可重建。

孤立点（度 0，即 orphan）各自成单元素社区（不报 `isolated_community`，规模 < `MIN_COMMUNITY_SIZE`）。
空图 / 单节点：返回平凡社区，不崩。**同一 wiki 两次跑社区分配完全一致、`graph.json` 字节级一致**（§3.5）。

### 3.2 `graph.json` 富化（additive，契约不破）

```jsonc
{ "generated_from": "wiki/",
  "stats": { "nodes": 312, "edges": 540, "broken": 7, "orphans": 18,
             "communities": 9 },                         // ← 新增
  "nodes": [ { "id": "defi", "title": "DeFi", "type": "entity",
               "path": "wiki/entities/DeFi.md", "community": 3 } ],  // ← 每节点新增 community
  "edges": [ … ] }                                       // edges 不变
```

只**新增** `stats.communities` 与每节点 `community`；既有键/顺序/排序一字不动（决策P3.5-4）。`dump_json` 的字节
稳定性靠确定性社区号保持。

### 3.3 `graph.html` 富化（仍守决策P3-7：零 JS、零第三方库、纯静态列表）

**不引**力导向/sigma.js（决策P3-7 明确"无图形布局算法、纯列表结构"，P3.5 不破）。只在既有列表上加：

- 每个节点条目尾部一个社区徽标 `<span class='comm'>社区 3</span>`；
- 顶部摘要行加 `· 社区 9`；
- 末尾一个 **「拓扑提示」** 段，确定性列出枢纽 / 脆弱桥 / 孤岛（与 lint 同数据，文字呈现）。

所有内容 Python 端按稳定排序生成，天然字节稳定。

### 3.4 三类拓扑 `Finding`（建议非门禁）

在 `run_lint` 已建好的 `g` 上追加（接在现有 orphan/broken/missing_entity 之后，保既有顺序）：

- **`lint.hub_node`**（god node）：度 ≥ 均值+`HUB_SIGMA`σ 且 ≥ `HUB_MIN_DEGREE`。
  记在该页：`Finding(page, "lint.hub_node", "无向度 N（≥ 均值+2σ），疑为过载枢纽，考虑拆分")`。
- **`lint.fragile_bridge`**：两社区仅靠单边相连。**全局 finding**（`page=""`，跨两页，同 missing_entity 体例）：
  `Finding("", "lint.fragile_bridge", "社区 A↔B 仅靠 [[u]]—[[v]] 单边相连，建议补交叉引用")`。
- **`lint.isolated_community`**：规模 ≥2 且与全库零跨社区边的孤岛。**全局 finding**：
  `Finding("", "lint.isolated_community", "社区{页 a/b/c}与其余 wiki 零互链（孤岛），建议补桥接")`。

阈值不达 / 图太小（节点数 < 阈值地板）时自然一条都不报——小库不被噪声淹没（决策P3.5-8）。

### 3.5 确定性与幂等

社区号规范化（§3.1 步骤 5）+ 所有列表稳定排序（按 id / (u,v) / 社区号）⇒ 同 wiki 两次：社区分配一致、
`graph.json` 字节一致、`graph.html` 字节一致、lint findings 顺序一致。与 `graph`/`lint` 既有幂等承诺同级。

## 4. 决策（沿用仓库 `决策PX-N` 体例）

- **决策P3.5-1（只做零 LLM 拓扑，排除 LLM 推断边）**：见 §0。推断边破坏 graph 可重建不变式，另开。
- **决策P3.5-2（确定性 Louvain）**：见 §3.1。固定遍历序 + 确定性 tie-break + 规范化重编号 + 无 RNG，
  守 `graph.json` 字节稳定。手写纯 Python、零新依赖。
- **决策P3.5-3（复用 build_graph 邻接表单一归口）**：graphstats 接 `Graph`，`graph` 与 `lint` 共用，不二次建图。
- **决策P3.5-4（graph.json additive）**：只增 `community`/`communities`，既有契约一字不动。
- **决策P3.5-5（graph.html 守决策P3-7）**：仍零 JS / 零第三方库 / 纯静态列表，只加文字注记与拓扑提示段，
  不引力导向布局。
- **决策P3.5-6（phantom-hub ≡ missing_entity，不重复）**：见 §0。被多页引用却无页的"幻影枢纽"就是既有
  `lint.missing_entity`（由 heal 物化）；P3.5 不另检测、不另命名，复用归口。
- **决策P3.5-7（拓扑 finding 走 lint 建议非门禁）**：新种类进既有 `Finding`/`report_json`/`EXIT_LINT_FINDINGS`，
  默认退 0、`--strict` 才 6——不引新码。
- **决策P3.5-8（阈值常量化 + 小库静默）**：`HUB_SIGMA`/`HUB_MIN_DEGREE`/`MIN_COMMUNITY_SIZE`/
  `LOUVAIN_RESOLUTION` 常量化；阈值与地板度数保证小库不被噪声淹没。"该不该连"的**语义**判断
  （surprising connections / should-connect 配对）需 LLM，属语义 lint，**推后**（与矛盾/过期论断同族）。
- **决策P3.5-9（无向化算拓扑）**：社区/桥/枢纽在 resolved 边的无向投影上算；方向只在建边时有意义。
- **决策P3.5-10（基于 resolved 边）**：断链不算连通（与 `compute_orphans` 口径一致，graph.py:151）。

## 6. 边界（明确不做）

- **不做 LLM 推断边**（决策P3.5-1，破坏 graph 可重建性，另开）；
- **不做语义 surprising-connections / should-connect 判断**（需 LLM，语义 lint 一族，推后）；
- **不重复检测 phantom-hub**（≡ 既有 `lint.missing_entity`，决策P3.5-6）；
- **不引力导向可视化 / 第三方 JS 库**（守决策P3-7，决策P3.5-5）；
- **不引外部图算法依赖**（手写确定性 Louvain）；
- **不加子命令 / 不加退出码 / 不写 `log.md`/`raw/` / 不起 LLM**。

## 7. 测试要点（扩 `tests/test_graph.py` + `tests/test_lint.py`，或新 `tests/test_graphstats.py`）

- **确定性**：同一 wiki 连跑两次，社区分配完全一致、`graph.json`/`graph.html` 字节一致。
- **社区正确性**：两个团（clique）由单边相连 → 恰 2 社区，且那条边报 `lint.fragile_bridge`。
- **枢纽**：星形图中心节点度高 → `lint.hub_node`；度未过阈值的普通节点不报。
- **孤岛**：两个互不相连的多节点簇 → 各报 `isolated_community`（规模 ≥2）；单节点孤儿不报（走 `orphan`）。
- **归口不重复**：被 ≥2 页引用的未解析目标仍只报 `lint.missing_entity`，**不**新增 phantom-hub 项。
- **additive 契约**：`graph.json` 新增 `community`/`communities`，**既有键/顺序/排序与现有 test_graph 全绿**
  （回归）；orphan/broken/missing_entity findings 一字不变。
- **`--strict`**：含拓扑建议时退 `EXIT_LINT_FINDINGS(6)`；默认退 0。
- **小图/空图/单节点**：不崩、不报噪声 finding；社区数合理（空图 0、单节点 1）。
- **无向 & resolved 口径**：断链边不参与连通/社区；自环不影响度统计（与 `compute_orphans` 一致）。
