## 0. 一句话

`guanlan audit`：一个**确定性粗筛（零 LLM）→ LLM 复核（复用 P2 写门禁）**的两层命令，对既有 wiki 做语义体检——先用结构信号圈出**疑点页**（零 LLM、可 `--dry-run` 纯读预览），再只对疑点交 Agentao+skill 做语义判断，确认则就地标 `## ⚠️ 矛盾与存疑` / 更新过期论断，驳回则记一笔。**精确复刻 P3.2 的 `lint.missing_entity`（检测）↔ `heal`（物化）配对**，只是把"检测"从结构升到语义触发、把"物化"换成"复核标注"。

## 1. 目标与范围

### 为什么不照搬 swarmvault

swarmvault 的语义层建在一张 **claim 图**上：每条抽取的论断带 `polarity ∈ {positive,negative,neutral}` + `confidence`，矛盾检测先零-LLM 用 polarity + token Jaccard≥0.3 粗筛候选、再交 LLM 复核（`analysis.ts` / `vault.ts:detectContradictions`）。这套有两个对 guanlan 致命的问题：

1. **它要在 wrapper 里重建一个 claim/polarity 数据模型** —— 直接违反 guanlan「`guanlan/` 不携带业务智能」的红线（CLAUDE.md 架构 §1）。论断抽取、极性判定本身就是语义活，本该归 LLM，不该塞进确定性脚本。
2. **它的极性检测器是英文正则**（`analysis.ts:detectPolarity`，`return "negative"` 命中英文否定式）。搬到 guanlan 的 CJK + 2-gram 语料上只会**高噪**——一个天天误报的 advisory 会被训练成噪音被无视（同 ESLint 噪音规则的命运）。

所以本设计**只借形状、不借实现**：双层分诊里 **Layer-1 只做最廉价的结构粗筛**（不重建 claim 图），**所有语义判断下放给 Layer-2 的 LLM**（Agentao + skill）。这正是 guanlan 在 P3.2 已经验证过的分工——`missing_entities()` 只做结构聚合，"这页该写什么"全交给 `heal` 的 LLM。

### 纳入 P3.7（本设计落地的）

- **`guanlan audit`** 命令：worklist（零 LLM）+ LLM 复核（复用门禁），契约对齐 `heal`。
- **第一类语义疑点：过期论断（source-drift）** —— 作为本阶段**唯一跑通的 worked example**（决策P3.7-2）。触发信号是结构的、零-LLM、低噪：页所引 `raw/` 源的内容指纹相对**建页时记录的指纹**已变 → 该页论断**可能**过期，交 LLM 复核。
- **一项新 provenance 约定**：**wrapper** 在 ingest 过完门禁后，把所摄 raw 文件的内容指纹 stamp 进对应 `wiki/sources/<slug>.md` 摘要页的 `raw_digest` 字段（决策P3.7-3，**非 Agent 写**）。这是整个阶段的**拱心石**——没有"建页时的源版本"这个锚，源漂移无从判定。

### 不纳入 P3.7（同框架的兄弟，留给 P3.7.x）

- **矛盾复检**：高价值、但 Layer-1 触发信号更难做到低噪——**已在 §11.1 展开为 P3.7.1 兄弟小设计**（复用同一 `audit` 命令、同一源指纹 provenance，主信号"多源页+源漂移"零新成本），按本设计放在 source-drift 跑通**之后**接入。
- **资料缺口**：与既有 `lint.missing_entity` / `health.stub_page` 大面积重叠，价值增量小，暂不单列。
- 不引入 claim/polarity 数据模型、不引入 confidence 数值、不引入衰减分写回页面（swarmvault `decayScore` 那套要求宿主往页写派生态，见 §10 风险）。
- 不新增退出码、不碰 `check` 门禁语义、不动 `raw/`。

## 2. 双层分诊架构（对齐 P3.2，不另起炉灶）

```
                既有 P3.2                      本设计 P3.7
  ┌─────────────────────────┐      ┌─────────────────────────────┐
  │ Layer-1  零 LLM 归口      │      │ Layer-1  零 LLM 归口          │
  │ missing_entities()       │  ⇒   │ audit_candidates()           │
  │ ＝ 结构聚合断链           │      │ ＝ 结构比对源指纹 → 疑点页    │
  └───────────┬─────────────┘      └───────────┬─────────────────┘
              │ 共享 worklist                    │ 共享 worklist
  ┌───────────┴─────────────┐      ┌───────────┴─────────────────┐
  │ Layer-2  LLM via 门禁     │      │ Layer-2  LLM via 门禁         │
  │ heal: 物化缺失页          │  ⇒   │ audit: 复核并就地标注/更新     │
  │ run_guarded_write_result │      │ run_guarded_write_result     │
  └─────────────────────────┘      └─────────────────────────────┘
```

复用点（逐一对齐 `heal.py`）：

- **零-LLM 归口被两处共享**：`heal` 里 `missing_entities()` 同时喂 `lint`（advisory 展示）与 `heal`（worklist）。本设计 `audit_candidates()` 同时喂 `audit --dry-run`（advisory 预览）与 `audit`（worklist）。**单一口径、永不分叉**（决策P3.2-1 的复用）。
- **写路径复用 P2 单写者门禁**：`audit` 的 LLM 写走 `gate.run_guarded_write_result`（与 `ingest`/`heal`/`backfill` 同一条编排核心、同一 `raw/` 快照前后比对）。**不另开门禁**。⚠️ 但 `run_guarded_write_result` 的 `page_guard` **默认 `False`**（`gate.py:423`；heal 直调即取默认 False 逐字节，决策P2.1-10）——audit 要"既有 `sources` 只增不减"的 P2.1 源不回退闸，**必须显式传 `page_guard=True`**（决策P3.7-4），**不会自动继承**。
- **结构化核 + 薄壳**：`run_audit_result(...) -> AuditRun`（不打印的编排核，仿 `run_heal_result`）+ `run_audit(...)` CLI 薄壳 + `audit_entrypoint`。`--dry-run` 纯读零-LLM 不进核（仿决策P3.2-5）。
- **写后确定性回执**：复核完成后零-LLM 复查（§7），回答"这页到底被标了 / 被更新了没"，不靠 LLM 自述。

## 3. Layer-1：确定性粗筛 `audit_candidates()`（零 LLM）

签名（仿 `missing_entities()` / `compute_worklist`）：

```python
@dataclass(frozen=True)
class AuditCandidate:
    page: str                       # 相对库根 posix（source 摘要页 或 引用它的页）
    reason: str                     # "source-drift"（直接） / "cites-drifted-source"（传播）
    drifted_slugs: tuple[str, ...]  # 触发本候选的、已漂移的 source slug（升序）

def audit_candidates(wiki: Path, raw: Path) -> list[AuditCandidate]:
    ...
```

判定分**两步**（全程读文件 + 比 hash，零 LLM、确定性、可字节稳定排序）：

**第一步——找漂移的 source 摘要页**：遍历 `wiki/sources/*.md`，对每张带 `raw_digest` 的页：
- 解出其 `raw_digest` 的 `<raw路径>@sha256:<记录值>`，**先过路径准入**（决策P3.7-12：须相对库根、以 `raw/` 开头、`realpath` 解析后仍在 `<kb>/raw/` 下；绝对路径 / `../` 越界 / 不以 `raw/` 起 / 格式坏 → **当无信号跳过**，不报错不崩），通过后读该 raw 文件现字节、算现 sha256，比对：
  - 不等 → 该 source slug **已漂移**，进漂移集 `S`；摘要页本身记一条候选 `reason="source-drift"`。
  - raw 文件已不存在 → **不在此报**（那是 `check.sources.unresolved` / 缺源的活，正交不抢口径）。
  - 页**无** `raw_digest`（旧页 / P3.7 前建）→ **跳过不罚**（向后兼容，决策P3.7-5；同 swarmvault `lastConfirmedAt` 缺失不罚）。

**第二步——沿 `sources:` slug 图传播到引用页**：`iter_pages(wiki)` 遍历非 config 页（同 `health`/`check` 口径），对每页读 `sources:`（容错档 `load_page`），若其 slug 集与 `S` 有交 → 记一条候选 `reason="cites-drifted-source"`、`drifted_slugs` = 该交集。

输出按 `(page)` 升序稳定。**指纹只在 source 页比一次**（不是每张引用页各算各的），引用页只做廉价的 slug 集合求交——这正是把指纹收在 source 页一份的红利。

**候选天然按漂移源成组**（决策P3.7-8 的结构基础，**无需新字段**——组从既有 `drifted_slugs` 即可导出）：一个漂移 slug `s` 产出**一组**候选 = {`wiki/sources/<s>.md` 摘要页本身（`source-drift`）} ∪ {所有 `sources:` 含 `s` 的引用页（`cites-drifted-source`）}。这一组是 Layer-2 复核与指纹刷新的**原子单元**：刷新 `s` 的 `raw_digest` 必须等**整组都复核完**（§4.2 / §7），否则会把没轮到的引用页（尤其 `--limit` 没覆盖到的）连带从候选里误清。一张引用页可同属多组（它 `drifted_slugs` ≥2）——它复核一次即同时计入各组，某源的组**当且仅当其全部成员都被复核**才算完成。`audit_candidates()` 仍返回 per-page 候选，分组只是消费侧（worklist / 刷新）按 `drifted_slugs` 聚合，**不动 `AuditCandidate` 形态**。

**为什么这个信号配做 Layer-1**（反向评审的唯一幸存者）：纯**结构**——只比 hash + 求 slug 交集，无 NLP、无 claim 图、对 CJK 无偏。低噪：只有 raw 真被替换过的源才触发；没被覆盖过的库 `audit` 一条不报（§4.4）。召回够用：一个源一漂移，它的摘要页 + 所有引它的页都被点名，逐页交 Layer-2 判"到底过期没"。

## 4. 拱心石：`raw_digest` provenance（**已定稿**的约定）

读 `check.py` / `conventions.md` / `ingest.py` 后，上一轮初稿这节有三处错，逐一校正后定稿如下。

### 4.0 先纠三处错（只有读码才看得出）

1. **`sources:` 列的不是 raw 路径，是 `wiki/sources/<slug>.md` 摘要页的 slug**（`check._check_sources`：每个 slug 必须解析到 `wiki/sources/<slug>.md`；conventions §frontmatter：「列 source 页的 slug，不含路径/扩展名」）。真实链条是 `entity/concept 页 → sources:[slug] → wiki/sources/<slug>.md 摘要页 → 摘 raw/<file>`。
2. **故源指纹不该复制到每张引用页**（上一轮 `source_digests:{slug:hash}` 既冗余、又会多页各记一份互相打架）——它该**只落在 source 摘要页一份**，那里与 raw 文件 1:1 绑定。漂移经既有 `sources:` slug 图**传播**到引用页，不必各记。
3. **不能让 Agent 写指纹**：`ingest.py:INGEST_PROMPT` 明令 Agent「**不要运行 shell 命令**」，而 LLM 不可能可靠地算/逐字抄一条 sha256。指纹**只能由 wrapper 确定性算**。

### 4.1 字段形态（定稿）

`source` 型页（`wiki/sources/<slug>.md`）增一个**可选标量** frontmatter 字段：

```yaml
raw_digest: 'raw/2024-财报.md@sha256:1a2b3c…'   # 本摘要页所摘 raw 文件 + 被摘时的内容指纹
```

- **单标量**（`<raw相对路径>@sha256:<hex>`）、单引号、路径+hash 自含；**一张 source 页一条**。entity/concept/synthesis 页**不带**此字段。
- 选标量而非 mapping：YAML 最稳（conventions 力推单引号标量），且 source↔raw 恒 1:1、无需 mapping。含路径是为稳健——对 `convert` 改名 / CJK slug 不靠"猜 raw 名"。
- **写入绝不手拼**（决策P3.7-9）：上面的单引号只是示意；落盘**绝不裸拼** `raw_digest: '<值>'`——raw 路径可能含 `'`/`:`/`#`/空格（CJK 文件名常见），裸拼会生成坏 frontmatter。一律经 YAML-safe 归口序列化（`yaml.safe_dump` 标量，**复用 `rawio.apply_origin` 写 `origin` 的同款做法**，`rawio.py:130`），写后再 check。

### 4.2 谁来写指纹：wrapper 门禁后直写（Option A，已定稿）

> **原全文唯一未拍板项（未决P3.7-3a）已定：取 Option A——指纹由 wrapper 在门禁后确定性直写 source 页，不交 Agent。** 与被否的 Option B 的取舍见 决策P3.7-3a（§8）。下文（及 §3/§5/§7/§9）按此定稿铺陈。

- ingest 成功**过完 P2 门禁后**，wrapper 对本次摄入的 `raw/<stem>.md` 算字节 sha256，经 **YAML-safe 归口**（`split_frontmatter` → 设 `raw_digest` → `yaml.safe_dump`，决策P3.7-9）stamp 进 `wiki/sources/<stem>.md`（按 conventions 既有 slug==rawname 约定定位），**写后对该页跑一次 check** 确认 frontmatter 仍合法。定位不到 / 该页 frontmatter 本就不可解析 / 写后 check 不过 → **回滚到写前字节、记一句降级，不报错、不阻断 ingest**（既有降级姿态扩到"宁可不 stamp 也绝不留坏页"）。
- **只 stamp 本次摄入那一张 source 页**——**绝不**顺手刷新别的 source 页指纹（否则会把别处真实漂移一起抹平，是致命 bug）。
- Option A 的代价（也是当初要拍板的越线点）：**头一回让 wrapper 对 *内容* 页做确定性写**（以往 wrapper 只 verify，仅 `reindex` 写过 `index.md` 那种 *config* 页）。接受这一小步，换来零 LLM、幂等、确定、零腐败的指纹——正当性见 决策P3.7-3a。
- `audit` 刷新指纹是 **按漂移源分组原子**的（决策P3.7-8）：仅当某漂移 slug `s` 的**整组**（`s` 的 source 摘要页 + 全部引 `s` 的页）都在本批**复核留痕**后，**wrapper** 才把 `wiki/sources/<s>.md` 的 `raw_digest` 刷成现值（= swarmvault `resetDecay` 的 provenance 版，零数值衰减；同款 YAML-safe 归口 + 写后 check）。**"复核留痕" = 该页在本批新增的 `log.md` audit suffix 里有对应 JSON 行、且其 `drifted_slugs` 与 target 精确一致（决策P3.7-10，"本批"由写前快照界定、行格式为单行 JSON、slugs 精确匹配——决策P3.7-13/14/15），不是"门禁通过"**——门禁只证文件合法、不证 Agent 逐页读过（"仍准"又允许零正文改动，故 gate 通过 + 无改动 ≠ 已复核）。组没复核全（含 `--limit` 没轮到、组内某页报错、或某页**漏写 log 行**）→ **整组不刷新、留待下次**。**绝不复核一页就刷源指纹**——`raw_digest` 是 source 级、复核是 page 级，提前刷会经 slug 传播把同源其余未复核引用页连带误清出候选。刷新与 stamp 同主体（wrapper，Option A 一以贯之）。

### 4.3 它是 provenance、不是派生索引（守住不变量）

记的是"本摘要页建自该 raw 的**这个版本**"——历史事实，性质同学术引用版次、同 `convert` 写进 raw frontmatter 的 `origin`。它**不需**从当前 markdown 可重建（不是对当前内容的派生），故不违反「markdown 唯一真相 / 派生物可幂等重建」。

### 4.4 raw 不可变，漂移从哪来？（必须讲清的前提）

guanlan 的 raw **只在 agent 写期间被快照硬锁**；但**人手替换 / `convert --overwrite` / Web 投喂覆盖**可在两次 ingest 之间换掉一个 raw 文件。`source-drift` 抓的正是这个事件：**「重转/替换了源、但 wiki 页还没重新综合」**。典型用例:拿更新版 PDF `convert --overwrite` 盖掉旧 `raw/`，`audit` 即告诉你哪些 wiki 页该重摄。若一个库的 raw 永不被覆盖，`audit` 永远空跑——这是**正确**的（无事件即无候选），不是缺陷。

### 4.5 check 契约不变 + 向后兼容

- **`check` 完全不动**：`raw_digest` 是可选未知字段，`check` 既不要求、也不校验它——**不新增 Violation、门禁字节稳定**（决策P3.7-3b）。合法性由 `audit` 自己宽松读（缺/格式坏 → 当无信号跳过），不进门禁。**正因 check 不校验它**，audit 宽松读时**必须防手写字段越界**：raw 路径须过 决策P3.7-12 的准入（相对 + `raw/` 前缀 + realpath 仍在 `<kb>/raw/` 下），越界/绝对/坏格式一律当无信号跳过。
- 无 `raw_digest` 的旧 source 页 → 无漂移信号，下次 ingest 触及时由 wrapper 自然补 stamp。**安全退化：少报不误报**（决策P3.7-5）。
- `conventions.md` / `examples/SCHEMA.md`：把 `raw_digest` 记为 source 页上**wrapper 托管、请勿手改**的 provenance 字段；Agent 知其存在但不写、不读它做判断。

> 决策P3.7-3 是本阶段**唯一动数据模型**的改动，至此**全部定稿**：字段在 source 页、单标量含路径、只 stamp 本次那一张、check 不可见、向后兼容跳过，**写入主体取 wrapper 门禁后直写（Option A，决策P3.7-3a）**。

## 5. Layer-2：LLM 复核（复用 P2 门禁 + skill 工作流）

`AUDIT_PROMPT`（仿 `HEAL_PROMPT`，薄；真步骤在 skill）：

```
请按 `guanlan-wiki` skill 的 audit 工作流，逐一复核下列目标页。**每行已由 wrapper 钉好 `page | reason | drifted_slugs | raw_paths`，你无需自己推断漂移源或 raw 路径**：
{targets}
对每页：读该页正文 + 该行 `raw_paths` 列出的 `raw/` 源**现版本**，判断页中论断是否仍被现源支持。
 - `reason=cites-drifted-source`：对照本页跨这些源的综合是否仍成立；`reason=source-drift`（该行就是 source 摘要页自身，其 raw 在它自己的 `raw_digest` 里、不在 `sources:`）：对照本页摘要与其 raw 现版本。
 - 仍准：无需改正文（指纹刷新由 wrapper 在你返回后自动处理，**你不要碰 raw_digest**），status=`confirmed`。
 - 已过期 / 现源与页冲突：就地按 conventions 标 `## ⚠️ 矛盾与存疑`(status=`flagged`)，或最小化更新该论断(status=`updated`)，其余正文一字不动。
 - **逐页留痕（强制）**：在唯一一条 `## [<日期>] audit | <一句话批次说明>` 标题下，对**每个目标页**追加一行**单行 JSON** `- {"page":"<相对库根 posix>","drifted_slugs":["…"],"status":"confirmed|flagged|updated"}`（一行一个对象、不折行；`drifted_slugs` **照抄该 target 行给你的那串、不增不删**——少写会被判未复核，决策P3.7-15）。这是 wrapper 判定"整组复核完"的**唯一凭据**，**漏写 = 该组不刷新指纹**；本次只能新增这一段 audit 标题，不要改历史。
遵循 AGENTAO.md 硬约束。**永不修改 `raw/`。** 不要运行 shell 改文件、不要自跑 check（wrapper 会强制校验）。
完成后用一两句说明哪些页确认、哪些标了存疑。
```

- **指纹刷新不进 prompt**：`raw_digest` 由 wrapper 在门禁后确定性 stamp（§4.2），**不交 Agent**——LLM 抄 hash 不可靠且 ingest 禁 shell。prompt 只让 Agent 做语义判断 + 标注/更新正文。
- 走 `run_guarded_write_result(..., page_guard=True)`（**显式开**，决策P3.7-4——该核心默认 `page_guard=False`）：Agent 只能写 `wiki/`，`raw/` 由快照前后比对硬拦；既有 `sources:` slug 由 P2.1 源不回退闸保只增不减。
- `{targets}` 由确定性 `_format_targets` 归口生成（决策P3.7-11）：本批选中**若干完整漂移源组**去重后的页集（`--limit` 限**组**数不限单页，决策P3.7-8；一页同属多组只列一次、只复核一次），**每行钉死 `page | reason | drifted_slugs | raw_paths`**——`raw_paths` 由 wrapper 从相关 source 页 `raw_digest` 预解析（**含 source 页自身**：它的 raw 在自己的 `raw_digest`、不在 `sources:`）。Agent 不推断漂移源、不解析 hash。
- **"整组复核完"的可验证判据（决策P3.7-10）**：刷新前 wrapper 先按**写前 `log.md` 快照**取本次新增 suffix（`before_log` → append-only 校验 → 仅解析 suffix，决策P3.7-13），再从 suffix 内**恰好一个**新 `## [<日期>] audit` 段逐行 `json.loads` 那些 `- {…}` JSON 行（决策P3.7-14）；**每行的 `drifted_slugs` 须与 `_format_targets` 给该 `page` 的 `drifted_slugs` 排序后完全一致**（决策P3.7-15）才算该页有效留痕——不一致 / `page` 不在本批 targets → 视作未留痕。**唯有某漂移源组全部成员页都有有效留痕**才刷新其 `raw_digest`。门禁通过只证文件合法、**不证 Agent 逐页读过**；漏写行 / slugs 不匹配的页视作未复核、其所在组不刷新（自愈：下次整组重进候选）；suffix 非 append-only / 不是恰好一段 / 任一行非合法 JSON → **任何组都不刷新**（整批留候选）。不靠 LLM 自述、不加每页 ack 字段、不新建数据文件——只把既有"一条批次标题"细化为"标题 + 逐页 JSON 行"。
- 空 worklist 短路 `EXIT_OK`、不触 Agentao（仿决策P3.2-6）。
- 复核**就是 ingest 已会的动作**：`ingest.py:INGEST_PROMPT` 里"发现矛盾就地标 `## ⚠️ 矛盾与存疑`"已是现成行为——`audit` 只是把它从"摄入时顺手"变成"可定向、可批量、可复查"。**不教 LLM 新把戏**。

## 6. CLI 契约（逐项对齐 `heal`）


## 7. 写后回执 / 幂等 / 不变量

- **写后确定性回执**（零 LLM 可验证，仿 `_build_receipts`）：复核后 wrapper 以**写前 `log.md` 快照**为界取本次新增 suffix（`before_log` + append-only 校验，仿 `heal.py:421`/`:433`，决策P3.7-13），从中恰好一段 audit 标题下逐行 `json.loads`（决策P3.7-14）、**校验每行 `drifted_slugs` 与该 `page` 的 target 精确一致**（决策P3.7-15）得"已复核页集"，**只对全部成员页都有有效留痕的漂移源组**刷新其 source 页 `raw_digest`（决策P3.7-8），再重算 `audit_candidates()`——完整复核的组应**整组离开**候选集；漏写行 / slugs 不匹配 / 组没复核全 / suffix 解析失败者其整组仍在集内 = 待下次，回执如实标"未复核 / 组未完成"。**判据是本批新增 log suffix 里的 JSON 行（page + 精确 drifted_slugs）、不是门禁通过、更不是 LLM 自述。**
- **幂等**：连跑两次 `audit`，第二次完整复核过的组 worklist 为空（指纹已由 wrapper 刷新）→ 短路、零写、零 Agentao。raw 再被替换才再进候选。
- **不破坏既有页**：P2.1 的"既有 `sources` 只增不减 + 正文骤缩告警"**不是自动继承的**——`run_guarded_write_result` 默认 `page_guard=False`，audit **必须显式 `page_guard=True`** 才得到这层防护（决策P3.7-4）。
- **markdown 唯一真相**：本阶段零新派生态文件。`raw_digest` 是 source 页内 provenance（§4.3），随页走、随页重建，不构成需独立重建的索引。
- **零 LLM 脚本 vs LLM 工作流**：Layer-1（比 hash + 求 slug 交集）零 LLM；Layer-2（语义判断）全经 Agentao。指纹计算与 stamp 是 wrapper 确定性代码，**不持 LLM client**。红线全守。

## 8. 决策记录

- **决策P3.7-1**：语义 lint **不进 `lint`**，新开 `audit` 命令（LLM 档），与 `heal` 平级。理由：`lint.py:19` 自承 lint 零 LLM 只做结构；把 LLM 判断塞进 lint 会破其教义。`lint`/`health` 至多展示零-LLM 的**触发信号**（`health.source_drift`），复核动作归 `audit`。
- **决策P3.7-2**：本阶段**只跑通"过期论断 / source-drift"一类**作 worked example。理由：它是反向评审里唯一"触发信号纯结构、零 LLM、对 CJK 无偏、低噪"的幸存者；矛盾复检触发更难低噪（§11.1），留兄弟项。
- **决策P3.7-3（已定稿，本阶段唯一动数据模型）**：引入 `raw_digest` provenance——**`source` 摘要页**上的可选单标量 `'<raw路径>@sha256:<hex>'`，记本页所摘 raw 的版本指纹。理由：`sources:` 是指向 source 页的 slug（非 raw 路径），故指纹只该在 source 页一份、经 slug 图传播；它是出处事实非派生索引（§4.3），是 source-drift 可判定的拱心石。
  - **决策P3.7-3a（已拍板：取 Option A）**：`raw_digest` **由 wrapper 在门禁后直写 source 页**，不交 Agent。两个选项的共同前提（都成立）：**只 stamp 本次摄入那一张 source 页**、绝不顺手刷新别页（否则抹平别处漂移）。分歧在写入主体，最终取 A：
    - **Option A（采纳）——wrapper 门禁后直写 source 页**。优：值由代码算、零腐败、零 LLM、幂等确定。缺：**头一回让 wrapper 碰 *内容* 页**（以往 wrapper 只 verify；仅 `reindex` 写过 `index.md` 那种 *config* 页），越过"Agent 写 wiki、wrapper 只验"旧默契一小步。
    - **Option B（否决）——wrapper 算值塞进 prompt、Agent 用文件工具写 + wrapper 回读校验**。守住"Agent 是 wiki 唯一写者"，但要 LLM 逐字抄 64 位 hex，**易腐**，须加"写后回读比对、不符则 wrapper 兜底改正或报错"一整层。
    - **取 A 的理由（可靠性压倒纯粹性）**：指纹是不可变 raw 的 sha256，**性质纯 provenance、非内容创作**；Agent **结构上做不到**——`INGEST_PROMPT` 明禁 shell，LLM 也抄不准 64 位 hex；越线幅度极小且有先例（类比 `reindex` 让 wrapper 确定性写 config 页），相比 B「LLM 抄 hash + 回读兜底」一整层易腐机制，A 简单且零腐败。
    - **判例边界（不可越界引用）**：A 越的这一小步只是「宿主对内容页做**纯 provenance 的确定性写**」，是一个**窄**判例——它**不等于**"宿主可任意写/删 wiki 内容"。P3.9 源撤回那种**删除**内容页仍是更重的一步，须各自评估、不由本决策自动获批（见 [`backlog/notes/未来工作计划-反向评审收口.md`](backlog/notes/未来工作计划-反向评审收口.md) §3）。
  - **决策P3.7-3b（已定稿）**：`check` **完全不动**——`raw_digest` 对 check 不可见、不校验、不阻断；合法性由 `audit` 宽松自读。门禁字节稳定、不新增 Violation。
- **决策P3.7-4（评审加固·中）**：复用 `run_guarded_write_result` 写路径，**不另开门禁、不新增退出码**。⚠️ 该核心 `page_guard` **默认 `False`**（`gate.py:423`；heal 直调即取默认 False 逐字节，决策P2.1-10），故 audit **必须显式传 `page_guard=True`** 才启用 P2.1 源不回退闸——"继承 P2.1"**不会自动发生**（原稿 §2/§5/§7 误称自动继承，已改）。
- **决策P3.7-5**：无 `raw_digest` 的旧 source 页**跳过不罚**（向后兼容），随下次 ingest 触及自然补 stamp。
- **决策P3.7-6**：**不引入 claim/polarity/confidence/衰减分**。所有语义判断下放 LLM，wrapper 只比 hash。明确与 swarmvault `analysis.ts` 路线分道。
- **决策P3.7-7**：`audit` 确认页仍准时刷新该 source 页 `raw_digest`（= `resetDecay` 的 provenance 版），使幂等成立、避免同页反复进候选。**刷新由 wrapper 执行**（与 stamp 同主体，决策P3.7-3a / Option A）；**刷新时机受 决策P3.7-8 的分组原子约束**。
- **决策P3.7-8（评审加固·高）**：指纹刷新**按漂移源分组原子**，`--limit` 限**源组**数、不限单页。一个漂移 slug 的组 = 其 source 摘要页 + 全部引它的页；唯有**整组复核成功**才刷新该源 `raw_digest`，否则整组留待下次。理由：`raw_digest` 是 **source 级**、复核是 **page 级**——若复核一页就刷源指纹，会经 slug 传播把同源**其余未复核**引用页（尤其 `--limit` 没覆盖的）一并误清出候选，造成"看似审完、实则漏审"的高危静默缺口。**不引入每页 ack**（那会把设计复杂化）——分组原子已足够，且**天然自愈**（没刷新的组下次重新整组进候选）。`audit_candidates()` 仍返回 per-page 候选、`AuditCandidate` 形态不变，分组在 worklist/刷新消费侧按 `drifted_slugs` 聚合。
- **决策P3.7-9（评审加固·中低）**：两处 stamp（ingest 后初 stamp / audit 后刷新）都经**同一 YAML-safe 归口**写入——`split_frontmatter` → 设 `raw_digest` → `yaml.safe_dump(…, sort_keys=False, allow_unicode=True)`，**绝不裸拼** `raw_digest: '<值>'`（raw 路径含 `'`/`:`/`#`/CJK 空格会生成坏 frontmatter）；**复用 `rawio.apply_origin` 写 `origin` 的同款做法**（`rawio.py:130`，已是验证过的归口）。**写后对该页跑一次 `check`** 确认 frontmatter 仍合法；不过 / 该页 frontmatter 本就不可解析 → **回滚到写前字节**（ingest：记降级、不阻断；audit：该源不刷新、整组留待下次）。注意这只是"确认没把页写坏"——**`check` 仍不认识 `raw_digest` 字段本身**（决策P3.7-3b 不变：不校验其语义、不新增 Violation）。
- **决策P3.7-10（评审加固·高）**："整组复核完"以**结构化 `log.md` 行**为唯一可验证判据，不靠门禁通过、不靠 LLM 自述。Agent 须在唯一一条 `## [<日期>] audit | <说明>` 标题下，对**每个目标页**追加一行（行格式见 决策P3.7-14 的单行 JSON）；wrapper 刷新前确定性解析这些行（"本批"界定见 决策P3.7-13），**唯有某漂移源组全部目标页都留痕**才刷新其 `raw_digest`。理由：gate 通过只证文件合法、"仍准"又允许零正文改动——二者都不能证明 Agent 真读过组内每页；若 Agent 跳过某页而 gate 仍过，缺 log 行判据就会误刷源指纹、清掉未审页（评审 finding 1）。**不引入新数据文件、不加每页 ack 字段**——复用既有 `log.md`，只把"一条批次标题"细化为"标题 + 逐页行"。
- **决策P3.7-11（评审加固·中）**：`{targets}` 由确定性 `_format_targets` 归口钉死，**每行含 `page | reason | drifted_slugs | raw_paths`**，`raw_paths` 由 wrapper 从相关 source 页 `raw_digest` 预解析。理由：source 摘要页自身是 `source-drift` 候选，但它的 raw 在**自己的 `raw_digest`** 里、**不在 `sources:`**——原 prompt"读该页 `sources:` 中漂移源对应 raw"对 source 页**不适用**（评审 finding 2）。把漂移源与 raw 路径全由 wrapper 供给，Agent 不推断、不解析 hash，两类 reason 都有明确读取对象。
- **决策P3.7-12（评审加固·中）**：`raw_digest` 的 raw 路径**宽松读但严格准入**——仅接受"相对库根、以 `raw/` 开头、`realpath` 解析后仍在 `<kb>/raw/` 下"的路径（形状仿 P5.2.1 `_admit_image_ref`）；绝对路径 / `../` 越界 / 不以 `raw/` 起 / 格式坏 → **当无信号跳过**（不报错、不崩、不抢 `check` 缺源口径）。理由：`check` 不校验 `raw_digest`（决策P3.7-3b），手写/损坏的字段可能含越界路径，audit 读它去比 hash（甚至传给 Agent 当 `raw_paths`）前必须自挡路径穿越（评审 finding 3）。
- **决策P3.7-13（评审加固·高）**：「本批 log 行」严格由**写前 `log.md` 快照**界定，杜绝旧留痕误判。audit 像 heal 一样在调门禁前记 `before_log = _read_log(kb)`（`heal.py:421`），写后只解析 `after_log` 去掉 `before_log` 前缀的**新增 suffix**，且须 append-only（`after_log.startswith(before_log)`，同 `heal.py:433` 决策P3.2-11）。要求 suffix 内**恰好一个**新 `## [<日期>] audit | …` 段；**非 append-only / suffix 解析失败 / 不是恰好一段 → 任何组都不刷新**（整批留候选、下次重来）。理由：同日可能已有旧 audit 段——若只按"当天 audit 段"找留痕，Agent 本次漏写某页时会从旧段读到该页旧留痕、误判已复核而刷新（评审 finding 1）。
- **决策P3.7-14（评审加固·中）**：逐页留痕行用**单行 JSON** 而非 `|` 分隔——`- {"page":"<相对库根 posix>","drifted_slugs":["…"],"status":"confirmed|flagged|updated"}`。wrapper 去掉行首 `- ` 后 `json.loads`。理由：page 路径 / slug 理论上可含 `|`（手写 wiki 尤甚），`|` 分隔确定性解析会歧义；JSON 解析稳定、仍落在既有 `log.md`、**不引入新状态文件**。任一 bullet 非合法 JSON / 缺 `page` / `status` 非法 → 该行无效（叠加 决策P3.7-13 的"解析失败则整批不刷新"，评审 finding 2）。**注意 `{targets}`（wrapper→Agent、单向人读、不被代码回解析）仍用 `|` 表，无需 JSON**。
- **决策P3.7-15（评审加固·中）**：逐页 JSON 留痕的 `drifted_slugs` 须与 `_format_targets` 给该 `page` 的 `drifted_slugs` **排序后完全一致**，wrapper 才认其为该页有效留痕；不一致 → 该页视作未留痕（其所属的**所有**组都不算该页完成）。理由：一页可同属多个漂移源组（`drifted_slugs` ≥2）——若只按 `page` 集合判完成，Agent 对该页只留 `{"drifted_slugs":["a"]}` 而 target 实为 `["a","b"]` 时，**b 组会被误判完成、误刷指纹**（评审 finding）。精确匹配把 JSON 行从"我处理过这页"升为"我对照过这页的**这些**漂移源"，是有意义的断言；prompt 要求 Agent **照抄 target 行给的 `drifted_slugs`**（决策P3.7-11 已把它喂给 Agent），合规即天然一致。配套：该页所属各组都因它"未留痕"而不刷新——**安全偏严**（宁可整页留候选下次重审，不误清）。

## 9. 测试要点（仿 `tests/test_heal.py` + `test_lint.py`）

- `audit_candidates()`：raw 改 1 字节 → 其 source 页（`source-drift`）+ 所有引该 slug 的页（`cites-drifted-source`）全进候选 / raw 未动 → 空 / source 页无 `raw_digest` → 跳过不报 / raw 文件被删 → 不抢 `check` 缺源口径 / 多源页只一源漂移 → `drifted_slugs` 只列变的那个 / 候选按 page 升序字节稳定 / **只对 source 页比一次 hash**（断言引用页不重复算 raw hash）。
- wrapper stamp：ingest `raw/X.md` 成功后 → `wiki/sources/X.md` 被写入 `raw_digest`、值 = 该 raw 字节 sha256；**别的 source 页 `raw_digest` 一字不动**（防"顺手刷新抹平漂移"回归）；source 页定位不到 → 跳过、不报错。
- 向后兼容：现有不带 `raw_digest` 的 fixture 库跑 `audit --dry-run` → 零候选、`EXIT_OK`；跑 `check` 不受影响（`raw_digest` 不参与校验）。
- 门禁复用（mock runner）：`audit` 走 `run_guarded_write_result`、Agent 试改 `raw/` → 被快照拦；空 worklist → 短路不触 runner、`EXIT_OK`。
- 回执 / 幂等：mock 一次确认写 + wrapper 刷新指纹 → 复算候选该页离开；连跑两次第二次空。
- **分组原子刷新（决策P3.7-8，高危回归守门）**：一源漂移 → 其组 = source 页 + 引用页 A/B；`--limit 1` 选该组、mock **仅 A 复核成功而 B 报错** → 该源 `raw_digest` **不刷新**、source 页与 A、B **下次仍在候选**（防"复核一页误清全组"）；整组都成功 → 整组离开候选；**一页同属两源组**（`drifted_slugs` 两个）→ 复核一次计入两组、各组独立判完整（一组完成刷一源、另一组没完不刷）；`--limit` 计的是**组**数不是页数。
- **门禁 `page_guard`（决策P3.7-4）**：断言 audit 调 `run_guarded_write_result` 时**显式 `page_guard=True`**；Agent 复核中删掉既有 `sources` 一项 → 被 P2.1 源不回退闸阻断（反证：若按 heal 那样取默认 False 裸调，该删除不会被拦——此用例即守门）。
- **stamp 安全（决策P3.7-9）**：raw 路径含 `'`/`:`/空格 → stamp 后 frontmatter 可被 `yaml.safe_load` 重解析、`raw_digest` 值往返一致（验证不裸拼）；目标 source 页 frontmatter **预先损坏** → stamp 跳过 + 回滚 + 降级，**不留坏页**；mock 写后 check 失败 → 回滚、ingest 不阻断 / audit 该源不刷新。
- **整组判据 = log 行（决策P3.7-10/14，高危回归守门）**：mock Agent **门禁通过但漏写**组内 B 页的 JSON 行 → 该源 `raw_digest` **不刷新**、B（及整组）下次仍在候选（验证判据是 log 行、非门禁通过）；三页全留痕 → 整组离开；逐页行 `json.loads` 出 `{page,drifted_slugs,status}`、status 取值合法（confirmed/flagged/updated）；**某行非合法 JSON / page 含 `|` 仍正确归属**（验证不靠 `|` 切分）。
- **本批界定（决策P3.7-13，高危回归守门）**：`log.md` 已有**同日旧 audit 段**含 P 页留痕、本次 Agent 漏写 P → wrapper 只看写前快照后的新增 suffix、**不从旧段误读 P**，P 所在组不刷新；suffix **非 append-only**（截断/改史）或含 **>1 个新 audit 段** → **任何组都不刷新**、整批留候选。
- **slugs 精确匹配（决策P3.7-15，高危回归守门）**：一页 P 同属源 a、b 两组（`_format_targets` 给 P 的 `drifted_slugs=["a","b"]`），Agent 留痕只写 `["a"]` → P 视作未留痕、**a 与 b 两组均不刷新**（至少缺失的 b 组不刷新）；照抄 `["a","b"]` → 两组各按其余成员是否齐全独立判完成；slugs 顺序不同但集合相同（如 `["b","a"]`）→ 排序后相等、算有效。
- **targets 契约（决策P3.7-11）**：`_format_targets` 每行含 `page | reason | drifted_slugs | raw_paths`；**`source-drift` 目标页**的 `raw_paths` 取自其**自身 `raw_digest`**（非 `sources:`）；`cites-drifted-source` 页的 `raw_paths` 取自所引漂移源页的 `raw_digest`。
- **raw 路径准入（决策P3.7-12）**：`raw_digest` 为绝对路径 / `../` 越界 / 不以 `raw/` 起 / realpath 逃出 `<kb>/raw/` → audit **当无信号跳过**（无候选、不崩、不抢 check 缺源口径）；正常 `raw/…` 路径照常比对。
- `--dry-run` 纯读零 LLM：不触 runner、必 `EXIT_OK`、不写任何文件（**不 stamp、不刷新指纹**）。
- `audit_result_dict` JSON 契约字段稳定（仿 `heal_result_dict`）。

## 10. 风险

- **指纹漂移≠语义过期**：raw 改一个错别字也会触发候选 → Layer-2 LLM 复核会判"仍准"、wrapper 刷新指纹，成本是一次 LLM 调用。缓解：`--limit` 限漂移源组数（决策P3.7-8）；nightly 而非每次 ingest 跑。可选增"仅当 raw body 变化超 X%"的二阶过滤（但这又滑向语义，倾向不做，让 LLM 兜）。
- **新约定的采纳缺口**：旧库、手写 source 页没 `raw_digest` → 永不进候选（决策P3.7-5）。这是**安全**的退化（少报，不误报），但意味着 P3.7 的价值随 ingest 重写逐步累积，非一上来全覆盖。文档需讲清。
- **provenance 写入主体已定（决策P3.7-3a / Option A）**：实现时 wrapper 在门禁后直写 source 页 `raw_digest`——这让 gate 之后宿主再碰一次 *内容* 页（越"Agent 写 wiki"默契一小步，正当性见 §8）。落地仍需：`conventions.md` + `examples/SCHEMA.md` 把 `raw_digest` 标注为"托管字段、勿手改"；落地点小心**只 stamp 本次那一张**（误刷别页会抹平别处真实漂移，是致命 bug，必须有回归测试守，见 §9）。它比纯加法 CLI 重（含一项越线写 + 一项数据模型新增），是它当初排在 §1"需先拍板"的原因——拍板取 A 后即可进实现排期。

## 11. 后续工作（P3.7 之后，同框架兄弟）

### 11.1 矛盾复检（P3.7.1 — 高价值兄弟，与本阶段共命令共拱心石）

矛盾复检是反向评审认定的**最高价值**项，但 Layer-1 触发信号最难做到"低噪 + 零 LLM"。本节把它展开成一个**可落地的兄弟小设计**：复用 `audit` 同一命令、同一 `raw_digest` provenance、同一门禁，**只在 Layer-1 加一个零成本判别、在 Layer-2 加一条 prompt 分支**。

#### 11.1.0 先承认边界（决定了整套设计的形状）

必须诚实划清：**有一类矛盾，任何零-LLM 结构信号都抓不到——首次 ingest 时就并存、且当时 LLM 没标出来的"静态矛盾"。** 它们本质是语义的，没有任何结构痕迹。

- swarmvault 试图抓静态矛盾：靠一张 claim 图 + polarity + token Jaccard≥0.3 全库两两比。代价是**高噪**（英文正则极性、上限 5000 比较/100 结果）+ **一个 guanlan 红线禁止的 claim 数据模型**。
- guanlan 的**有意取舍**：**放弃**用确定性手段抓静态矛盾，把它留在两道已有防线——(a) ingest 时 LLM 就地标 `## ⚠️ 矛盾与存疑`（`ingest.py:23` 现成行为），(b) 人工/`query` 复核。`audit` 的矛盾复检**只负责变更驱动的那一类**：源变了 / 加了新源之后，新老论断之间**新产生**的冲突。

> 这正是反向评审推崇的姿态：**显式声明约束的代价**，而非假装全覆盖。`audit` 的矛盾复检是"变更触发的增量调和"，不是"全库矛盾扫描"。文档与 `--help` 都要这样讲清，避免给用户"跑完就没矛盾了"的错觉。

#### 11.1.1 三个候选 Layer-1 信号（逐一做噪/召回对抗分析）

- **信号 A —「多源页无矛盾标记」（否决）**：页引 ≥2 源且无 `## ⚠️ 矛盾与存疑` → 疑似"未审视的一致"。**否决**：召回虚高、噪声爆炸——绝大多数多源页本就无矛盾，等于把半个 wiki 报成候选。这就是第一轮我自己批过的"every multi-source page"反模式。

- **信号 C —「多源引用页命中漂移」（采纳，主信号，零新成本）**：复用 §3 第二步已经在产的 `cites-drifted-source` 候选——**当一张候选页本身 `sources:` ≥2 个 slug**时，不仅它论断可能过期，它**跨源综合的结论**也可能因某源被替换而与其余源**新生冲突**。
  - 召回：抓住"源变更引入的矛盾"这一变更驱动核心类。
  - 噪声：**与 source-drift 同档（已低）**——不新增任何候选页，只在既有 `cites-drifted-source` 候选上多读一个"是不是多源"的布尔，把多源的那些**路由到更狠的 prompt**。
  - 成本：**零新 bookkeeping**。`sources` slug 计数本就在 frontmatter，Layer-1 加一个 `multi_source: bool` 即可。这是它当主信号的决定性理由——它**搭 P3.7 拱心石的车，不再动数据模型**。
  - 局限：只覆盖"源**变化**引入"的矛盾，不覆盖首次并存的静态矛盾（见 §11.1.0，那本就不归它）。

- **信号 E —「既有矛盾标记 + 源漂移」（采纳，免费搭车）**：已带 `## ⚠️ 矛盾与存疑` 的页，其引用的源发生漂移 → 旧的存疑结论可能**已被现源解决或加剧**，值得复核"这条存疑是否还成立"。
  - 噪声：极低（**只在已标存疑且源又变的页**触发，集合很小）。
  - 成本：零——`audit_candidates()` 顺手 grep 该 ATX 标题即可。
  - 价值：补 source-drift 的盲区——存疑页的"维护"，让矛盾标记不会标完就烂掉。

- **信号 B —「新源未调和」（推迟，需新 provenance）**：页 `sources:` 相对上次 audit **新增了一个 slug**，但正文**未被该源 ingest 触及** → "加了料没重新综合"。
  - 噪声：低（结构性"未调和"，非"看起来像否定"）。
  - 成本：**需新 provenance**——ingest 要可靠地记"本次触了哪些页"。现状不够：`INGEST_PROMPT` 只保证 `log.md` 追一条 `## [日期] ingest | <标题>` 标题、"触及哪些页"只在**给用户的回执**里（自然语言、不落盘、不可靠解析）。要可靠就得结构化落盘，又一项约定改动。
  - 局限：guanlan **正常** ingest 流里 Agent 本就会更新实体页，"加源不动页"主要出现在**手改/半截 ingest**等非常规路径——召回不如表面高。
  - 结论：**推迟**。等真有非常规流的需求、或 touched-pages provenance 因别的原因落地了，再以同框架接入。当前 C+E 已无新成本地覆盖变更驱动主类。

#### 11.1.2 落地增量（在 P3.7 之上几乎全是加法）

- **Layer-1**：`AuditCandidate` 增两个布尔 `multi_source`（页 `sources:` ≥2）与 `has_conflict_mark`（正文含 `## ⚠️ 矛盾与存疑`）——都零成本读 frontmatter / grep 标题。**`reason` 仍是 §3 的 `{"source-drift", "cites-drifted-source"}`、不新增取值**；矛盾路由靠这两个布尔，不靠扩 reason（避免 reason 维度爆炸）。同一份 `audit_candidates()` 归口产出，**不分叉**。
- **Layer-2**：`AUDIT_PROMPT` 据布尔分支——`multi_source` 或 `has_conflict_mark` 为真的页追加一句："此外，对照该页 `sources:` 指向的源的现版本，判断被替换的源是否与其余源或页中既有综合**新生矛盾**；若是，按 conventions 就地标/更新 `## ⚠️ 矛盾与存疑`。" 单源页走原 §5 过期论断分支。**仍不教 LLM 新把戏**——只是把 ingest 已会的标注动作定向到"跨源对照"。
- **CLI**：无新命令、无新 flag、无新退出码。`audit --dry-run` 的候选行附 `reason` + 两布尔即可区分疑点类别。

#### 11.1.3 决策与测试增量

- **决策P3.7.1-1**：矛盾复检**只覆盖变更驱动类**，显式放弃用确定性手段抓静态矛盾（留给 ingest 内联标注 + 人工）。`--help`/文档必须讲清边界。
- **决策P3.7.1-2**：主信号取 **C（多源页 + 源漂移）**，因其**零新 bookkeeping、噪声与 source-drift 同档**；E 免费搭车；**B 推迟**（需 touched-pages provenance）。
- **决策P3.7.1-3**：**仍不引入 claim/polarity/Jaccard**（决策P3.7-6 的延续）。跨源矛盾判断整体下放 Layer-2 LLM；Layer-1 只贡献"哪些漂移页是多源 / 带存疑标记"的零-LLM 路由。
- 测试增量（仿 §9）：多源引用页命中漂移 → `reason="cites-drifted-source"`、`multi_source=True` / 单源页命中 → `multi_source=False` / 带 `## ⚠️ 矛盾与存疑` 且命中漂移 → `has_conflict_mark=True` / **无漂移的多源页 → 不进候选**（验证 C 不退化成信号 A 的全多源页洪水）/ Layer-2 prompt 按布尔正确分支（mock runner 断言多源候选的 prompt 含跨源对照句、单源候选不含）。
