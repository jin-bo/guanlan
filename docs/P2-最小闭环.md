# P2 — 最小闭环（ingest + query + 确定性门禁）实现设计

> 本文是 [`DESIGN.md`](DESIGN.md) §4.4 / §4.6 / §4.7 / §5.1 / §7-P2 的**实现级细化**，是 P2 编码的落地依据。
> 高层理念与原则以 `DESIGN.md` 为准；本文只在 P2 范围内把"怎么做"钉死到模块、数据结构、契约与测试。
> 文中标注的 **\[决策]** 是对 `DESIGN.md` 的具体化或受控偏离，集中记录在 §10。

---

## 1. 目标与范围

**P2 = "能在 CLI 里稳定摄入 `.md` 并查询的文件型知识库" 的最小闭环。**

纳入 P2：

- `guanlan ingest raw/<file>.md` —— wrapper 调用 Agentao + `guanlan-wiki` skill 摄入一篇 `.md`，收尾**强制**确定性门禁。
- `guanlan query "…"` —— 默认只读问答；`--backfill` 把好答案回填 `wiki/syntheses/`，走与 ingest 同一门禁。
- `guanlan check` —— 独立的确定性校验（frontmatter + wikilink 断链 + `sources` 解析），零 LLM。
- 确定性门禁 = `check`（frontmatter + 断链 + `sources` 解析）+ `raw/` 调用前后快照比对。

**不**纳入 P2（推后，见 §11）：多格式摄入（P5，MVP 只吃 `.md`）、health / 结构 lint / graph（P3）、语义 lint（P3 之后）、Web UI（P4）、`--commit` 原子性、并发锁。

## 2. 数据流总览

```
guanlan ingest raw/x.md
  ├─ 0. 前置校验：require_kb_root(root, writable=True)（见 §6）；目标在 raw/ 下且为 .md  → 否则 EXIT_USAGE(1)
  ├─ 1. before = snapshot_raw(root)                              # 确定性，零 LLM
  ├─ 2. r = run_agent_task(ingest_prompt, wd=root,              # 唯一 LLM 步骤，经 agentao run + skill
  │        permission_mode="workspace-write")
  │        └─ not r.ok → 仍比对 raw 快照：变了即 EXIT_RAW_MUTATED(4)，否则 EXIT_AGENT_ERROR(5)（不跑 check）
  ├─ 3. enforce(root, before)：                                 # 内部再 snapshot_raw 取 after
  │        a. raw 快照 diff 非空        → EXIT_RAW_MUTATED(4)
  │        b. guanlan check（wiki/）失败 → EXIT_CHECK_FAILED(3)
  └─ 4. 全过 → 打印 r.final_text + 触及页面 → EXIT_OK(0)
              失败 → 报告违规项，wiki/ 改动**原样留在磁盘**供人工修正

guanlan query "…"           默认只读：run_agent_task(query_prompt, permission_mode="read-only") → 打印 r.final_text
                            非写路径：不取 raw/ 快照、不跑 check（read-only 姿态已拦写入）
guanlan query --backfill "…" workspace-write + 同 ingest 的完整门禁（允许写 wiki/syntheses/）

guanlan check               仅跑 guanlan check（wiki/）（无"调用前"基线，故不含 raw/ 快照比对） → 0 / EXIT_CHECK_FAILED(3)
```

## 4. Agentao 集成（`guanlan/runtime.py`）

> 本节据 `agentao` 的 [`docs/EMBED_FOR_AGENTS.md`](../../agentao/docs/EMBED_FOR_AGENTS.md) 审核并修订。

**\[决策5 / 关键修订] P2 用 `agentao run` 子进程驱动，不在进程内嵌入 `Agentao(...)`。** 理由：

- **DESIGN 边界**：`DESIGN.md` §5.1/§5.2 把 in-process 库嵌入（`Agentao(...)`）明确划在 **P4**；P2 是 CLI 级集成（文件约定 + skill discovery + 薄 wrapper）。原草案选 in-process 嵌入，越过了这条边界。
- **EMBED 指南导向**：EMBED_FOR_AGENTS.md §0 把"跑一段 prompt、拿退出码"这一需求直接导向 `agentao run --prompt`，并叮嘱此类场景**不必走嵌入指南**。
- **零手搓**：`agentao run` 原生提供 wrapper 所需的一切（激活、姿态、非交互、结构化输出、退出码），无需我们自建构造/日志治理（旧草案的踩坑见 §4.3）。

### 4.1 调用形态

wrapper 以子进程调用，工作目录 = 知识库根：

```
agentao run --prompt "<task>" --format json \
            --skill guanlan-wiki \
            --permission-mode <read-only|workspace-write> \
            --interaction-policy reject \
            [--model M] [--max-iterations N]
```

stdout 是 `RunResult` JSON 信封（实测 `agentao==0.4.8`；消费侧按 `extra=ignore` 向前兼容）：

```jsonc
{ "status": "ok"|"error", "final_text": "<agent 最终文本>",
  "session_id": "...", "cwd": "...", "model": "...",
  "error": { "type": "permission_required|permission_denied|interaction_required|max_iterations|runtime_error|invalid_spec|interrupted",
             "message": "..." } }
```

> 字段名是 `error.type`（实测 `agentao/cli/run_models.py::RunErrorEnvelope`），**不是** `error.kind`；解析时取 `result["error"]["type"]`。

- `--skill guanlan-wiki` —— **用文档化的 CLI flag 显式激活** skill（取代旧 \[决策3] 对 `SkillManager.activate_skill` 内部 API 的耦合）。实测：skill 虽被发现但默认**不自动激活**（构造后 `get_active_skills()` 为空），故显式激活是必要的。
- `--interaction-policy reject` —— 非交互；agent 若想发问即判失败，wrapper 不会挂起等输入。
- `--permission-mode` —— 设定权限姿态（§8）：query 默认 `read-only`，ingest / `--backfill` 用 `workspace-write`（默认值）。
- `--format json` —— 解析 `final_text` 作为答案/摘要，`status`/`error.type` 决定成败。
- 进程创建、`AGENTAO.md` 自动加载、skill 发现（`<wd>/skills` 与 `~/.agentao/skills`）、`.env`/`~/.agentao` 的密钥与 provider —— 全部由 `agentao run` 按 CLI parity 处理；**wrapper 不读密钥、不构造 `Agentao`、不污染知识库的日志**。
- **依赖面**：仅依赖 `agentao run` 的 CLI flag、`RunResult` 信封与退出码表（其文档化的稳定契约）。EMBED §9 建议声明兼容区间——`pyproject` 维持 `agentao>=0.4.8`，升级到 0.5.x 前需复核该 CLI 契约。

### 4.2 单一集成函数

```python
# guanlan/runtime.py（签名草案）
@dataclass
class AgentRunResult:
    ok: bool
    final_text: str
    error_type: str | None = None     # 取自信封 error.type 或退出码
    raw: dict | None = None           # 原始 RunResult，便于排错

AgentRunner = Callable[..., AgentRunResult]   # 测试注入点

def run_agent_task(
    prompt: str,
    *,
    working_directory: Path,
    permission_mode: str = "workspace-write",
    skills: tuple[str, ...] = ("guanlan-wiki",),
    model: str | None = None,
    max_iterations: int = 100,
    runner: AgentRunner | None = None,   # None → 默认 subprocess 调 `agentao run`
) -> AgentRunResult: ...
```

默认 runner（`runner is None`）：拼上述命令行 → `subprocess.run([...], cwd=working_directory, capture_output=True, text=True)` → 解析 stdout JSON 为 `AgentRunResult`。子进程非零退出或 `status=="error"` → `ok=False`，`error_type` 取自信封 `error.type` 或退出码。

### 4.3 旧 in-process 草案的踩坑（已据 EMBED 指南规避）

原草案 `Agentao(working_directory=…, model=None, skill_manager=sm).chat()` 有四处问题，正好被 `agentao run` 规避——记录以免回退：

1. **构造即报错**：EMBED §1——纯注入 `Agentao(...)` 必须给 `llm_client` 或 `api_key+base_url+model`，否则 `ValueError`；`model=None` **不会**自动读环境密钥（只有 `build_from_environment` 才读 `.env`/`~/.agentao`）。原 §4.2 "None → 从环境解析" 对纯注入路径是**错的**。
2. **skill 不自动激活**：实测发现即不激活，需显式指定（`agentao run --skill`）。
3. **日志污染**：EMBED §8——`Agentao(...)` 会往 `<wd>/agentao.log` 写日志并把 `agentao` logger 抬到 DEBUG；而 `<wd>` 是用户知识库，会被污染（且单传 `log_file=None` 仍抬 DEBUG，必须传 `logger=`）。
4. **耦合内部 API**：从 host 调 `SkillManager.activate_skill` 不在 EMBED 的稳定 host 契约内。

### 4.4 退出码映射与可测试性

- `agentao run` 的退出码表（0 ok / 1 runtime / 2 usage / 3 permission·interaction / 4 max-iterations / 130 interrupted）由 runtime 归一为 `AgentRunResult.ok`。
- **agent 失败（`not r.ok`）仍要兜底 `raw/` 完整性**：agent 可能先改了 `raw/` 再失败，硬约束不能漏判。故失败路径**只做 `raw/` 快照 diff**（不跑 check —— wiki/ 半成品无意义）：`raw/` 变了 → `EXIT_RAW_MUTATED(4)`，否则 → `EXIT_AGENT_ERROR(5)`（§9）。
- `runner` 注入仍是测试关键：fake runner 模拟"写合规/不合规 `wiki/` 文件 + 返回摘要"，**不起子进程、不打真实 LLM**（§12）。

## 5. 确定性门禁

### 5.1 `raw/` 快照（`guanlan/gate.py`）

**\[决策4] 用内容 SHA256，按相对路径建键** —— 不用 mtime（易漂移/可伪造）。

```python
def snapshot_raw(root: Path) -> dict[str, str]:
    """递归遍历 root/raw/ 下所有普通文件，返回 {相对posix路径: sha256_hex}。"""

def diff_raw(before: dict[str, str], after: dict[str, str]) -> list[RawChange]:
    """新增/删除/修改/重命名都会体现为 added/removed/modified 条目；任意非空即违规。"""
```

- 覆盖增、删、改、重命名（重命名 = 一删一增）；空 `raw/`（仅 `.gitkeep`）也纳入快照。
- 调用时机：ingest / `--backfill` 在 `run_agent_task` **前**取 before，**后**取 after。
- 任何非空 diff → 硬完整性违规（`EXIT_RAW_MUTATED`）。MVP **不保留内容副本，故不自动还原**，只**报告**被改动的 `raw/` 路径，由用户处理（单用户 MVP 可接受）；`DESIGN.md` §4.7 的 deny 规则为可选纵深防御。

### 5.2 校验器（`guanlan/check.py`）

**扫描范围**：`wiki/` 下递归所有 `*.md`，**排除 config**：`wiki/index.md`、`wiki/log.md`、`wiki/overview.md`（`SCHEMA.md` 在根，不在 `wiki/` 下，天然不扫）。`raw/` 不参与。

**frontmatter 校验**（每张被扫页面）：

- 文件以 `---\n … \n---` 起始；用 `yaml.safe_load` 解析该块 **\[决策2]**。
- 必备键与类型：
  - `title`：非空 `str`
  - `type`：∈ `{source, entity, concept, synthesis}`
  - `tags`：`list`
  - `sources`：`list[str]`
  - `last_updated`：匹配 `^\d{4}-\d{2}-\d{2}$` 的 `str`
- 缺键 / 类型不符 / 块缺失或无法解析 → 各记一条 violation。

**wikilink 断链校验**：

- 解析范围 = frontmatter 之后的正文；提取 `[[…]]`，对每个目标先剥 `|别名` 与 `#锚点`，再取 stem。
- **解析集 = `wiki/` 下所有 `*.md` 的 stem（含 index/log/overview，按存在性判定，大小写不敏感）。** 注意：扫描排除（决定哪些页面被 frontmatter 校验）与链接解析集（决定链接目标是否存在）是**两件事**——config 页不被校验，但可作为合法链接目标。
- 指向解析集外 → 断链 violation。

**`sources` 解析校验**：frontmatter `sources` 里每个 slug 必须对应存在的 `wiki/sources/<slug>.md`，否则 violation。

**输出契约**：

```jsonc
// guanlan check --json  （wrapper 内部直接 import 调用，走同一函数）
{ "ok": false, "pages_checked": 12,
  "violations": [
    {"page": "wiki/entities/Foo.md", "kind": "frontmatter.missing_key", "detail": "缺 last_updated"},
    {"page": "wiki/concepts/Bar.md", "kind": "wikilink.broken",         "detail": "[[Baz]] 无对应页面"},
    {"page": "wiki/sources/x.md",    "kind": "sources.unresolved",      "detail": "sources 列出的 'y' 无 wiki/sources/y.md"}
  ] }
```

无 `--json` 时打印人类可读的逐行报告。**退出码**：`ok` → 0，否则 → 3。

`kind` 枚举（稳定，便于测试与未来 lint 复用）：`frontmatter.block_missing` / `frontmatter.unparsable` / `frontmatter.missing_key` / `frontmatter.bad_type` / `wikilink.broken` / `sources.unresolved` / `wiki.missing`（作为写门禁收尾时 `wiki/` 被删/改名的兜底）。

### 5.3 门禁组合（`gate.py`）

```python
def enforce(root: Path, snapshot_before: dict[str, str]) -> GateResult:
    raw_changes = diff_raw(snapshot_before, snapshot_raw(root))   # 先判完整性
    if raw_changes: return GateResult.raw_mutated(raw_changes)
    check = run_check(root / "wiki")                              # 再判内容合规
    if not check.ok: return GateResult.check_failed(check.violations)
    return GateResult.ok()
```

`raw/` 完整性**先于** check（完整性破坏更严重）。两项均零 LLM、确定性、可测。

## 6. CLI 契约（`guanlan/cli.py`）

保持 `init` 不变。把 `ingest`/`query`/`check` 接到真实实现：

```
guanlan ingest <raw-file.md> [--model M]
guanlan query  <question> [--backfill] [--model M]
guanlan check  [--json]
```

- 全局：可选 `-C/--dir <path>`（默认 cwd）设知识库根，便于在库外调用。
- **前置校验收口为单一 helper `require_kb_root(root, *, writable)`**（落在 `guanlan/paths.py`，供 ingest/query/check 复用；不做自动发现/修复，只查文件存在性），失败即 `EXIT_USAGE(1)` 并提示 `guanlan init`：
  - **写入口**（`ingest`、`query --backfill`，`writable=True`）：要求 `raw/`、`wiki/`、`AGENTAO.md`、`SCHEMA.md` 齐全。
  - **只读路径**（`query`、`check`，`writable=False`）：仅要求 `wiki/`（`check` 只扫 `wiki/`；plain query 只读 `wiki/`）。

## 7. ingest 工作流细化（`guanlan/ingest.py`）

```python
def run_ingest(target: str, *, root: Path, model=None, runner=None) -> int:
```

1. **前置校验**：`require_kb_root(root, writable=True)`（§6）；`target` 解析后位于 `<root>/raw/` 下、存在、后缀 `.md`（否则 `EXIT_USAGE(1)`；非 `.md` 提示"多格式摄入属 P5"）。
2. `slug = kebab-case(target.stem)`（仅用于错误提示与日志；真正建页由 agent 按 conventions 做）。
3. `before = snapshot_raw(root)`。
4. `r = run_agent_task(INGEST_PROMPT.format(rel=raw相对路径), working_directory=root, model=model, runner=runner)`。
5. **`not r.ok`**（agent 失败）：仍取 after 快照比对 `raw/`——变了 → `EXIT_RAW_MUTATED(4)`，否则 → `EXIT_AGENT_ERROR(5)`（不跑 check）。
6. **`r.ok`**：`answer = r.final_text`；`result = enforce(root, before)`。`result.ok` → 打印 `answer` + "门禁通过" → `EXIT_OK(0)`；否则按 `result.kind` 打印违规并返回 `EXIT_CHECK_FAILED(3)` / `EXIT_RAW_MUTATED(4)`，并提示 **wiki/ 改动已留在磁盘**待人工修正。

**INGEST_PROMPT（薄；真正步骤在 skill）**：

> 请按 `guanlan-wiki` skill 的 ingest 工作流摄入资料 `raw/{rel}`：读该 `.md` 源与 `wiki/index.md`、`wiki/overview.md` 建上下文；写/更新 source·entity·concept 页（frontmatter 齐全、术语转 `[[wikilink]]`）；更新 `index.md` 与 `overview.md`；发现矛盾就地标 `## ⚠️ 矛盾与存疑`；向 `log.md` 追加一条 `## [<日期>] ingest | <标题>`。遵循 `AGENTAO.md` 硬约束与 conventions 默认。**永不修改 `raw/`。** 完成后用一两句说明触及了哪些页面。

## 8. query 工作流细化（`guanlan/query.py`）

```python
def run_query(question: str, *, root: Path, backfill=False, model=None, runner=None) -> int:
```

- **默认（只读）**：组 `QUERY_PROMPT`（强调只读、不写 `wiki/`），`run_agent_task(..., permission_mode="read-only")` —— 既靠 prompt 也靠**权限姿态**双重兜底 → 打印答案。**不取 `raw/` 快照、不跑 check**：非写路径无需每次扫全量 `raw/`，`read-only` 模式本身已拦下写入（含 shell），快照只保留在两个写入口。
- **`--backfill`**：`permission_mode="workspace-write"`；prompt 追加"把好答案回填 `wiki/syntheses/<slug>.md`（`type: synthesis`）"。**失败语义与 ingest 完全一致**：`not r.ok` → 仍取 after 快照只比对 `raw/`，变了 → `EXIT_RAW_MUTATED(4)`，否则 → `EXIT_AGENT_ERROR(5)`（不跑 check）；`r.ok` → 走 §5.3 **完整门禁**（check + raw 快照）。
- **`raw/` 快照只在写入口**（ingest、`query --backfill`）：那里 `workspace-write` 允许写 `<wd>` 内任意路径（含 `raw/`），故**权限姿态不保护 `raw/`**，必须靠 §5.1 快照作硬保证（DESIGN §4.7）。默认 query 走 `read-only`，不需快照。
- **CJK 召回在 agent 内**：P2 不做独立检索脚本——agent 读 `index.md`、对中文用 2-gram 粗召回；不中时扫相关目录或请用户补关键词（graceful fallback，见 conventions §CJK）。可选 `recall.py` 助手列为**非阻塞**增强，不进 P2 关键路径。

**QUERY_PROMPT（薄）**：

> 请按 `guanlan-wiki` skill 的 query 工作流回答：{question}。先读 `wiki/index.md` 定位相关页（中文用 2-gram 粗召回；不中时扫相关目录或请我补关键词）；综合出**带 `[[页]]` 引用**的答案；无可靠来源时明说、不编造。**默认只读，不要写 `wiki/`。**

## 9. 退出码与失败处理（`guanlan/errors.py`）

| 码 | 常量 | 含义 | 触发 |
|----|------|------|------|
| 0 | `EXIT_OK` | 成功 | 命令完成且门禁全过 |
| 1 | `EXIT_USAGE` | 用法/IO 错误 | 非知识库根、目标不在 `raw/`、非 `.md`、文件不存在 |
| 3 | `EXIT_CHECK_FAILED` | 内容校验失败 | frontmatter / 断链 / sources 违规 |
| 4 | `EXIT_RAW_MUTATED` | `raw/` 被改动 | 调用前后快照 diff 非空 |
| 5 | `EXIT_AGENT_ERROR` | Agentao 运行时错误 | `agentao run` 子进程非零退出 / `status=="error"` / runtime 解析 stdout 失败 |

> 占位码 `2`（P1 的"未实现"）在 P2 随三个命令落地而退出历史；保留给未来仍未实现的子命令。

**失败时的磁盘状态**：门禁失败**不回滚** `wiki/`（DESIGN §4.7）；改动留在磁盘，报告具体违规项，由用户手动修正或删除。需要一键撤销者自行把 `wiki/` 纳入 git（可选，非 MVP 机制）。`raw/` 违规因未存内容副本而无法自动还原，只报告路径。

## 10. 对 DESIGN.md 的偏离与决策记录

- **\[决策1] check 实现落在包内，而非 skill 的 `scripts/check.py`。** DESIGN §4.6 把 `check.py` 列在 skill `scripts/` 下；但 wrapper 门禁也要调它，而 skill 目录在开发期（repo `skills/`）与安装期（`~/.agentao/skills/`）路径不同，从 wrapper 解析该路径会引入额外机制与"双实现"分叉风险。**改为**：唯一实现在 `guanlan/check.py`（可 import、可测），暴露为 `guanlan check` 子命令与 `python -m guanlan.check`；wrapper 直接 import 同一函数，skill 指示 agent shell 调 `guanlan check`（装包后在 PATH 上）。单一实现、单一可信源。
- **\[决策2] 新增 `pyyaml>=6` 运行时依赖**，用于稳健解析 frontmatter 的列表/引号/多行；不手写 YAML 子集解析器（易错、需大量边界测试）。
- **\[决策3] skill 由 wrapper 显式激活，经文档化的 `agentao run --skill guanlan-wiki` flag**（不再耦合 `SkillManager.activate_skill` 内部 API）。实测 skill 被发现但默认不自动激活，故显式激活必要。
- **\[决策4] `raw/` 快照用内容 SHA256 + 相对路径**（DESIGN §4.7 写"文件名 + size + mtime，必要时 SHA256"）：直接用 SHA256 在个人级数据量上成本可忽略，且不受 mtime 漂移/伪造影响，判定更确定。
- **\[决策5] P2 经 `agentao run` 子进程驱动 LLM，不在进程内嵌入 `Agentao(...)`**（in-process 嵌入按 DESIGN §5.2 留待 P4）。据 `agentao` 的 `docs/EMBED_FOR_AGENTS.md` §0 审核：本场景属"跑 prompt、拿退出码"，应走 `agentao run`；它原生提供 `--skill`/`--permission-mode`/`--interaction-policy reject`/`--format json` 与稳定退出码表，规避了 in-process 草案的创建报错、skill 不激活、`<wd>/agentao.log` 污染、内部 API 耦合四处问题（§4.3）。
- **\[决策6] skill 随包携带并在安装态自动装入全局，而非要求用户手工安装。** Agentao 仅在 `~/.agentao/skills/`、`<wd>/.agentao/skills`、`<wd>/skills` 三处发现 skill（`--skill` 取名不取路径）。开发期 `<wd>=仓库根` 命中 `skills/`，免安装；但安装态下用户库三处都没有 skill，`agentao run --skill guanlan-wiki` 会立即失败。故 wheel `force-include` 把 `skills/guanlan-wiki` 打到 `guanlan/_skill/`，默认 runner 在调 `agentao run` 前经 `skill.ensure_skill_available` **幂等拷贝到全局**（best-effort、only-if-absent，与 Agentao 自带 skill bootstrap 一致），另暴露 `guanlan install-skill [--force]`。注入 runner 的测试不触发此拷贝（逻辑落在默认 subprocess runner，不在 `run_agent_task`）。

## 11. 不纳入 P2（推后）

- **多格式摄入**（docx/pdf/web clip + 转换 adapter）→ P5；MVP ingest 只吃 `.md`。
- **health / 结构 lint / `build_graph.py` + `graph.html`** → P3。
- **语义 lint**（矛盾复检、过期论断、资料缺口，需 LLM）→ P3 之后。
- **`--commit` 原子性、并发锁** → 非 P2；P2 假设单用户、单进程。
- **独立 CJK 检索脚本 / aliases / 分词** → 仅在漏召回有实证后再评估（DESIGN §8 / `docs/backlog/notes/cjk-retrieval-enhancements.md`）。
