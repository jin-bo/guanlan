> **\[决策P4.7-N]** 是对既有契约的具体化或受控偏离,集中记录在 §7。

## 1. 目标与范围

**P4.7 = 浏览器界面右上角加一个语言切换按钮,把**界面 chrome**整体在中文/English 间翻转,选择持久化、刷新不丢;
内容真相(wiki 正文、Agent 对话、源、报告/自省正文)一律保持其本来语言,不被翻译。**

### 核心边界:翻 chrome,不翻内容(决策P4.7-1)

界面上的字分两类,本相位**只动第一类**:

| 类别 | 例子 | 归属 | P4.7 |
|------|------|------|------|
| **界面 chrome**(框架文案) | 顶栏按钮「新会话/历史/投喂/入库/补全/校验/健康/巡检/图谱/暂存区」、tooltip、composer 占位符、tagline、弹层标题(「报告」「暂存区」「历史会话」)、姿态徽标 tooltip、`/help` 命令列表、前端生成的提示/确认/错误文案(「无匹配页面」「加载页面失败…」「会话写作业进行中…」「同名已存在」) | `static/`(前端硬编码) | ✅ **翻译** |
| **内容真相**(数据/诊断) | Wiki 页正文(`render_page` 渲染的 markdown)、Agent 流式回答、`raw/` 源、`check/health/lint` 报告正文、`/status /context /skills /tools` 自省正文、文件名/frontmatter/slug | 后端(Python CLI / Agentao)按其本来语言产出 | ❌ **不译**(原样透传) |

**为什么这条线**:① 与原则「Markdown 是唯一真相」同构——内容由其作者/Agent 以某种语言写定,界面是**取景框**,
不该改写取景框里的水。② 翻内容要么把所有 Python CLI 文案(CLAUDE.md 定:用户可见输出为中文)再复制一份英文、
与单一真相源**漂移**,要么上机器翻译——**后者违反「零-LLM 脚本不得自带 LLM」**。③ 用户诉求是「**界面**双语切换」,
正落在 chrome。故 chrome 在 `static/` 内闭环切换,后端一字不改、不需要传 `lang` 参数给任何端点。

> **小注**:`/status` 等自省正文确含少量中文标签(「模型/姿态/上下文用量」),`check/health/lint` 报告亦然。
> 它们是**后端产出的诊断数据**,本相位**不翻**(后端不动);若日后要让它们也双语,属另一相位(需后端按 `Accept-Language`/
> 显式 `lang` 重排文案,工作量与漂移风险都大,见 §1「不纳入」)。本相位把线划在「前端能独立闭环」处。

### 纳入 P4.7

- **语言词表 `static/i18n.js`**(§2):`I18N = { zh, en }` 两套**键值对照表**(稳定 key → 文案),加 `t(key, ...args)`
  取词函数(支持 `{0}` 位置插值,给「加载页面失败:{0}」这类动态串用)。**单文件、可评审、`<script>` 先于 `app.js` 载**。
- **声明式静态标注 `data-i18n*`**(§3):`index.html` 里每条 chrome 文案标 `data-i18n="key"`(textContent)/
  `data-i18n-title` / `data-i18n-placeholder` / `data-i18n-aria`;一个 `applyI18n(root)` 遍历器据当前语言回填,
  **不在 JS 里逐个 `querySelector`**。
- **右上角切换按钮**(§4):顶栏 `.actions` 末位加 `#lang-btn`(地球图标 + 目标语言码),点击翻转 `zh ⇄ en`。
- **持久化 + 初始化**(§5):`localStorage["guanlan.lang"]` 记忆**最后一次语言设置**;载入即沿用上次选择
  (首访/无记忆时写死默认 `zh`,**不读 `navigator.language` 探测**)。同步更新 `document.documentElement.lang`。
- **动态文案过 `t()` + 切换即重渲染开着的动态面**(§6):`app.js` 里运行时生成的 chrome 串改用 `t()`;切换时
  `applyI18n(document)` 刷静态节点 + 重渲染「当前 wiki 视图 / 打开着的 `#overlay` / composer 附件徽章」等少数动态面
  (从各自状态重建,**不重载页面、不丢对话上下文**)。

### 不纳入 P4.7(按边界推后)

- **后端产出的内容/诊断双语**(`check/health/lint` 报告正文、`/status` 等自省正文、Agent 回答):**不译**——见决策P4.7-1。
  让报告/自省也随 `lang` 切换属另一相位(后端改造,非纯前端)。
- **Wiki 内容机翻 / 多语种页**:违反「Markdown 唯一真相」与零-LLM,不做。
- **第三、四种语言 / 完整 i18n 框架(ICU、复数规则、构建期抽取)**:本相位只要 zh/en 两套手写词表,
  **不引 npm/构建/CDN/任何运行时库**(决策P4-3 不破)。位置插值够用,不做复数/性别等形态规则。
- **RTL/字体子集化**:zh/en 均 LTR,无需 RTL;字体沿用现有 `app.css`。

## 3. 声明式静态标注 `data-i18n*`(`index.html`)

`index.html` 里每条 chrome 文案改成「标 key、由遍历器回填」,**默认内容仍写中文**(故 zh 用户零闪烁、即便 JS 失败也可读):

```html
<!-- 顶栏按钮:textContent 走 data-i18n,tooltip 走 data-i18n-title -->
<button id="chat-new" class="btn" data-i18n-title="tip.newChat" title="开启新会话(清空上下文)">
  <svg class="ico"><use href="#i-new"/></svg><span class="btn-zh" data-i18n="btn.newChat">新会话</span></button>

<!-- tagline / 占位符 / 搜索框 -->
<span class="tagline" data-i18n="tagline">观水有术,必观其澜</span>
<textarea id="chat-input" data-i18n-placeholder="composer.placeholder" placeholder="向 agent 提问…"></textarea>
<input id="wiki-search" type="search" data-i18n-placeholder="wiki.search" placeholder="Wiki Search…">
```

一个遍历器在「初始化」与「每次切换」时统一回填属性:

```js
function applyI18n(root = document) {
  root.querySelectorAll("[data-i18n]").forEach(el => { el.textContent = t(el.dataset.i18n); });
  root.querySelectorAll("[data-i18n-title]").forEach(el => { el.title = t(el.dataset.i18nTitle); });
  root.querySelectorAll("[data-i18n-placeholder]").forEach(el => { el.placeholder = t(el.dataset.i18nPlaceholder); });
  root.querySelectorAll("[data-i18n-aria]").forEach(el => { el.setAttribute("aria-label", t(el.dataset.i18nAria)); });
}
```

- **支持四种载体**:`data-i18n`(textContent)、`-title`(tooltip)、`-placeholder`、`-aria`(`aria-label`,无障碍)。一节点可同时带多个(如按钮既有可见文字又有 tooltip)。
- **保留中文默认值**:静态 HTML 的可见文字仍写中文(decision:zh 是项目默认语言、`<html lang="zh">`)。zh 用户**无需任何 JS pass**即得正确界面(零 FOUC);en 用户载入后一次 `applyI18n` 翻成英文(§5 FOUC 处置)。

## 5. 持久化与初始化(localStorage,FOUC 处置)

```js
function initLang() {
  const saved = localStorage.getItem("guanlan.lang");
  const lang = (saved === "zh" || saved === "en") ? saved : "zh"; // 用上次设置,无则默认 zh
  applyLang(lang);
}
```

- **记忆「最后一次语言设置」**:`localStorage["guanlan.lang"]` ∈ `{zh, en}`;刷新/重开标签页**一律沿用上次选择**。
- **默认 `zh`(不做浏览器探测)**:首次访问、或 `localStorage` 被清后,**写死默认 `zh`**——与 `<html lang="zh">`、项目工作语言、静态 HTML 默认文案一致,且行为可预测。**刻意不读 `navigator.language` 兜底**:语言以「用户上次显式选择」为准,首访统一 zh 最不意外(若日后想「非中文浏览器首访默认 en」是一行决策,本相位不做,决策P4.7-5)。
- **FOUC(决策P4.7-5)**:静态 HTML 默认写中文,故 **zh 用户零闪烁**(`initLang()` 对 zh 是 no-op 级);**en 用户**在 `app.js` 启动首帧会有一瞬中文→英文切换。`app.js` 在 body 末尾,`initLang()` 应作为其**第一个动作**(早于首批 `fetch`)以把这一瞬压到最短。若日后要彻底消除 en 用户的瞬切,可在 `<head>` 内联一小段「读 localStorage → 设 `documentElement.dataset.lang`」并配 CSS,**本相位不做**(收益小、复杂度高)。

## 6. 动态文案与切换重渲染(`app.js`)

`app.js` 运行时生成的 chrome 串(弹层正文、暂存区两段、附件徽章、错误/确认/提示、`/help` 列表、姿态徽标 tooltip)改用 `t(key)` / `t(key, arg)`:

```js
// 之前: view.innerHTML = `<div class="empty">加载页面失败:${escapeHtml(pagesError)}…</div>`;
// 之后:
view.innerHTML = `<div class="empty">${escapeHtml(t("wiki.loadFail", pagesError))}</div>`;
// 空态:三元判分支、key 各自字面量(勿写成 t(cond ? "a" : "b") —— 会绕过字面量扫描,决策P4.7-8):
view.innerHTML = `<div class="empty">${allPages.length ? t("wiki.emptyMatch") : t("wiki.emptyAll")}</div>`;
```

> **铁律:`t()` 首参恒为字符串字面量**(决策P4.7-8)。**绝不**把判断塞进首参(`t(cond ? "a" : "b")`、
> `t("pre." + x)`、`t(varKey)`)——那会让 §8 的字面量扫描漏掉这些 key,拼错只在运行时露 key。需要条件取词就
> **在外层判分支、每分支各自 `t("字面量")`**。极少数确需动态 key 的(如按 `type` 拼 key),走**集中常量映射**
> + 显式测试白名单(§8),不散落裸 `t(var)`。

**切换时重渲染策略(决策P4.7-6):刷静态 + 用「已取回的数据」纯渲染,绝不重载页面、绝不重发网络请求、不丢对话上下文。**

> **关键纪律:切换走「纯渲染」路径,不走「取数」路径。** 现有多数渲染入口是 `async opener`——`renderPage()`
> 会 `fetch /api/page`,各 overlay 打开函数会打 `/api/report/*`、`/api/raw`、`/api/conversations`、`/api/workspace`。
> 切换语言**绝不能重跑这些 opener**,否则就重发请求、可能改内容/引入失败态,直接破「只换 chrome 皮」验收口径。
> 故凡切换要刷的动态面,都要把它拆成**「取数(async,只在用户操作时跑)」+「纯渲染(同步,吃已缓存数据,可被切换重跑)」**两段;
> 切换只调纯渲染段。

| 动态面 | 切换时怎么刷(**只纯渲染,不取数**) |
|--------|-------------|
| 顶栏/composer/搜索框等**静态 chrome** | `applyI18n(document)` 一遍即可(声明式) |
| **Wiki 列表态**(`{kind:"index"}`) | 纯本地数据(`allPages`/`pagesError` 已缓存)→ 直接 `renderIndex(currentView())`,只重算空态/错误/分组 chrome,**不取数** |
| **Wiki 单页态**(`{kind:"page"}`) | **不重拉 `/api/page`**:页正文 HTML 已在 DOM 里、是内容(不变)。切换只 `applyI18n` 刷该页**外围 chrome**(导航条/`page-meta` 里的框架字);正文原样保留。即 `renderPage` 拆出一个**纯 chrome 刷新**分支(不碰已渲染正文),切换只调它,**不调取数分支**(决策P4.7-6) |
| **打开着的 `#overlay`**(报告/暂存区/历史/raw 选择) | **不重跑 async opener**:opener 拆成「取数 → 存入 `overlayState.data` → 纯渲染 `renderOverlay(overlayState)`」;切换只在 overlay 可见时重跑**纯渲染**(吃 `overlayState.data`,不打 `/api/report/*`·`/api/raw`·`/api/conversations`·`/api/workspace`)。标题等走 `data-i18n` 的部分由 `applyI18n` 直接刷 |
| **composer 附件徽章**、提示行 | 从内存 `attachments` 状态纯重建徽章(文件名是内容、图标/「移除」aria 是 chrome) |
| **聊天气泡正文**(Agent/用户消息) | **不动**——是内容(决策P4.7-1);只有气泡内的 chrome(如「停止」按钮 aria)随 `applyI18n` 刷 |

`rerenderDynamic()` 即把上面后四行的**纯渲染段**串起来(各吃自己已缓存的数据)。**关键不变量:重渲染只换 chrome 皮——不重发任何网络请求、不改对话历史、不动 wiki 正文 / overlay 里已取回的内容数据。** 这要求 C2 落地时先做一次轻量拆分:给 `renderPage` 加纯 chrome 刷新分支、给各 overlay opener 抽出「数据 + 纯渲染函数」(取数与渲染解耦),切换只触达纯渲染侧(决策P4.7-6)。

- **`/help`**(`SLASH_HELP`)是前端静态 chrome → 改为 `t("slash.help.status")` 等逐条取词(它本就是客户端展示,决策P4.4-5);**`/status /context /skills /tools` 的正文仍由后端端点产出 → 不翻**(决策P4.7-1)。
- **`escapeHtml` 仍包所有插值**(`t()` 产出当文本插入时同样要转义),XSS 不变量不破。

## 7. 决策记录

- **\[决策P4.7-1] 只翻界面 chrome,不翻内容真相;前端独立闭环,后端一字不改。** Wiki 正文、Agent 回答、`raw/` 源、`check/health/lint` 报告正文、`/status` 等自省正文,皆为「内容/诊断数据」,由后端按本来语言产出,**原样透传**。理由:① 同构「Markdown 唯一真相」——界面是取景框,不改写水;② 翻内容要么复制全部 Python CLI 文案致与单一真相源漂移,要么上机翻**违反零-LLM**;③ 用户诉求是「界面双语」,正落 chrome。故无需给任何端点传 `lang`,后端无改动。
- **\[决策P4.7-2] 文案唯一真相在 `i18n.js` 词表;HTML/JS 不再硬编码可切换文案,只引 key。** 词表 `{zh, en}` 两套键值对照,`index.html` 静态文案标 `data-i18n*`、`app.js` 动态文案过 `t(key)`。杜绝「同一句话散落多处、改一处漏一处」。词表是 chrome 的呈现派生物,如 `index.md`/`graph.json` 之于 markdown,**永不反客为主、不进入内容**。
- **\[决策P4.7-3] 切换按钮置顶栏 `.actions` 末位(右上角),显「目标语言」码。** zh 态显 `EN`、en 态显 `中`(点它将变成的语言),比显当前语言更符直觉;复用 `.btn` + 地球图标 `i-globe`,不改两栏布局(决策P4-3)。窄屏折叠为纯图标,无需额外响应式。
- **\[决策P4.7-4] `t()` 永不崩、永不显 `undefined`。** 缺 key 时回退另一语言、再回退 key 字面量;位置插值 `{0}` 缺参补空串。宁可短暂露出 key 也不让界面碎成 `undefined`。
- **\[决策P4.7-5] 默认 `zh`,记忆「最后一次语言设置」,不做浏览器探测;静态 HTML 默认写中文 → zh 用户零 FOUC,en 用户载入一次性翻。** `localStorage["guanlan.lang"]` 记忆上次显式选择,刷新/重开沿用;首访/无记忆**写死默认 zh**(与 `<html lang>`/工作语言/静态默认文案一致),**刻意不读 `navigator.language` 兜底**——语言以「用户上次选择」为准、首访统一 zh 最可预测(「非中文浏览器首访默认 en」是一行决策,本相位不做)。`initLang()` 作 `app.js` 首个动作把 en 用户的瞬切压到最短;`<head>` 内联预设属本相位外。
- **\[决策P4.7-6] 切换=刷静态 `data-i18n` + 用「已取回数据」纯渲染少数动态面,绝不重载页面、绝不重发网络请求。** 重载会丢内存里的对话上下文(项目重视不丢上下文),故 `applyI18n(document)` 刷声明式节点 + 动态面从各自**已缓存状态**重渲。**纪律:切换走「纯渲染」、不走「取数」**——现有 `renderPage` 会 `fetch /api/page`、各 overlay opener 会打 `/api/report/*`·`/api/raw`·`/api/conversations`·`/api/workspace`,**切换绝不能重跑这些 opener**(否则重发请求、可能改内容/引失败态,破「只换 chrome 皮」)。落地前置:把这些入口拆成「取数(async,仅用户操作时跑)+ 纯渲染(同步,吃缓存数据,可被切换重跑)」——`renderPage` 加纯 chrome 刷新分支(不碰已渲染正文)、overlay opener 抽「`overlayState.data` + `renderOverlay()`」。**Wiki 单页态不重拉 `/api/page`、overlay 不重打数据端点**;聊天气泡正文(内容)不变,仅其内 chrome(按钮 aria 等)随刷。重渲染只换 chrome 皮:不重发请求、不改对话历史、不动 wiki 正文 / overlay 已取回的内容数据。
- **\[决策P4.7-7] 不引任何 i18n 库/构建/CDN;手写两套词表 + 位置插值。** 守 P4-3「vanilla JS、无 npm/构建/CDN/第三方运行时」。不做复数/性别/ICU 形态规则、不做构建期抽取、不做第三/四语言;zh/en 两套手写表足矣。第三语言或形态规则若将来需要,再议(词表结构已可平滑扩 key,加语言只需加一套 + 测试守恒放宽)。
- **\[决策P4.7-8] `t()` 首参恒为字符串字面量,条件取词在外层判分支。** `t()` 缺 key 静默回退字面量,**拼错 key 不报错、只运行时露 key**,故唯一拦截是 §8 的字面量静态扫描——而扫描只能看见字面量首参。因此**禁止** `t(cond ? "a" : "b")`、`t("pre."+x)`、`t(varKey)`:它们把 key 藏到运行时、绕过扫描。需要条件取词就**在外层判分支、每分支各自 `t("字面量")`**(`cond ? t("a") : t("b")`);极少数确需动态 key 的走**集中常量映射 + 显式测试白名单**。测试侧:凡 `t(` 首参非引号开头即判失败(非白名单)。**唯一的初始白名单**是 `applyI18n()` 内四处 `t(el.dataset.i18n*)`——它们的 key 由 HTML `data-i18n*` 属性喂入,取值域恰被「标注 key 有定义(HTML)」测试覆盖,故豁免字面量规则而不留盲区(只是兜底从 JS 侧转到 HTML 侧)。这条把「key 拼错」从运行时缺陷前移成 CI 可拦的静态错误。

## 8. 测试要点(`tests/test_web.py` 或新增 `tests/test_web_i18n.py`,零依赖、不打 LLM)

前端无 JS 测试运行时,但**词表守恒**与**标注完整**可由 Python 端**静态扫文件**守住(防漂移),无需浏览器:

| 测试 | 覆盖 | 手段 |
|------|------|------|
| 词表 key 守恒 | `I18N.zh` 与 `I18N.en` 的 key 集**完全相等**(无单边缺漏) | 正则从 `i18n.js` 抽两段对象的 key 集,断言相等(差集为空,报出缺哪边) |
| 标注 key 有定义(HTML) | `index.html` 中每个 `data-i18n*` 的值都在词表里有 key | 抽 `data-i18n(-title/-placeholder/-aria)="X"` 的 X 集,断言 ⊆ 词表 key 集 |
| **引用 key 有定义 + 首参必为字面量(JS)** | `app.js` 中每个 `t(...)` 调用:① 首参**必须是字符串字面量**(`t(cond?…)`、`t("a"+x)`、`t(varKey)` 一律**判失败**,除非在显式白名单);② 该字面量 X 都在词表里有 key——`t()` 缺 key 静默回退字面量,**拼错只在运行时露 key,静态测试是唯一拦截点**(决策P4.7-8)。**白名单**:`applyI18n()` 里的四处 `t(el.dataset.i18n*)` 是**由 HTML 的 `data-i18n*` 属性喂入的动态 key**,其取值域恰是「标注 key 有定义(HTML)」测试覆盖的集合,故**豁免字面量规则**——它们不绕过校验,只是改由 HTML 侧兜底 | 正则抽所有 `\bt\(` 调用的首参:匹配 `\bt\(\s*["']([^"']+)["']\s*[,)]` 收字面量集、断言 ⊆ 词表 key 集;**凡 `\bt\(` 后首参非 `["']` 开头者(动态 key)即 fail**,除非命中显式白名单(初始白名单 = `applyI18n` 的 `t(el.dataset.i18n)`/`t(el.dataset.i18nTitle)`/`t(el.dataset.i18nPlaceholder)`/`t(el.dataset.i18nAria)` 四条;其取值域由 HTML `data-i18n*` 测试证 ⊆ 词表) |
| 插值占位一致 | 同一 key 的 zh/en 文案含**相同的 `{n}` 占位集**(防一边漏插值致参数丢失) | 对每个 key 比对两侧 `\{\d+\}` 集合相等 |
| 无空翻译 | en 侧无空串/纯空白值(缺译应回退而非显空) | 遍历 `I18N.en` 断言每值 `strip()` 非空 |
| 边界回归(可选,webapp-testing/playwright) | 点 `#lang-btn` → 顶栏按钮文字翻成英文、`localStorage` 写入、刷新保持;Wiki 正文/Agent 气泡**不变**(内容不译) | Playwright 起本地 `guanlan web` 点按断言(重,标 optional) |

> 词表守恒 + HTML/JS 两侧引用 key 校验,皆为**纯静态文本扫描**,不依赖 `web` extra、不起服务、不触 LLM,放进现有 `test_web.py`
> 的非 importorskip 段或独立 `test_web_i18n.py` 皆可。它们专防「加了 zh 文案忘加 en」「`data-i18n` 拼错 key」「`app.js` 里
> `t("…")` 拼错 key」三类回归——尤其第三类,因 `t()` 缺 key 静默回退字面量,静态扫描是唯一拦截点。
