# P4.17 — MCP 远程传输 `guanlan mcp --transport http`（Streamable HTTP，可选叠加层）

> P4.10 把只读核心搬进**任意本地 Agent**（MCP **stdio**，子进程拉起）。P4.17 给同一套只读核心补**第二种 MCP
> 传输**——官方 **Streamable HTTP**（spec `2025-03-26` 起、取代 legacy HTTP+SSE），让 wiki 能被**跨进程 / 跨机**的
> MCP 客户端消费,而不必与调用方同机、由其作子进程拉起。**同一套只读工具逻辑、同一零写契约、同一零业务智能,只换
> 传输**（stdio 七工具;http 默认六个零 LLM 工具、可 `--allow-ask` 显式开 `ask`,见 §3.2）;`serve_mcp` 里加一个
> `--transport {stdio,http}` 分支即可,`tools.py` 零改、`build_mcp` 仅加一个 `allow_ask` 注册门。
>
> **状态**:**已实现**(`guanlan/mcp/server.py` 加 http 分支 + 辅助 `_http_security`/`_BearerTokenMiddleware`/
> `_build_http_app`/`_serve_http`、`cli.py` 六旗标、`tests/test_mcp.py` P4.17 套件含真 uvicorn 端到端往返)。它是一个
> **有界第一步**:Streamable HTTP 传输 + 无状态姿态 + **默认仍绑 `127.0.0.1`** + 一道 bearer-token 闸;**真正的远程
> 暴露故事**(TLS、OAuth resource-server per MCP `2025-06-18`、source 级多租户 scoping)**显式留给 E2**、本半阶段不做
> ——那正是 [`P4.10-MCP宿主.md`](P4.10-MCP宿主.md) §0 刻意画给 **E2(多租户权限/信任边界)** 的边界:P4.17 只跨出传输 +
> localhost-first + 单 token 闸这一步,不动数据模型、不做租户隔离。

---

## 0. 定位:为什么是 P4.17,而不是 P4.10.1 / 新里程碑 / 直接 E2

编号已在会话里辨过,这里钉死结论与理由:

- **为什么不是 P4.10.1**:`.1` 子半阶段(P4.6.1 / P5.2.1)的定性是**「补漏 + 不改任何既有契约」**。P4.17 恰恰相反
  ——stdio 单传输不是 P4.10「漏了 http」,而是它**主动画的设计边界**(P4.10 §0/决策P4.10-1:"HTTP/远程/多租户 =
  E2,不做")。补进 http 是**扩边界 + 加新契约**(新增 `--transport`/`--host`/绑定地址/鉴权),这正是 `.1` 惯例明令
  "不改"的东西。故不用 `.1`。
- **为什么不是新里程碑/P6**:它仍是「可选宿主层」同一主题下的一个自包含新能力,与 P4.11–P4.16 兄弟半阶段同性质
  ——挂 P4 家族、接在 P4.16 之后为 **P4.17**,不新开顶层。
- **为什么不是直接 E2**:多租户 + OAuth + source 级作用域 = E2(动数据模型,排 P 轨之后)。P4.17 **只出传输 + 无状态 +
  localhost-first + 单 token 闸**,不动数据模型、不做租户隔离——它和 P4.9/P4.10 一样:**本体 P4 半阶段,同时作
  "远程/scoped MCP" 的 E2 前哨**(DESIGN §7 的 E2 行已把 MCP 宿主定性为该前哨)。

### 0.1 信任边界的方向必须与 P4.11 区分

[`P4.11-信任边界.md`](P4.11-信任边界.md) 是**提示词注入**面(`raw/`/wiki 正文 = "数据、不是指令"),**纯语义**、与网络无关。
P4.17 引入的是**网络/传输信任面**(谁能连、绑哪个地址、要不要 token、DNS-rebinding),是**另一条正交的信任边界**。文档
与帮助文案须写明:**「P4.17 的鉴权/绑定 ≠ P4.11 的注入防御」**,两者不可互相顶替(决策P4.17-9)。

### 0.2 同名反向陷阱(沿用 P4.10 §0.1)

仍是**guanlan 作 MCP 服务端**把 wiki 只读暴露出去,**不是** DESIGN §1.22「Tool 注入」(Agentao 作 MCP 客户端消费外部
工具)。P4.17 只把服务端的**传输**从 stdio 扩到 http,方向不变。

## 1. 命令契约

```bash
guanlan -C <kb> mcp                              # 默认 stdio(与 P4.10 完全一致,零行为变更)
guanlan -C <kb> mcp --transport stdio            # 显式 stdio(等价默认)
guanlan -C <kb> mcp --transport http             # Streamable HTTP,默认绑 127.0.0.1:8766
guanlan -C <kb> mcp --transport http --port 9000
guanlan -C <kb> mcp --transport http --host 0.0.0.0 --allowed-host kb.example.internal --auth-token-env GUANLAN_MCP_TOKEN  # 非环回:强制 token + 反代域名放行(见 §4)
guanlan -C <kb> mcp --transport http --allow-ask # 显式开放昂贵的 ask 工具(默认 http 下**不**暴露 ask,见 §3.2)
```

- **`--transport {stdio,http}` 默认 `stdio`**(决策P4.17-1):P4.10 用户零感知、字节等价;只有显式 `--transport http`
  才走新路径。**向后兼容是硬约束**——现有 `guanlan mcp` 调用(含 `~/.claude.json` 里注册的 stdio server)不得因本半
  阶段行为漂移。
- **绑定红线沿用决策P4-4**(决策P4.17-2):`--host` **默认 `127.0.0.1`**,与 Web 宿主 `server.py:24 HOST="127.0.0.1"`
  同一条红线。绑到**非环回地址**(`0.0.0.0` 或具体网卡 IP)是**显式 opt-in 且强制要求 auth token**——无 token 时 CLI
  **拒绝启动**并报错(`EXIT_USAGE`),绝不裸暴露一个无鉴权的 wiki 到网络。
- **端口** `--port` 默认 **8766**(决策P4.17-3):与 Web 的 8765 错开一位,避免同机跑 web+mcp 撞端口。
- **`--allow-ask` 默认 `False`**(决策P4.17-6):`serve_mcp` 用 **`allow_ask=(transport=='stdio' or --allow-ask)`**
  ——**stdio 传输永远注册全部七工具**(默认 False 不影响 stdio,与 P4.10 逐字节同);**http 传输默认只六个零 LLM 工具**,
  显式 `--allow-ask` 才补上 `ask`(它拉 agentao 子进程、慢+费 token,网络面须显式承担,§3.2)。
- **无新退出码**(沿用决策P4.10-7):正常停服 `EXIT_OK`;非知识库根 / 缺 extra / 用法错(含"非环回却无 token"、
  "端口越界 1–65535"、"绑 `0.0.0.0`/`::` 却无 `--allowed-host`")= `EXIT_USAGE`(就地拒启、不监听任何端口)。
  工具内错误仍走 MCP **in-band tool error**,不映射退出码。

## 2. 模块与归口:`serve_mcp` 承担传输/绑定/鉴权,`tools.py` 零改,`build_mcp` 仅加一个 `allow_ask` 门

P4.10 已把「建 server + 注册工具」收进 `build_mcp(root, *, model, runner, search_cache) -> FastMCP`,把「起传输」收进
`serve_mcp`。P4.17 的改动**分工明确、面极小**:

- **`tools.py` 零改动**——七个工具函数(含 `tool_ask`)与传输、注册集合都无关。
- **`build_mcp` 仅新增一个 `allow_ask: bool = True` 参数**——**只**把 `ask` 的注册包进 `if allow_ask:`(现状
  server.py:139 是无条件注册);六个零 LLM 工具仍无条件注册、与 P4.10 逐字节同。**不**在 build_mcp 里塞传输/绑定/
  鉴权逻辑(那会污染它的单一职责)。
- **`serve_mcp` 承担全部传输接线**——加 `--transport` 分派、按传输算出 `allow_ask` 传给 build_mcp、http 分支配
  FastMCP 的 host/port/无状态/DNS-rebinding/token 校验(§4)。

```
guanlan/mcp/
  server.py   # build_mcp 仅加 allow_ask(if allow_ask 才注册 ask);serve_mcp 加 transport 分派 + http 绑定/鉴权接线
  tools.py    # 零改(七个工具函数与传输、注册集合都无关)
  cli.py      # _add_mcp_parser 加 --transport/--host/--port/--auth-token-env/--allowed-host/--allow-ask;_cmd_mcp 透传
```

`build_mcp` 唯一增量 = 注册开关;`serve_mcp` 现状(P4.10)末尾 `mcp.run(transport="stdio")` 扩为按 `transport` 分派:

```python
def build_mcp(root, *, model=None, runner=None, search_cache=None, allow_ask=True):
    mcp = FastMCP("guanlan", instructions=_INSTRUCTIONS, log_level="WARNING")   # 与 P4.10 同
    ...  # search/read_page/list_pages/graph/health/lint 六工具无条件注册(逐字节同 P4.10)
    if allow_ask:                       # 唯一增量:http 默认 False → 不注册 ask(决策P4.17-6)
        @mcp.tool(name="ask", ...)      # 其余不动,tool_ask 逻辑零改
        async def ask(...): ...
    return mcp

def serve_mcp(root, *, transport="stdio", host="127.0.0.1", port=8766, auth_token_env=None,
              allowed_host=None, allow_ask=False, model=None, runner=None) -> int:  # CLI --allow-ask 默认 False
    kb = require_kb_root(root, writable=False)               # 沿用只读门(决策P4.10-3)
    cache = CorpusCache(); cache.prewarm_async(kb / "wiki")  # 沿用 P5.4 候选① 预热(长驻,http 同样受益)
    # ask 门控(决策P4.17-6):stdio 永远七工具;http 默认六工具,--allow-ask 才七
    mcp = build_mcp(kb, model=model, runner=runner, search_cache=cache,
                    allow_ask=(transport == "stdio" or allow_ask))
    if transport == "stdio":
        mcp.run(transport="stdio")
    else:  # http:全部传输/绑定/鉴权接线在 serve_mcp(见 §4),build_mcp 不参与
        _configure_http(mcp, host=host, port=port, allowed_host=allowed_host, auth_token_env=auth_token_env)
        mcp.run(transport="streamable-http")
    return EXIT_OK
```

> **`_configure_http` 落点(实现期,§4 展开)**:把 host/port/`stateless_http=True`/`TransportSecuritySettings` 配到
> 返回的 FastMCP——首选 `mcp.settings` 后置赋值(已核对 `Settings` 有这些字段);若实测 SDK 要求**构造期**传入,则
> 改由 `build_mcp` 加一个可选 `http_settings` 形参透传,**仍不碰 `tools.py`/工具逻辑/六工具注册**。bearer-token 校验
> 见 §4(最小 ASGI middleware 或 `token_verifier=`,常量时间比对,不引 OAuth)。

- **SDK 复用,零依赖变更**(决策P4.17-4):官方 `mcp` SDK 的 FastMCP **原生支持 `transport="streamable-http"`**
  ——已核对本仓库装的 SDK:`FastMCP.run` 的 transport `Literal` 含 `'stdio'|'sse'|'streamable-http'`,`Settings`
  有 `host`/`port`/`streamable_http_path`/`stateless_http`/`json_response`,`mcp.server.transport_security.
  TransportSecuritySettings` 有 `enable_dns_rebinding_protection`/`allowed_hosts`/`allowed_origins`。且 **mcp SDK
  自身已把 `starlette>=0.27` + `uvicorn>=0.31.1` 列为硬依赖**(`importlib.metadata.requires("mcp")` 实测),故
  Streamable HTTP 所需 ASGI 栈随 `guanlan-wiki[mcp]` 一并到位——**`[mcp]` extra 与 SDK 下界均零改动**
  (`mcp>=1.27,<2` 已含该传输)。
- **`ask` 的子进程/嵌入姿态与 P4.10 完全一致**:仍走 `run_agent_task(..., permission_mode="read-only")` CLI query
  路径,与传输无关。http 不引入新的 `ask` 实现。

## 3. 工具集:与 P4.10 完全同集,只在 http 下对 `ask` 加闸

### 3.1 六个零 LLM 只读工具(http 下**默认全开**)

`search` / `read_page` / `list_pages` / `graph` / `health` / `lint` ——与 P4.10 §3.1 **逐字节同集、同信封、同 path 口径**
(相对库根带 `wiki/` 前缀,决策P4.10-9)。它们**无状态、纯读、可离线**,正好吃 Streamable HTTP 的**无状态模式**
(决策P4.17-7):每个 `tools/call` 是独立 POST、服务端不留 session,**无 `Mcp-Session-Id`、无事件重放存储**——与 P5.0
"stateless 检索边界"同构。这是把它们放到网络上的天然理由:调用方是 LLM,给它原料自综合,零 LLM 面既便宜又无副作用。

> **载荷上限**(沿用 P4.10-10 附):`list_pages`/`graph`/`read_page` 仍**无分页**。stdio 下"撑爆调用方 context"是唯一
> 代价;**http 下多一层"网络传大 body"代价**,大 wiki 上 `graph()` 倾倒全图可能是几十 MB。v1 仍接受(与 CLI/Web/stdio
> 一致、不引新截断口径),工具描述明示无分页;`graph` 的 summary 档留 E1。

### 3.2 `ask`(唯一 LLM 工具)在 http 下**默认关闭**,需 `--allow-ask` 显式开

http 传输把 `ask` 的两个既有代价从"本地单客户端"放大成"网络面":

- **成本/DoS 面**(决策P4.17-6):`ask` 每次拉起一整个 agentao 子进程(慢 + 费 token)。stdio 下调用方就是拉起本进程
  的那一个本地 Agent、自担成本;**http 下任意能连到端口的客户端都能触发付费 LLM 调用**——这是一个真实的成本放大 /
  拒绝服务面。故 **http 默认不注册 `ask`**,要开须 `--allow-ask` 显式承担,且(非环回时)必然已过 token 闸。
- **`ask` 子进程的 KB 写探问被放大**(沿用决策P4.10-3 的"待实证"):`ask` 的 agentao 子进程以 `<kb>` 为
  working_directory,**可能自落** `agentao.log`/`.agentao/sessions/`。stdio 下这是"待实证零写";http 下若多客户端
  高频 `ask`,任何非零写都会被放大。故:**开 `ask` over http 前,§7 的"只读保证"用例必须先对 `ask` 子进程实测零写**
  (或补等价写抑制),不得带病上网络。

> 结论:http 的**默认画像 = 六个零 LLM 只读工具 + 无状态 + localhost**。`ask` 是需显式承担成本与写探问的重路径增量,
> 不进默认远程面。

### 3.3 resources(沿用 P4.10 §3.3)

把 wiki 页暴露为只读 MCP resources 仍是 `read_page` 的自然补充,与传输正交;不阻塞 P4.17 主体,可随后补。

## 4. 鉴权与绑定:localhost-first + 强制 token 闸(有界,非完整 E2)

P4.17 **不做**完整的 MCP `2025-06-18` OAuth resource-server 鉴权(那是 E2)。v1 只给一道**够用且安全默认**的闸:

- **环回默认无 token**(决策P4.17-2):`--host 127.0.0.1`(默认)时**可不带 token**——与 Web 宿主"靠 localhost-only
  绑定作信任边界"同姿态(Web 至今无 token,靠 `127.0.0.1` 红线)。这覆盖"同机、跨进程"的主用例(如本机另一个容器/
  另一个 Agent 进程连本机 wiki)。
- **非环回强制 bearer token**(决策P4.17-2/8):`--host` 非 `127.0.0.1`/`::1`/`localhost` 时,**必须**经
  `--auth-token-env <ENVVAR>` 提供一个 bearer token(从环境变量读、**绝不**从命令行明文传、**绝不**落盘),server 校验
  `Authorization: Bearer <token>`;缺 token 即 `EXIT_USAGE` 拒启。**无鉴权裸暴露到网络 = 直接拒绝**,这是红线。
  env 值经 `.strip()`——**空白-only(误设 `' '` 或 `$(cat)` 带尾换行)不算有效 token**、一律拒启,免弱鉴权裸暴露。
  - **落点(决策P4.17-2,实现已定,不提前带进 E2 复杂度)**:token 校验用**最小实现**——一层薄 ASGI middleware
    (`_BearerTokenMiddleware`)包住 `mcp.streamable_http_app()`(**未取** `token_verifier=`,因后者需一整套 `auth`
    settings/issuer_url 的 OAuth resource-server 机制,与"不引 OAuth"相悖)。它拆 `Authorization` 头的
    `scheme + credential`:scheme 按 **RFC 7235 大小写不敏感**(`bearer`/`BEARER` 均合法)、容多空格;凭据本体走
    `hmac.compare_digest` **常量时间比对**,失败即 401。非 http scope(lifespan/websocket)原样透传,不拦掉
    `session_manager` 生命周期。**明确不引** OAuth / resource-server 元数据 / scope / 用户模型 / token 发放轮换
    ——那些是 E2。这层是 P4.17 唯一的鉴权代码,约十余行。
- **DNS-rebinding 防护 + 反代 Host**(决策P4.17-5):Streamable HTTP 即使绑 localhost 也需防浏览器侧 DNS-rebinding。
  用 MCP SDK 的 `TransportSecuritySettings`(`allowed_hosts`/`allowed_origins`,SDK **默认已启用** rebinding 防护并
  预置环回三件套)校验 `Host` 头。**allowed_hosts 派生自 `--host`、不写死 `127.0.0.1`**:环回默认放行
  `127.0.0.1:*`/`localhost:*`/`[::1]:*`;绑**具体非环回地址**时补该地址(**IPv6 字面量加方括号**——客户端发的是
  `[fe80::1]:port`,不加括永不匹配);绑非环回或**经反代**时(§5 的 `kb.example.internal`,到达服务端的 `Host` 是**外部
  域名**、与环回默认冲突),须把该对外域名放进 allowed_hosts——由一个**很小的** `--allowed-host <host[:port]>`(可重复)
  显式补入(**大小写归一为小写**,因 DNS 名不区分大小写而 SDK 按字节精确匹配;自动补一档任意端口通配兼容裸域名/域名:端口)。
  **`0.0.0.0`/`::` 是绑定通配符、不是可用 `Host` 值**:绑它们而**不**配 `--allowed-host` 会导致白名单只含环回、每个远程
  请求都 421(起而不可达),故 serve_mcp 此情形**拒启并明示**(替代"静默不可达")。**不做复杂 origin 策略**——只校 `Host`
  (origins 随 hosts 同步派生以免误杀反代下浏览器客户端)。
- **TLS 由外层承担**(决策P4.17-8):v1 **不内建 TLS**。真要跨机明文之外的加密,推荐前置反代(nginx/caddy)或
  SSH 隧道终止 TLS——与"完整远程故事推给 E2"一致。文档给出反代片段示例(§5),但 guanlan 本体只出明文 http +
  token 校验。

## 5. 客户端注册(文档示例,非代码)

**同机跨进程(默认 localhost,无 token):**
```jsonc
{ "mcpServers": {
    "guanlan-http": { "type": "streamable-http", "url": "http://127.0.0.1:8766/mcp" }
} }
```

**跨机(经反代 + token):**
```jsonc
{ "mcpServers": {
    "guanlan-remote": {
      "type": "streamable-http",
      "url": "https://kb.example.internal/mcp",
      "headers": { "Authorization": "Bearer ${GUANLAN_MCP_TOKEN}" }
} } }
```
服务端侧:`GUANLAN_MCP_TOKEN=… guanlan -C <kb> mcp --transport http --host 0.0.0.0 --port 8766 --allowed-host kb.example.internal --auth-token-env GUANLAN_MCP_TOKEN`,
再由 caddy/nginx 在前面终止 TLS 并转发到 `127.0.0.1:8766`。**`--allowed-host kb.example.internal` 不可省**——反代通常透传原始
`Host: kb.example.internal`,不补进 allowed_hosts 会被 DNS-rebinding 防护(决策P4.17-5)拒掉(§4)。

## 6. 边界(明确不做)

- **不做完整 OAuth / MCP `2025-06-18` resource-server 鉴权**(E2):v1 只有单 bearer token 闸,无租户、无 scope、无
  token 轮换/发放。
- **不做 source 级作用域 / 多租户隔离**(E2,动数据模型):所有连上的客户端看到的是**同一个整库只读视图**,与 stdio
  一致。
- **不实现 legacy HTTP+SSE 传输**(决策P4.17-10):它已被 spec `2025-03-26` deprecated;新建远程面只上 Streamable
  HTTP,**绝不**为兼容老客户端回头实现双端点 always-on SSE。
- **不内建 TLS / 不绑 `0.0.0.0` 而不带 token**:加密与真实网络暴露由外层反代 + token 闸承担(§4)。
- **不暴露任何写工具**(沿用决策P4.10-3):ingest/heal/backfill/raw/upload 全不注册,http 下亦然。
- **不做服务端会话状态**:http 走**无状态模式**(`stateless_http`),不留 `Mcp-Session-Id`、不做事件重放/断线续传
  (resumability 属后续,若真需要长流 `ask` 的可恢复性再评估 EventStore,决策P4.17-7)。
- **不改 stdio 路径**:`--transport stdio`(默认)与 P4.10 字节等价;stdio 的"stdout 洁净"最高不变量(决策P4.10-13)
  在 http 下**不适用**(http 不用 stdout 作传输帧),但 http 分支**不得**回头污染 stdio 分支的洁净假设。

## 7. 测试要点(`tests/test_mcp.py`,装有 mcp extra + http 依赖时跑;缺则 skip,镜像 P4.10)

- **默认 stdio 零漂移**(决策P4.17-1):不传 `--transport` 时行为与 P4.10 **字节等价**(同一套 P4.10 用例全绿);
  `--transport stdio` 显式等价默认。
- **http 传输冒烟**:`--transport http` 起服后,用 MCP streamable-http **客户端**(SDK 自带或测试夹具)跑通
  `tools/list` + 六个零 LLM 工具的 `tools/call`,断言结果与 stdio / CLI / Web 经 `search_result_dict`/`report_dict`
  **同形**(证明只换传输、无逻辑分叉)。
- **无状态姿态**(决策P4.17-7):连续多个 POST **不**携带/不要求 `Mcp-Session-Id`,每个独立成功;并发多 `search`
  经共享 `CorpusCache` 自持锁不错乱重建(沿用决策P4.10-15,http 下同样是单进程多 in-flight)。
- **绑定红线 + token 闸**(决策P4.17-2/8):(a) `--host 0.0.0.0` 且**无** `--auth-token-env` → CLI 拒启、返
  `EXIT_USAGE`、不监听任何端口;(b) 带 token 时,缺/错 `Authorization: Bearer` 的请求被拒(401),正确 token 放行;
  (c) 默认 `127.0.0.1` 无 token 可正常连。
- **DNS-rebinding 防护 + `--allowed-host` 派生**(决策P4.17-5):(a) 环回默认下,伪造 `Host` 头(非
  `127.0.0.1:<port>`/`localhost`)的请求被 `TransportSecuritySettings` 拒;(b) 起服带 `--allowed-host
  kb.example.internal` 时,该域名的 `Host` 被**放行**(证明反代路径不被 rebinding 防护误杀)、其余仍拒——覆盖 finding
  修正后的"派生自 `--host`/`--allowed-host`、不写死 localhost"规则。
- **`ask` http 门控**(决策P4.17-6):默认 `--transport http`(无 `--allow-ask`)时 `tools/list` **无** `ask`;
  加 `--allow-ask` 才出现。且开 `ask` over http 的**只读保证**用例:跑 `ask`(mock runner)后 KB 仍零字节写、无
  `agentao.log`/`.agentao/sessions/`(与决策P4.10-3 同,http 下**实测**,不得跳过)。
- **依赖门控**:缺 `guanlan-wiki[mcp]` 时,`guanlan mcp --transport http` 优雅降级提示安装、返 `EXIT_USAGE`、不吐
  traceback(镜像 P4.10 决策P4.10-2,用 `monkeypatch` 模拟 ImportError 以便装有 extra 的 CI 也覆盖降级路径)。
  uvicorn/starlette 随 mcp SDK 到位(决策P4.17-4),**无独立 http 依赖门**。
- **只读 + 无写工具**(沿用 P4.10):http 下 `tools/list` 无任何写工具;`read_page("../outside.md")`/绝对路径 → in-band
  越界错误,不读到 `wiki/` 外。
- **方向不混**(沿用 P4.10-6 / 决策P4.17-9):帮助文案/README 明确"服务端 http 传输",且区分"P4.17 网络信任边界 ≠
  P4.11 注入信任边界"。

## 8. 决策清单(沿用仓库 `决策PX-N` 体例)

- **决策P4.17-1(`--transport {stdio,http}` 默认 stdio,向后兼容硬约束)**:P4.10 用户零感知;只有显式 `http` 走新路。
- **决策P4.17-2(绑定红线 + 非环回强制 token)**:`--host` 默认 `127.0.0.1`(沿用决策P4-4);非环回**必须**带
  `--auth-token-env`,否则拒启 `EXIT_USAGE`。绝不无鉴权裸暴露。
- **决策P4.17-3(端口默认 8766)**:与 Web 8765 错开,防同机撞端口。
- **决策P4.17-4(SDK 复用,零依赖变更)**:FastMCP 原生 `transport="streamable-http"`,`mcp>=1.27,<2` 已含;**已实测**
  mcp SDK 自带 `starlette>=0.27`+`uvicorn>=0.31.1` 硬依赖,ASGI 栈随 `[mcp]` extra 到位,**extra 与 SDK 下界均不改**。
- **决策P4.17-5(DNS-rebinding 防护;allowed_hosts 派生自 `--host`,不写死 localhost)**:用 `TransportSecuritySettings`
  校验 `Host`;环回默认放行 `localhost`/`127.0.0.1`,非环回/反代由 `--allowed-host`(可重复)显式补对外域名(否则反代的
  外部 Host 会被拒),缺省从 `--host` 派生;不做复杂 origin 策略。核对 SDK 是否已默认启用。
- **决策P4.17-6(`ask` over http 默认关;门控公式钉死)**:CLI `--allow-ask` 默认 `False`;`serve_mcp` 传
  **`allow_ask=(transport=='stdio' or --allow-ask)`** 给 build_mcp——**stdio 永远七工具**,http 默认六工具、`--allow-ask`
  才七。理由:http 把 `ask` 的成本/DoS 面与子进程写探问放大,默认远程面只含六个零 LLM 只读工具。
- **决策P4.17-7(无状态优先)**:http 走 `stateless_http`——无 `Mcp-Session-Id`、无事件重放,契合零 LLM 工具的
  stateless 边界与 P5.0/P5.4。长流 `ask` 的可恢复性(EventStore)留后续。
- **决策P4.17-8(TLS 外置)**:v1 不内建 TLS;加密由反代/隧道承担,与"完整远程推 E2"一致。
- **决策P4.17-9(网络信任边界 ≠ P4.11 注入信任边界)**:两条正交信任线,不互相顶替;文档写明。
- **决策P4.17-10(不实现 legacy HTTP+SSE)**:已 deprecated;只上 Streamable HTTP。
- **决策P4.17-11(`tools.py` 零改;`build_mcp` 仅加 `allow_ask` 门;传输接线全在 `serve_mcp`)**:`tools.py` 逐字节不动;
  `build_mcp` 唯一增量是 `allow_ask: bool=True`(把 `ask` 注册包进 `if allow_ask:`,六工具无条件注册照旧);host/port/
  无状态/DNS-rebinding/token 校验全落 `serve_mcp` 的 http 分支(经 `mcp.settings` 后置赋值 + 最小 token middleware)。
  七工具逻辑与传输正交,复用 P5.4 预热、P4.10-15 并发锁、P4.10-16 in-band error 总壳,全部照旧。
