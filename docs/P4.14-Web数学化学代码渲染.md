## 1. 目标与范围

**P4.14 = 让 Web 宿主把页面里的数学/化学/代码块渲染成富呈现。** 现状：

```text
$E = mc^2$               → 字面 "$E = mc^2$" 文本（python-markdown 不识 math 分隔符，原样穿过）
$$\nabla \cdot \vec E$$  → 同上，字面
$\ce{2H2 + O2 -> 2H2O}$  → 字面文本（化学 mhchem 须在数学分隔符内，决策P4.14-2）
```python\nfor x in xs: ...\n```  → <pre><code class="language-python">for x in xs: …</code></pre>（无高亮、纯灰）
```

——浏览器里数学是源码、代码是无色等宽。LLM 综合 wiki 页时常写公式表达定义/推导、写 `$\ce{}$` 表达反应式、贴代码
片段佐证；用户也会在源里手写。P4.14 补这层「最后一公里」：**同一段 markdown 在浏览器里成公式/成色，在 CLI / 纯文本
回退里仍是 honest 的源码**（markdown 仍是唯一事实来源，§DESIGN 原则 1 不破）。

落点是**纯前端**，与 P4.13 同骨架但**多一处本质差异**——**载体不同**（决策P4.14-3 的核心张力）：

| 内容类型 | 服务端载体（`render.py` 现状产物） | 前端拾取 | 渲染器 |
|---|---|---|---|
| **代码高亮** | ` ``` ` 围栏 → `<pre><code class="language-X">…转义源…</code></pre>`（**与 mermaid 同一载体**，fenced_code 契约） | `querySelectorAll('pre>code[class*=language-]')` 读 `.textContent` | **highlight.js**（vendored UMD） |
| **数学公式 / 化学** | **无围栏载体**——`$…$`/`\(…\)`/`\[…\]`（化学 mhchem `\ce{}` 写在分隔符内）作**行内/块文本**原样穿过，落成 HTML 里的**转义文本节点** | KaTeX `renderMathInElement` **遍历文本节点**按分隔符就地排版 | **KaTeX + mhchem + auto-render**（vendored） |

代码高亮的载体与 mermaid **完全相同**（`fenced_code` 已 emit `code.language-X`），故几乎是 P4.13 的机械复刻；
**数学/化学没有围栏载体**——`$…$` 不是围栏块，python-markdown 把它当普通文本穿过、落成转义文本节点。这就是本相位
**唯一的新机制**：用 KaTeX auto-render 的「遍历容器文本节点、按分隔符排版」（DOM 走查）拾取，而非靠某个 class。§3 详述，
§4 论证它为何仍是「读文本、不读 HTML」因而安全闸不破。

### 纳入 P4.14

- **代码高亮**（`static/code_enhance.js`，约 50 行）：`highlightCode(container)` 扫
  `container.querySelectorAll('pre>code[class*="language-"]')`，**跳过 `language-mermaid`**（那归 mermaid）、**跳过
  已高亮**（`.hljs`，避免 highlight.js v11 的重复高亮告警）、**跳过语言未注册的块**（`hljs.getLanguage(lang)` 为空 →
  保持纯文本、不猜不报错，决策P4.14-7）；命中者 `hljs.highlightElement(code)` 就地高亮。**懒加载**（首个可高亮块出现才
  注入 vendored `highlight.min.js`）。
- **数学/化学**（`static/math_enhance.js`，约 70 行）：`typesetMath(container)` →（懒加载 KaTeX 后）
  `renderMathInElement(container, OPTS)`。auto-render **遍历文本节点**按分隔符 `$$…$$`/`\[…\]`（块级）、`$…$`/`\(…\)`
  （行内）就地排版，`ignoredTags` 含 `pre/code/...`（默认即含，代码块内 `$` 不排版、保字面）。化学经 **mhchem**
  （`\ce{}`/`\pu{}`，**须写在数学分隔符内**如 `$\ce{H2O}$`——mhchem 是 math-mode TeX 宏，KaTeX 只在分隔符内拾取、
  **裸 `\ce{}` 不渲染**，决策P4.14-2）——**与数学共用同一运行时**（mhchem 是 KaTeX contrib 扩展、非独立库），故「数学+
  化学」只引一套运行时。
- **编排器**（`static/content_enhance.js`，约 20 行）：`enhanceContent(container)` = `enhanceMermaid` +
  `highlightCode` + `typesetMath` 顺序调用，**把 P4.13 的单一 mermaid 钩子泛化为内容增强编排器**（决策P4.14-9）。
  四类注入点把现有 `enhanceMermaid(x)` 调用**替换为 `enhanceContent(x)`**（一标识符之改，不增删注入点）。三个增强彼此
  **区域不相交**（代码在 `pre>code`、数学跳过 `pre/code`、mermaid 在 `language-mermaid`），故顺序仅为整洁。
- **vendored 运行时**（钉版、随 `web` extra 打包、**非 CDN**、离线可用，决策P4.14-1/-2）：
  - **KaTeX 资产树** `static/vendor/katex/`：`katex.min.js`（核心，~280KB）、`contrib/mhchem.min.js`（化学扩展，载入后
    自动注册 `\ce`/`\pu` 进 KaTeX）、`contrib/auto-render.min.js`（暴露 `window.renderMathInElement` 的文本节点走查器）、
    `katex.min.css`（**必需**——定位排版字形 + `@font-face`）、`fonts/KaTeX_*.woff2`（字体文件，~20 个；`katex.min.css`
    内以**相对路径** `url(fonts/…)` 引用，故 css 与 `fonts/` 须同级 vendored，决策P4.14-5）。**非单文件**（带 css+字体），
    但仍**无构建、无 CDN、离线自洽**（字体经 `@font-face` 由浏览器同源拉取，无运行时 JS 动态加载）。**总资产 ~0.8MB**
    （比 MathJax tex-svg-full ~1.4MB 更小，决策P4.14-2）。
  - `static/vendor/highlight/highlight.min.js`：highlight.js v11 「common」构建（~35 语言），自包含 UMD。
- **安全配置硬编码**（决策P4.14-4 信任铰链，本相位实质）：
  - KaTeX：`renderMathInElement` 选项硬编码 **`trust: false`**（**KaTeX 默认即此**，显式写明、无放宽旋钮）——禁
    `\href`/`\url`/`\includegraphics`/`\html*`（`\htmlClass`/`\htmlId`/`\htmlData`/`\htmlStyle`），令其渲成错误色而非生效；
    **不传共享 `macros` 对象**（每次 `render(container, {...KATEX_OPTS})` 展开新选项）——auto-render 默认 `macros:{}`
    **每容器一份**，故 `\gdef`/`\newcommand` **不跨容器（页/气泡/预览）泄漏**（决策P4.14-4 隔离铰链）；**容器内**多公式共用
    那一份 `{}`，页内 `\gdef` 可生效（页内局部 preamble，benign、受 `maxExpand` 限），守的是**跨容器**不污染、非禁绝页内宏；
    `throwOnError: false`（语法错退回错误色源码）、`strict: 'ignore'`（仅静默告警，非安全项）、`ignoredTags` 含
    `pre/code/...`（代码块内 `$` 不排版、保字面）。
  - highlight.js：v11 **默认即安全**——`highlightElement` 读 `.textContent`（已转义源）、产物**只**是
    `<span class="hljs-*">转义文本</span>`、**剥除一切未转义 HTML 并告警**（v11 移除 HTML 透传）。喂转义 `textContent`、
    **每块只高亮一次**即合规。
- **优雅降级 + 失败封闭**（决策P4.14-8，同 P4.13-6）：
  - 数学：运行时加载失败 → `$…$` **文本原样留存**（KaTeX 未成功前不动文本）；**单个公式语法错** → `throwOnError:false`
    令其退回**错误色源码**（可见、不可执行）；**被禁命令**（`\href` 等，`trust:false`）→ 同样渲成错误色、**不生效**。
  - 代码：运行时加载失败 / 单块异常 → `<pre><code>` **保留为可读纯文本代码**（本就可读，比 mermaid 更软）。
  - 两者**任何异常路径都不产生未消毒注入、永不空白、源码 honest**。
- **观澜配色主题（最小、单档昼澜、不预留切换）**（决策P4.14-6，同 P4.13-5）：
  - highlight.js **不 vendored 第三方主题 CSS**（github.css 等），改在 `app.css` 写一档 **~15 行昼澜调色 hljs token
    主题**（`.hljs-keyword`/`.hljs-string`/`.hljs-comment`/…映射 `:root` 昼澜变量），与界面同水墨澜调、零额外 vendored CSS。
  - KaTeX **必需其 vendored `katex.min.css`**（排版定位 + 字体，不可省）；`app.css` 仅补 `.katex`/`.katex-display` 的
    **颜色继承**（`color: var(--lan-ink)`）+ 宽公式 `.katex-display{overflow-x:auto}` 防溢出（~3 行），不改 KaTeX 排版骨架。
  - **仅昼澜一档、不预留深色切换接口**（决策P4-9 推后的「夜澜」真落地时再加，避免过度设计）。
- **i18n**：**无新增 key**——数学语法/被禁命令由 KaTeX 就地错误色渲染（自含、无文案）、代码降级为纯文本（无需文案）。
  保持 i18n 最小（较 P4.13 更省，连 `renderFail` 都不需要）。`test_web_i18n.py` 的 zh/en key 平价**自动维持**。

### 不纳入 P4.14（按 P4.13 同款收窄推后）

- **服务端预渲染 / 远程渲染**（决策P4.14-5 否决，同 P4.13-2）：KaTeX/highlight 均需 JS+DOM；服务端预渲染要背 headless
  浏览器（重依赖），远程渲染会外发内容 + 破离线（撞 §8 仅 `127.0.0.1`/隐私姿态）。渲染**只在浏览器、按需**。
- **服务端 math 扩展（`pymdownx.arithmatex` 等把 `$$…$$` 包成 `<span class=arithmatex>`）** → 不做（决策P4.14-3）：
  会**改 `render.py`、破其字节稳定不变量、加服务端依赖**。前端 DOM 走查（auto-render）足够，且服务端零改是 P4.13 立下
  的硬约束。代价是 `$…$` 与字面美元号的歧义残留，按下文处置（presentation-only、源码 honest）。
- **MathJax（决策P4.14-2 取 KaTeX 而否 MathJax）** → 不做：MathJax `tex-svg-full` 单文件含 mhchem 看似更自洽，但其安全
  约束**需在一个「默认激活全部扩展」的合并包上做减法**（`tex.packages: {'[-]': [...]}` 移除 `html`/`href`/`require`/
  `autoload`/`newcommand`/`setoptions`…），而 ① 纯数组 `packages:[...]` 的「替换 vs 合并」语义 MathJax 文档未明确保证；
  ② `[-]` 黑名单的**完备性难以担保**（`-full` 还激活 `setoptions`——允许 TeX 源在运行时改 MathJax 选项；且 `\href` 是否属
  可移除包亦不确定）。把安全压在「一个全量包的减法黑名单」上是脆弱地基。KaTeX `trust:false` 是**默认即安全、单旗、文档
  明载**的白名单式铰链，结构上消除整类不确定——详见决策P4.14-2。
- **非 mermaid 图语言 / 别的标记 DSL**（plantuml/graphviz/asciimath…）→ 不做：无实证需求，每多一种 = 多一个第三方
  运行时 + 多一片安全面。AsciiMath 亦不做（只认 LaTeX `$…$` 与其内的 mhchem `$\ce{}$`，与 LLM 主流输出一致）。
- **裸 `\ce{}`/`\pu{}` 自动包裹成 math** → 不做：mhchem 是 math-mode 宏，须由作者写在数学分隔符内（`$\ce{H2O}$`，mhchem
  标准用法）。前端预处理「扫文本把裸 `\ce{}` 包成 `$…$`」会**扩大解析面 + 引入误包裹歧义**，与「读文本不改语义」相悖，不做。
  裸 `\ce{}`（无分隔符）按 honest 字面文本留存（不渲染）。
- **CLI / MCP 端渲染** → 不做（决策P4.14-8/同 P4.13-7）：文本通道（终端 / JSON-RPC），渲染是宿主表现层职责；CLI
  `query` 回字面源码（honest），MCP 回 markdown 原文由客户端自渲。
- **语言未在 highlight「common」构建内的代码块** → 降级为纯文本（不动态拉语言子包——那会破离线 / 触发网络），
  `vendor/README.md` 记录覆盖语言集；不在覆盖内者**可见地保留纯文本代码**（非静默，决策P4.14-7「不静默截断」）。
- **跨页/跨容器共享宏 preamble / 公式编号交叉引用** → 不做，**跨容器结构上隔离；页内局部宏不禁绝（benign）**：KaTeX
  auto-render 的 `macros` 默认是**每次 `renderMathInElement` 调用一份**的空对象，并在该次调用内传给多公式（官方文档明载）。
  本相位 `render(container, {...KATEX_OPTS})` 不传共享 `macros` → ① **跨容器**（页/气泡/预览各一次调用）各自独立，
  `\gdef`/`\newcommand` **不跨页/跨气泡泄漏**（这是「页面自洽」要守的、也是评审关切的污染面）；② **容器内**（同一页/同一
  气泡）多公式共用那份 `{}`，故页内 `\gdef` 可生效——属**页内局部 preamble**，受 KaTeX `maxExpand`（默认 1000）限、无法
  膨胀爆炸、`trust:false` 下也注入不了 HTML，故 benign，**不专门禁绝**（要全禁需绕开 auto-render 自扫分隔符逐公式传
  `fresh {}`，代码/攻击面更大，收益仅是挡住无害的页内宏，不值，决策P4.14-4）。无需「每次重绘重建 TeX 状态」——跨容器
  隔离靠 auto-render 的「每调用一份默认 macros」+ 我们每次展开新选项，是默认行为。
- **实时编辑器 / 输入即预览 / 图源编辑** → 不做（同 P4.13）：本相位只读渲染既有页里的公式/代码。
- **KaTeX `\href` 站内导航 / 代码块内点击交互** → 不做（被 `trust:false` 关死，安全优先；站内导航仍走 `[[wikilink]]`）。
- **自动给页面注入公式/代码** → 不做：写不写公式是 LLM 综合 / 用户手写的内容决策，宿主只渲染既有源。

## 3. 拾取与渲染（三增强 + 一编排器）

### 3.1 代码高亮（`code_enhance.js`）——机械复刻 mermaid 载体

```js
let _hljsPromise = null;                                  // 懒加载单例（同 mermaid_enhance.js 范式）
function loadHljs() {
  if (_hljsPromise) return _hljsPromise;
  _hljsPromise = new Promise((resolve, reject) => {
    if (window.hljs) return resolve(window.hljs);
    const s = document.createElement("script");
    s.src = "/static/vendor/highlight/highlight.min.js";
    s.onload = () => window.hljs ? resolve(window.hljs) : (s.remove(), reject(new Error("no hljs")));
    s.onerror = () => { s.remove(); reject(new Error("hljs 加载失败")); };
    document.head.appendChild(s);
  }).catch((e) => { _hljsPromise = null; throw e; });     // 失败不缓存、可重试（同 P4.13）
  return _hljsPromise;
}

async function highlightCode(container) {
  if (!container) return;
  const blocks = [...container.querySelectorAll('pre > code[class*="language-"]')].filter(
    (c) => !c.classList.contains("language-mermaid") && !c.classList.contains("hljs")  // 跳 mermaid / 已高亮
  );
  if (!blocks.length) return;                              // 早退：无可高亮块零加载（懒，决策P4.14-1）
  let hljs;
  try { hljs = await loadHljs(); } catch { return; }       // 加载失败 → 保留纯文本代码（决策P4.14-8）
  for (const code of blocks) {
    if (!code.isConnected || code.classList.contains("hljs")) continue;  // await 期间可能被重绘/已高亮
    const lang = (code.className.match(/language-([\w-]+)/) || [])[1];
    if (!lang || !hljs.getLanguage(lang)) continue;        // 未注册语言 → 纯文本、不猜不报错（决策P4.14-7）
    try { hljs.highlightElement(code); } catch { /* 单块异常 → 保留纯文本 */ }
  }
}
window.highlightCode = highlightCode;
```

- 载体 `pre>code.language-X` 与 mermaid **同一契约**（`fenced_code` emit），故拾取逻辑几乎复刻；**唯三差异**：跳
  `language-mermaid`（归 mermaid）、跳 `.hljs`（highlight.js v11 重复高亮会告警「unescaped HTML」，§4）、跳未注册语言
  （保字面、不静默失败，决策P4.14-7）。
- highlight.js v11 `highlightElement` 读 `.textContent`（已转义源、反转义为代码文本）、产物**只**是
  `<span class="hljs-*">转义文本</span>`——**不把输入当 HTML**（v11 移除透传、剥未转义 HTML），故安全（§4）。

### 3.2 数学 / 化学（`math_enhance.js`）——新机制：DOM 文本走查 + `trust:false` 铰链

```js
const KATEX_OPTS = {                                       // 硬编码、无放宽旋钮（决策P4.14-4）
  delimiters: [                                            // 块级在前（$$ 须先于 $ 匹配）
    { left: "$$", right: "$$", display: true },
    { left: "\\[", right: "\\]", display: true },
    { left: "$", right: "$", display: false },
    { left: "\\(", right: "\\)", display: false },
  ],
  ignoredTags: ["script", "noscript", "style", "textarea", "pre", "code", "option"], // 代码块内 $ 不排版
  ignoredClasses: ["page-meta"], // 跳过 chrome：wiki.js paintPage 把 .page-meta（页路径/更新时间）与正文同放 view
                                  // 内（wiki.js:196），auto-render 扫文本节点会连 chrome 一起扫——令其跳过（P2 修）
  trust: false,            // 安全铰链（KaTeX 默认即此，显式写明）：禁 \href/\url/\includegraphics/\html*（§4）
  strict: "ignore",        // 仅静默告警，非安全项
  throwOnError: false,     // 语法错 → 退回错误色源码、不抛（决策P4.14-8）
  // **不设 macros** → auto-render 每次调用自造一份默认 macros:{}，**跨容器（页/气泡/预览）隔离**；
  // 容器内多公式共用那一份 {}，故页内 \gdef 可生效（页内局部 preamble，benign、受 maxExpand 限），
  // 但**不跨容器泄漏**——这正是「页面自洽」要守的边界（决策P4.14-4 隔离铰链，§4-③）。
};

let _katexPromise = null;                                 // 懒加载单例
function loadKatex() {
  if (_katexPromise) return _katexPromise;
  _katexPromise = new Promise((resolve, reject) => {
    if (window.renderMathInElement) return resolve(window.renderMathInElement);
    // css 必需（排版定位 + @font-face）；以 <link> 懒注入（决策P4.14-5）。
    if (!document.getElementById("katex-css")) {
      const l = document.createElement("link");
      l.id = "katex-css"; l.rel = "stylesheet"; l.href = "/static/vendor/katex/katex.min.css";
      document.head.appendChild(l);
    }
    // 按序加载：katex → mhchem（扩展 katex，须在其后）→ auto-render（暴露 renderMathInElement）。
    loadScript("/static/vendor/katex/katex.min.js")
      .then(() => loadScript("/static/vendor/katex/contrib/mhchem.min.js"))      // 注册 \ce/\pu 进 katex
      .then(() => loadScript("/static/vendor/katex/contrib/auto-render.min.js"))
      .then(() => window.renderMathInElement
        ? resolve(window.renderMathInElement)
        : reject(new Error("no renderMathInElement")))
      .catch(reject);
  }).catch((e) => { _katexPromise = null; throw e; });     // 失败不缓存、可重试（同 P4.13）
  return _katexPromise;
}

async function typesetMath(container) {
  if (!container) return;
  // 轻量预判：容器内无「**成对**数学分隔符」则零加载（懒，决策P4.14-1）。四支**都要求闭合**——`$$…$$`/`\(…\)`/`\[…\]`
  // （块级，跨行）/`$…$`（行内，≥1 非 $ 字符、不跨行）——故**孤立的开头符**（单个 `$5`/裸 `$$`/裸 `\(`/裸 `\[`，无配对
  // 闭合）**不触发**（P3 修，比「含开头符」保守、与「成对」口径一致）。化学 \ce 必在 `$…$` 对内、已覆盖；裸 \ce{} 不渲染、
  // 不探（决策P4.14-2）。**残留（启发式，benign）**：① `$…$` 对内含货币（如「$5 至 $10」）等罕见情形仍命中、付一次加载
  // （甚至误排版，决策P4.14-3 的 `$` 歧义）；② textContent 含**代码块**里的成对分隔符（如 LaTeX 代码样例）也会命中、付一次
  // 加载——但 auto-render 的 `ignoredTags` 随后跳过 `pre/code`、**不误排版**，仅一次无谓加载（首载后缓存）。两者皆 presentation-only。
  if (!/\$\$[\s\S]*?\$\$|\\\([\s\S]*?\\\)|\\\[[\s\S]*?\\\]|\$[^$\n]+\$/.test(container.textContent || "")) return;
  let render;
  try { render = await loadKatex(); } catch { return; }    // 加载失败 → $…$ 文本原样留存（决策P4.14-8）
  try { render(container, { ...KATEX_OPTS }); }             // 展开新选项对象/调用：auto-render 的默认 macros 每容器
  catch { /* 整体异常 → 文本保留、不注入 */ }                 // 一份、跨容器隔离（不在 const KATEX_OPTS 上累积宏）
}
window.typesetMath = typesetMath;
```
（`loadScript(src)` 为一个返回 Promise 的注入 `<script>` 小工具：`onload`→resolve、`onerror`→`s.remove()`+reject，
与 `mermaid_enhance.js` 的注入范式一致。）

- **新机制：DOM 文本走查**（决策P4.14-3）。数学无围栏载体，故不靠 class——`renderMathInElement(container, OPTS)`
  **遍历容器文本节点**，按分隔符就地排版。`ignoredTags` 含 `pre/code/...`（默认即含）→ **代码块内的 `$` 不排版、保字面**
  （与「代码字面语义」一致，决策P4-3）。
- **化学=数学同运行时**：`$\ce{2H2 + O2 -> 2H2O}$`（**须在分隔符内**，决策P4.14-2）经 mhchem 扩展（载入后注册进 KaTeX）
  排版，无需独立库；裸 `\ce{}`（无分隔符）不被拾取、按字面文本留存。
- **`$` 歧义残留**（决策P4.14-3）：`delimiters` 含 `$…$` → 文本里「$5 与 $10」可能被误当行内公式。缓解：优先用**无歧义
  的** `\(…\)`（行内）/ `\[…\]`、`$$…$$`（块级）；`$…$` 的美元号歧义为 **presentation-only 残留**——源码始终 honest，
  CLI/纯文本回退原样可见，且语法不通时 `throwOnError:false` 退回错误色源码（不静默吞）。**不为消歧而改服务端**（那破
  `render.py` 字节稳定）。

## 4. 安全分析（**本相位的实质**）　**\[决策P4.14-4]**

P4 的 XSS 纵深防御**全靠服务端转义**（`_EscapeHtmlExtension` 关原始 HTML + `_SafeLinkExtension` 中和危险协议）；
P4.13 首次让「内容→客户端 `innerHTML` 注入」（mermaid SVG，以 `strict` 闸守）。P4.14 新增**两个**这样的注入面
（highlight 高亮 span、KaTeX 排版 DOM），故安全分析仍是核心、不是附注。

**威胁模型**（同 P4.13）：被投喂的 `raw/` 可能夹带恶意载荷，经 ingest 进 wiki 页正文；攻击者写一段代码块或公式，企图
让产物携带 `<script>`/事件处理器/`javascript:` 链接，经客户端注入后以**同源**身份调本地写/读 API。

**逐渲染器论证「为何挡得住」：**

**① 代码高亮（highlight.js v11）：**
1. **输入是文本、非 HTML。** `highlightElement` 读 `code.textContent`——浏览器对 `<code>` 内实体已反转义为字面字符
   （`&lt;script&gt;` → 文本 `<script>`），highlight.js 把它当**代码源 token 化**，从不当 HTML 解析。
2. **产物只是转义 span。** highlight.js v11 产物形如 `<span class="hljs-keyword">…转义文本…</span>`；class 取自其**固定
   词表**（`hljs-*`）。v11 **移除了 HTML 透传**：任何未转义 HTML 被**剥除并 console 告警**——即 `<script>` 在产物里
   只会是转义文本，绝无活标签。**信任铰链 = 钉版 + v11 的「输入须转义、输出全转义」契约**；喂转义 `textContent`、
   **每块只高亮一次**（`.hljs` 跳重）即合规（重复高亮才会触发那条告警，§3.1 已跳）。

**② 数学/化学（KaTeX + `trust:false`）：**
1. **输入是文本、非 HTML。** auto-render 遍历**文本节点**拿到 TeX 串（浏览器已反转义），交 `katex.render` **解析 TeX、非
   HTML**。即便串里写 `<img onerror=…>`，它只是 TeX 解析器眼里一段会报错的文本。
2. **`trust:false`（铰链，且为 KaTeX 默认）。** KaTeX 文档明载：`trust:false`（默认）**禁用 `\href`/`\url`/
   `\includegraphics` 及全部 `\html*`**（`\htmlClass`/`\htmlId`/`\htmlData`/`\htmlStyle`），令其渲成**错误色**而非生效。
   故恶意公式里企图注入 `<a href=javascript:>`/HTML 属性的命令在产物里**被禁、不存在**。配置**硬编码 `trust:false`**、
   前端无任何放宽旋钮（用户/页面都改不动）——这是「无放宽旋钮、硬编码」的等价 strict 闸。**与 MathJax 路线的本质差异**：
   KaTeX 安全是**默认即开的单旗白名单**（结构上不需枚举/移除危险包），而 MathJax `-full` 需在「默认激活全部扩展」上做
   减法黑名单（语义未文档保证、完备性难担保），故取 KaTeX（决策P4.14-2）。
3. **隔离铰链：每容器一份默认 `macros`，杜绝**跨容器**宏污染（页内局部宏 benign、不专守）。** KaTeX auto-render 的
   `macros` 默认是**每次 `renderMathInElement` 调用一份**的空对象，并在该次调用内传给多公式（官方文档明载）。我们
   `render(container, {...KATEX_OPTS})` 不传共享 `macros` → **跨容器**（页/气泡/预览各一次调用）各自一份、`\gdef`/`\newcommand`
   **不跨页/跨气泡泄漏**——这是「页面自洽」要守的、也是评审关切的污染面。**容器内**（同一页/气泡）多公式确会共用那份 `{}`、
   页内 `\gdef` 可生效，但属**页内局部 preamble**：受 `maxExpand`（默认 1000）限不能膨胀爆炸、`trust:false` 下注入不了 HTML，
   故 benign，**不纳入须禁绝之列**（要全禁须绕开 auto-render，代码/攻击面更大、收益仅挡无害页内宏，不值）。这虽非典型
   「XSS」面，但**跨容器**那一半属同源单页「一处内容影响他处呈现」的污染，纳入安全/隔离边界守。隔离靠 auto-render 的「每
   调用一份默认 macros」+ 我们每次展开新选项，无需状态重置机制。
4. **产物是受限 HTML+CSS（spans/MathML）、注入当前页 DOM。** `trust:false` 下 KaTeX 产物仅其内置构建器生成的
   `<span class="…">` 与 MathML，**无活链接、无任意 HTML 属性、无 `<script>`/事件处理器**。auto-render 直接替换文本节点
   （非 `innerHTML` 任意串）。**信任铰链 = `trust:false` + 钉版 + 失败保留源码**。
5. **`ignoredTags` 含 `pre/code/script/style`** → 不在代码/脚本里排版（代码块内 `$` 保字面）。

**③ `_EscapeHtmlExtension` 全程不破**（同 P4.13-4）：服务端仍把一切原始 HTML 转义成文本（`render.py` 未动一行）。
两个渲染器读的都是**转义后的文本/textContent**，**从不绕过转义闸**。故服务端 XSS 纵深防御**字节稳定、全程有效**。

**④ 失败封闭。** 任一渲染器加载失败 / 单元异常 → 退回保留源（代码纯文本 / 数学字面文本 / KaTeX 错误色源码），
**任何异常路径都不产生未消毒注入、不空白**。

**残余风险与处置（保守表述，不承诺产物绝对可信）：** KaTeX/highlight.js 历史均有过 XSS CVE（KaTeX 多发于 `trust:true`
或旧版本，highlight 多发于 HTML 透传 / 旧版本）。**本期不把二者输出当无条件可信**——边界明确收在这组闸：① **钉版本**到
无已知绕过的发行版、`vendor/README.md` 记版本 + 各文件 SHA256，升级走显式 PR 复核；② 配置**硬编码**（KaTeX `trust:false`
+ 不共享 `macros`；highlight 喂转义 textContent + 单次高亮），前端无放宽旋钮；③ **失败保留源码**。即本期边界 = **KaTeX
`trust:false`（默认安全）+ 不共享 macros（隔离）+ SVG/受限 HTML 产物 + highlight v11 转义契约 + 钉版 + 失败保留源码**。

## 6. 决策记录

- **\[决策P4.14-1] 两个运行时（KaTeX 家族 + highlight.js）vendored（非 CDN）+ 懒加载，归类为「内容渲染器」——延续
  决策P4.13-1 的有界破例。** 二者与已 vendored 的 mermaid、服务端 `markdown` **同位阶**：都是「把一种标记渲染成富呈现」
  的内容渲染器，非 UI 框架。守住决策P4-3 三条实质底线：① **无 CDN**——随 wheel 打包、`127.0.0.1` 自发、离线可用
  （KaTeX 字体经 `@font-face` 同源拉取，无运行时 JS 动态加载）；② **无构建**——KaTeX/highlight 均用其 dist 预构建文件，
  经注入 `<script>`/`<link>` 加载读 `window.renderMathInElement`/`window.hljs`，无 npm/打包链；③ **无数学分隔符候选/无
  可高亮块的页零加载**——懒加载，`typesetMath`/`highlightCode` 先早退才付加载代价（数学探测要求**成对（含闭合）**分隔符，故
  孤立开头符——单个 `$5`/裸 `$$`/裸 `\(`/裸 `\[`——皆不触发；惟成对分隔符若现于代码块/巧合配对仍付一次无谓加载（auto-render
  随后跳 `pre/code`、不误排版），P3 的启发式残留，决策P4.14-3）。**KaTeX 非单文件**（带 css +
  ~20 个字体文件），但「单文件」本是「离线自洽」的**手段**而非目的——KaTeX 带 vendored 字体仍完全离线自洽、无构建、无
  CDN，故 ethos 在**实质**上守住（只是文件数多）。总资产 ~0.8MB（KaTeX ~0.5MB + highlight ~0.15MB + 字体），比 mermaid
  (3.3MB)/`logo.png`(1.6MB) 还小；`vendor/README.md` 钉版 + 各文件 SHA256。**退路同 P4.13**：若不接受第三方运行时，保持
  现状字面源码（决策P4.14-8 的 honest 回退本就常驻）。
- **\[决策P4.14-2] 数学/化学取 KaTeX 而否 MathJax——因 KaTeX 的安全铰链是「默认即安全、单旗、文档明载」，而 MathJax
  路线的安全压在「脆弱的全量包减法黑名单」上。** 两者都能离线含 mhchem（MathJax `tex-svg-full` 单文件更省文件数）。但
  **安全是本相位实质**，故铰链可靠性压倒文件数：① **KaTeX**——`trust:false` 是**默认值**，文档明确其禁用 `\href`/`\url`/
  `\includegraphics`/`\html*`；**跨容器**宏隔离靠「每容器一份默认 `macros`」（auto-render 默认行为，页内局部宏 benign）。
  两条都是**默认安全 + 白名单式**，可硬编码、可运行时验证、无须枚举危险面。② **MathJax `tex-svg-full`**——默认激活除 physics/colorv2 外**全部**扩展（含 `html`/`href`/
  `require`/`autoload`/`newcommand`/**`setoptions`**——后者允许 TeX 源运行时改 MathJax 选项），要安全须做**减法黑名单**
  `tex.packages:{'[-]':[…]}`；而纯数组 `packages:[…]` 的「替换 vs 合并」语义 MathJax 文档**未明确保证**、`[-]` 黑名单的
  **完备性难担保**（漏一个危险包即暴露，且 `\href` 是否属可移除包亦不确定）。把安全压在一个全量包的减法黑名单上是**脆弱
  地基**——这是评审复核 MathJax 路线时点出的关键风险。取 KaTeX 即结构上消除整类不确定。**代价**：KaTeX 须 vendored
  `katex.min.css` + ~20 个 `.woff2` 字体（文件数多、css 相对路径须与 fonts 同级，决策P4.14-5），但总字节反更小、仍离线
  无构建。**这是本相位唯一需权衡的运行时选型**（同 P4.13-1 的 ESM↔UMD 取舍）。
- **\[决策P4.14-3] 数学载体 = 前端 DOM 文本走查（KaTeX auto-render），服务端零改；不引服务端 math 扩展。** 数学无围栏
  载体（`$…$` 非围栏块），两条路：①（取）前端 `renderMathInElement([container])` 遍历文本节点按分隔符排版——服务端零改、
  `render.py` 字节稳定（P4.13 立的硬约束不破）；②（否）服务端 `pymdownx.arithmatex` 把 `$$…$$` 包成 `<span>` 占位——会
  **改 `render.py`、破字节稳定、加服务端依赖、扩转义攻击面**。取 ①。代价是 `$…$` 与字面美元号歧义残留，以优先用无歧义
  `\(…\)`/`\[…\]` 缓解；**残留是 presentation-only、源码始终 honest**（CLI/纯文本回退原样可见）。
  **不为消歧改服务端。** 代码块则**有**围栏载体（`code.language-X`、与 mermaid 同），故代码高亮无此张力、机械复刻。
- **\[决策P4.14-4] 「内容→客户端 HTML」本期安全/隔离边界 = KaTeX `trust:false`（默认安全、硬编码、无放宽旋钮）+ 每容器
  一份默认 `macros`（**跨容器**宏隔离）+ highlight v11 转义契约 + 钉版 + 失败保留源码；不承诺产物绝对可信。** 见 §4。
  `trust:false` 禁 `\href`/`\url`/`\includegraphics`/`\html*`（默认即此，显式写明杜绝被误改）；`render(container, {...KATEX_OPTS})`
  不传共享 `macros` → auto-render 每容器一份默认 `{}`、`\gdef`/`\newcommand` **不跨页/跨气泡泄漏**（**容器内**页内 `\gdef`
  可生效，属 benign 页内局部、受 `maxExpand` 限、`trust:false` 下注不进 HTML，故不专守——本边界守的是**跨容器**那一半）；
  highlight v11 默认安全（剥未转义 HTML + 全转义输出），喂转义 textContent + 单次高亮即合规。配置硬编码、无放宽旋钮；任何
  异常退回保留源、零未消毒注入；残余 CVE 以钉版 + 显式升级 PR 复核处置。**验证策略**（评审强化）：源码 `grep` 断言
  （`trust:false` 在、未把共享 `macros` 对象塞进 `KATEX_OPTS`）是**必要非充分**的廉价守门；真正生效与否的**运行时验证**
  （`\href`/`\htmlData` 确不生效、`\gdef` **不跨容器**泄漏）走 §7 浏览器手测——与 P4.13「渲染交互走手测、Python 层只锁服务端
  契约」一致。**全局 CSP 收紧是独立加固、不扩本期范围**（同 P4.13-3；附记 KaTeX 注入内联样式，将来补 CSP 需放行 `style-src`）。
- **\[决策P4.14-5] 服务端零改、转义姿态不破——代码靠 `fenced_code` 的 `code.language-X`、数学靠文本穿过 + 前端读文本
  节点；KaTeX css 与 `fonts/` 须同级 vendored。** `render.py` 不动一行：代码块已是转义文本载体（同 mermaid）；数学是普通

  关原始 HTML 透传**全程有效**，`render_markdown` 其余调用方零回归。不在服务端造 math `<span>` 占位（那要改转义姿态、扩

- **\[决策P4.14-6] 代码主题写进 `app.css`（不 vendored 第三方 CSS）、KaTeX 用其必需 vendored css + 最小颜色/溢出补丁；

  落地时另加（同 P4.13-5，避免过度设计）。
- **\[决策P4.14-7] 优雅降级常驻、且不静默截断。** 代码：运行时加载失败 / 单块异常 / **语言未在 common 构建注册** →
  保留**可读纯文本代码**（不猜语言、不报错、不动态拉子包破离线）；`vendor/README.md` 记覆盖语言集，未覆盖者可见地保留
  纯文本（**非静默**）。数学：加载失败 → 字面文本留存；语法错 / 被禁命令 → KaTeX `throwOnError:false`/`trust:false` 退回
  **错误色源码**（可见）。markdown 仍是唯一事实来源（DESIGN 原则 1）：渲染是**叠加增强**，剥掉它页面退回 honest 源码、

12. **宏不**跨容器**污染（安全运行时验证，决策P4.14-4）**：A 容器（某页/某气泡）写 `$\gdef\foo{X}\foo$`、切到 B 容器（另一页/另一气泡）写 `$\foo$` → B 的 `\foo` **渲成错误色/未定义**，**不**因 A 而生效——证每容器一份默认 `macros`、无**跨容器**泄漏。（**预期对照**：同一页内先 `$\gdef\foo{X}$` 后 `$\foo$` **会**生效，是 benign 页内局部 preamble，非缺陷，决策P4.14-4。）
13. **CLI**：`guanlan query` 命中该页 → 回**字面公式/代码源码**（决策P4.14-8，不渲染、不回归）。
