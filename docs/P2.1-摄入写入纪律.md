# P2.1 摄入写入纪律（既有页只增不毁 · 源不回退门禁 + 正文骤缩警告 + 合并纪律 · 零 LLM 门禁侧 · 无新命令/无新退出码）

> **定位（来自一次反向评审）**：本阶段不是「给 guanlan 加 llm_wiki 的功能」。对照 `../llm-wiki/llm_wiki`（一个成熟的
> Tauri 桌面应用）逐子系统评审后，绝大多数「可借鉴项」都因**把判断从 Agent 搬进工具**（四信号关联度、图谱洞察、
> 模糊级联删除……）而被否决——它们违反 guanlan 第一架构铁律「`guanlan/` 不携带 business intelligence、智能在 Agent」。
> **只有两条幸存**，且都不是「加算法」而是「加确定性验证 + 防御纪律」：
>
> 1. **re-ingest 合并不覆盖**（llm_wiki `page-merge.ts`：union `sources[]`、锁身份字段）——guanlan 当前完全靠 Agent
>    自觉「读既有页 → 合并」，无任何兜底。若某次 ingest 只看着一个源就把既有实体页**重写**成 `sources: [本次源]`，
>    旧的 `sources` slug 被**静默丢弃**——provenance 永久损毁，而现有门禁（`check` 只看当前态、无 before）**看不见**。
> 2. **正文不被改写**（llm_wiki `BODY_SHRINK_THRESHOLD=0.7` + enrich-links「LLM 提建议、代码确定性落地、绝不让 LLM
>    重写整页」）——同源问题的另一面：既有页正文被本次 ingest **整段覆盖重写**而非增补合并，旧论断丢失。
>
> 两条本质同根：**ingest 写入既有页时只该「增补」、不该「毁掉」**。P2.1 把它落成 guanlan-shaped 的形态——**不**抄
> llm_wiki「代码替 Agent 做 merge / 代码替 Agent 做 substitution」（那又把写入逻辑搬进工具），而是：**引擎侧**在
> `SKILL.md` 写清合并纪律（正向引导，智能仍归 Agent），**门禁侧**加确定性兜底**检测**违反并回喂有界自愈让 Agent
> 自己修（兜底，零 LLM）。这正是仓内 `lint↔heal`、`check↔gate` 一贯的「检测/引导 + 兜底」结构（决策P2.1-9）。
>
> **为何是 P2.1（P2 的首个半阶段），不是 P3.x**：源不回退/骤缩检测需要**写操作前的 before 状态**——而 `check`（P2）
> 与 `lint`/`health`（P3）都是**无状态**的（只扫当前 `wiki/`，无 before），结构上**不可能**检测「回退」。唯一持有
> before 快照的是写门禁 `run_guarded_write_result`（`gate.py:327` 已对 `raw/` 取 before）。故这是**写时门禁**的事
> （P2 关注域：ingest + check 门禁），不是事后维护（P3）。判别尺子：**写时 → P2；事后 → P3**（决策P2.1-1）。
>
> **作用范围（关键，见决策P2.1-10）**：P2.1 只覆盖 **ingest 与 `query --backfill`**（经 `run_guarded_write` 薄壳，
> `page_guard` 默认 True）。**heal 不在覆盖面内**——它直调 `run_guarded_write_result`（`page_guard` 默认 False），且其
> 自身写集审计（`heal.py:299` `_writeset`）已禁既有页正文/非别名 frontmatter 改动，源回退在 heal 契约下本不可能，
> 再加 P2.1 既冗余又会改其 `--json`/退出码。开关键落在 `page_guard` 缺省值（薄壳 True / 核心 False），故所有现有
> call site（`ingest.py:72`/`query.py:80`/`heal.py:419`）**一行不改**。
>
> **零新增**：无新命令、无新退出码、无新依赖、无新 SSE 契约。`sources.dropped` 复用既有 `EXIT_CHECK_FAILED` 与既有
> 有界自愈环；`body.shrank` 复用既有警告通道（与 `wikilink.broken` 同级，非阻断）。**机器 JSON 契约亦不变**：heal
> （`page_guard` 关、GateResult 不含新 kind）`--json` 逐字节不变；ingest/backfill 无暴露门禁 kind 的机器 JSON（门禁
> 结论经 `report_outcome` 走人读 stdout/stderr，web backfill 的 `result` 仍为 null、回执在 `job.output`），新 kind
> 仅现于**人读**门禁报告（决策P2.1-10）。文中 **\[决策P2.1-N]** 集中记录在 §4。

## 0. 定位：补的是「既有页被 ingest 静默腐蚀」这个洞

### 0.1 现状与洞

guanlan ingest 走 Agentao + `SKILL.md`，Agent 持文件工具**读既有页 → 改写**。`SKILL.md` step 3 已要求「扫 index.md +
现有页 aliases，是否已有该实体的变体页？是 → 更新既有页」「实质修改既有页务必把 `last_updated` 改成当天」——但
**「更新」具体是「合并」还是「覆盖」，全靠 Agent 自觉，无任何确定性兜底**。两类静默腐蚀因此可逃逸出库：

- **源回退（provenance 损毁）**：页 X 原 `sources: [a, b]`（由资料 a、b 共同贡献）。本次 re-ingest 资料 a，Agent
  只看着 a 重写 X、产出 `sources: [a]`——slug `b` 被静默丢弃。现有 `check._check_sources`（`check.py:105`）只校验
  「`sources` 里每个 slug 都有对应 `wiki/sources/<slug>.md`」，**`[a]` 完全合法**，门禁放行。**provenance 永久损毁，
  且不可恢复**（`raw/` 未变，但知识层与来源的链断了）。
- **正文覆盖（旧论断丢失）**：页 X 原是一张富页（多段论断、`## ⚠️ 矛盾与存疑` 节）。本次 ingest 把它整段重写成只
  反映本次源的薄页，旧论断/旧矛盾标记丢失。`sources` 可能还在（没触发上一条），但正文已被洗掉。

llm_wiki 之所以需要 `page-merge.ts` 的**确定性 union + 锁字段 + 70% 缩水守卫**，正因为它的 ingest 是模板化两步 LLM
调用、未必可靠 read-modify-write。guanlan 的 Agent 本该更靠谱，但「本该」不是「确定性保证」——guanlan 在别处（`raw/`
不可变、frontmatter 合规）从不靠「Agent 自觉」，这里也不该是例外。

### 0.2 为何门禁侧、为何不是 `check`/`lint`

「页 X **回退**了源 b」是一个**两态比较**命题（before 有 b、after 无 b），**无状态扫描器原理上测不了**：

- `check`（P2，`check.py`）：写门禁的收尾校验，只看**当前** `wiki/`，无 before。
- `lint`/`health`（P3）：事后审计，同样无状态。

唯一持有「写操作前快照」的是 `gate.run_guarded_write_result`——它已在写前取 `before = snapshot_raw(root)`
（`gate.py:327`）作 `raw/` 完整性基线。P2.1 在**同一处**再取一份 `before` 的**每页 frontmatter 快照**（源 slug 集 +
正文长度），写后比对。**这决定了 P2.1 必须落在 gate.py、属 P2 写时门禁**（决策P2.1-1）。

### 0.3 已有的同构先例：heal 的写集审计

heal（P3.2）已有一套**确定性写集审计**（`heal.py:224` `_snapshot_wiki` + `heal.py:299` `_writeset` +
`heal.py:269` `_is_safe_alias_collection`）：写前/写后对 `wiki/` 取每页快照，diff 出「删页 / 改既有页正文 / 建错
目录」标为 `unexpected_write`。P2.1 是同一思路在 **ingest 写入口**的应用，但**约束面必须更窄**（决策P2.1-7）：

| | heal（窄写入面） | ingest（宽写入面） |
|---|---|---|
| 改既有页正文 | **禁**（除安全别名收编）→ `unexpected_write` | **合法**（合并即改正文）——不可照搬 heal 的「改正文即越界」 |
| 删页 | 禁 | 本阶段不裁决（决策P2.1-8 边界） |
| 可检测的硬不变量 | 写集全审计 | **仅「既有页源不回退」**（窄、近无歧义） |

故 P2.1 **不复用、不重构 heal 的 `_snapshot_wiki`**——heal 禁正文改、ingest 允正文改，语义相反，强行共用反增耦合
与 heal 回归面（决策P2.1-7）。P2.1 在 `gate.py` 自带一份**更轻**的快照（只取 `sources` 集 + `body` 长度），复用
`pages.py` 的 `iter_pages`/`load_page` 容错原语。

## 1. 契约（无新命令 / 无新退出码 / 仅新增两个门禁信号 + SKILL 纪律）

P2.1 **不引入任何命令或参数**。它只在 **ingest / `query --backfill`** 两个写入口（经 `run_guarded_write` 薄壳，
`page_guard` 默认 True）里新增两个确定性信号，并收紧 `SKILL.md` 的合并纪律。**heal 不在覆盖面内**（直调
`run_guarded_write_result`、`page_guard` 默认 False，决策P2.1-10）：

| 信号 | 严重度 | 触发 | 处置 |
|---|---|---|---|
| **`sources.dropped`** | **阻断**（同 `sources.unresolved` 级）| 写前存在、写后仍存在的某页，其 frontmatter `sources` **丢失**了写前有过的 slug | 进 `check_failed` → **有界自愈**回喂 Agent 并回（最多 2 轮，沿用既有环），未修复 → `EXIT_CHECK_FAILED` |
| **`body.shrank`** | **警告**（同 `wikilink.broken` 级，非阻断）| 写前/写后均存在的某页，正文长度从 `L₀`（≥ `SHRINK_FLOOR`）缩到 `< SHRINK_RATIO × L₀` | 仅报告（与断链警告同通道），**不阻断、不自愈、不影响退出码** |

- **退出码不变**（决策P2.1-5）：`sources.dropped` 复用 `EXIT_CHECK_FAILED(3)`（它本就是 `check_failed` 一类新阻断性
  违规）；`body.shrank` 是警告、不影响退出码。`raw_mutated`/`agent_error` 路径逐字不动。
- **覆盖面 = ingest/backfill（不含 heal）**：仅 `run_guarded_write`（`page_guard` 默认开）的写操作才取每页快照；
  heal 经 `run_guarded_write_result`（默认关）**不**取、逐字节不变（决策P2.1-10）。独立 `guanlan check`/`lint`/
  `health` 也完全不变（它们无状态、测不了回退）。
- **`query --backfill`**：走 `run_guarded_write`（`page_guard` 开）。其**正常路径只新建** `syntheses/<slug>.md`
  （synthesis 是时点快照、不级联改写既有页，见 `SKILL.md` query 步 3），故通常不触发 `sources.dropped`；但若 Agent
  在 backfill 期间**越界改了某既有 entity/concept/source 页并丢源**，门禁仍会触发并自愈——这正是期望行为（决策P2.1-8）。

## 2. 模块与归口

### 2.1 `gate.py`：新增每页快照 + `enforce` 接入（核心，零 LLM）

```python
# gate.py 新增（与 snapshot_raw/diff_raw 并列）

# 正文骤缩告警阈值（决策P2.1-3）：比 llm_wiki 的 0.7 更宽松以压噪——只抓「腰斩级」骤缩，
# 且 before 正文须达 SHRINK_FLOOR 才判（桩页→桩页的微缩不触发）。
SHRINK_RATIO = 0.5
SHRINK_FLOOR = 200   # 字符

@dataclass(frozen=True)
class PageMetaFingerprint:
    """写操作前/后单页的轻量指纹，仅供源不回退/骤缩比对（不参与 raw 完整性，那是 snapshot_raw 的事）。"""
    sources: frozenset[str] | None  # 可信解析的 slug 集；frontmatter 坏/缺、或 sources 缺/类型非法 → None
    body_len: int                    # split_frontmatter 后正文字符数（总可算，body.shrank 用）

def snapshot_page_meta(root: Path) -> dict[str, PageMetaFingerprint]:
    """写操作前/后的每页指纹（{wiki/... posix 路径: PageMetaFingerprint}）。

    - 只扫 content 页：复用 `pages.iter_pages`（排除 index/log/overview，与 check/graph 同口径）。
    - 容错解析：复用 `pages.load_page`，**绝不抛**（决策P3-8）。`body_len` 总可算（坏块时 split_frontmatter 把全文当 body）。
    - **`sources` 仅在「可信」时记 frozenset，否则 None（决策P2.1-11，去重 check）**：
        meta=None（坏/缺 frontmatter 块）、或无 `sources` 键、或 `sources` 非「字符串列表」 → **None**；
        `sources` 是合法字符串列表（**含空 `[]`**）→ `frozenset(slug)`。
      这三类「不可信」恰好都已是 `check` 的**阻断**违规（frontmatter.unparsable / missing_key / bad_type），
      记 None 让 `_check_source_regression` **跳过该页 dropped 检测**，frontmatter 错误单独留给 check、不重复记账。
    """

def _check_source_regression(
    before: dict[str, PageMetaFingerprint], after: dict[str, PageMetaFingerprint],
) -> tuple[list[Violation], list[Violation]]:
    """对 before∩after（写前写后**都在**的页）逐页比对，返回 (blocking, warnings)。

    - blocking（`sources.dropped`）：**仅当 `before.sources` 与 `after.sources` 都非 None**（两侧均可信解析）才比；
      `before.sources - after.sources` 非空 → 每丢一 slug 一条
      `Violation(page, "sources.dropped", f"既有页丢失原有来源 {slug!r}（ingest 只增不减来源，须并回）")`。
      任一侧 None → **跳过**（after 坏 frontmatter 已被 check 阻断；before 已坏则无可信基线，决策P2.1-11）。
      注意空 `[]` 是**可信的 frozenset()**：`[a,b]`→`[]` 仍判 a、b 两条 dropped（真回退，区别于 None-跳过）。
    - warning（`body.shrank`）：`before.body_len >= SHRINK_FLOOR and after.body_len < SHRINK_RATIO * before.body_len`
      → 一条 `Violation(page, "body.shrank", …)`（与 sources 可信度**无关**，body_len 总可算）。
    输出按 (page, slug) 稳定排序，确定可重建。**只比 before∩after**：新页/删页都不在此判（决策P2.1-8）。
    """
```

`enforce` / `enforce_write_result` 接入（最小改动，新增可选 `page_before` 参数，缺省 None 时**逐字节保持原行为**——
独立 `enforce` 调用、heal 等不传则语义不变）：

```python
def enforce(root, snapshot_before, *, page_before=None, baseline=frozenset()) -> GateResult:
    raw_changes = diff_raw(snapshot_before, snapshot_raw(root))
    if raw_changes:
        return GateResult.raw_mutated(raw_changes)
    check = run_check(Path(root) / "wiki")
    blocking, warnings = partition_violations(check.violations)
    if page_before is not None:                                   # P2.1
        dropped, shrank = _check_source_regression(page_before, snapshot_page_meta(root))
        blocking = blocking + dropped                             # 进 check_failed → 自愈
        warnings = warnings + shrank                              # 进警告通道（非阻断）
    new_blocking = [v for v in blocking if _vkey(v) not in baseline]
    if new_blocking:
        return GateResult.check_failed(new_blocking, warnings=warnings)
    return GateResult.passed(warnings=warnings)
```

- **`sources.dropped` 天然免基线豁免**：`baseline` 来自 `check_baseline`（只跑 `run_check`，`gate.py:77`），其中
  **不含** `sources.dropped`（它非 `run_check` 产物、是 gate 写前/写后 diff 算出的）。故 `_vkey` 基线过滤**永不**误
  压制它（它必是「本次新引入」）——无需特殊处理，但须在测试钉死（§8）。
- **失败路径不动**：`enforce_write_result` 在 `run_result.ok is False`（agent 失败）时仍只判 `raw/`、不跑 check
  （半成品 wiki 无意义，`gate.py:277`）——故源不回退**只在成功路径**（`enforce` 内）计算。`page_before` 仅传到成功
  路径的 `enforce` 调用。

### 2.2 `run_guarded_write_result`：写前多取一份每页快照

```python
# 开关键在 page_guard 缺省值（作用范围一处可读，决策P2.1-10）：核心默认 False → heal（直调本核心）逐字节不变；
# 薄壳 run_guarded_write 默认 True → 其唯一两个调用方 ingest/backfill 自动守护，call site 不改。
def run_guarded_write_result(root, prompt, *, model=None, runner=None,
                             max_repair=MAX_REPAIR_ATTEMPTS, page_guard=False):
    before = snapshot_raw(root)
    page_before = snapshot_page_meta(root) if page_guard else None   # P2.1：仅守护时取写前每页指纹
    baseline = check_baseline(root)
    first_result = run_agent_task(...)
    gate = enforce_write_result(root, before, first_result, page_before=page_before, baseline=baseline)
    while gate.kind == "check_failed" and attempt < max_repair:
        ...
        last_result = run_agent_task(REPAIR_PROMPT.format(...), ...)
        gate = enforce_write_result(root, before, last_result, page_before=page_before, baseline=baseline)
    ...

def run_guarded_write(root, prompt, *, model=None, runner=None,
                      max_repair=MAX_REPAIR_ATTEMPTS, page_guard=True):   # 薄壳默认开（ingest/backfill）
    result = run_guarded_write_result(root, prompt, model=model, runner=runner,
                                      max_repair=max_repair, page_guard=page_guard)
    report_outcome(result.gate, result.run_result)
    return result.exit_code
```

- `page_guard=False`（heal / 未开守护）时 `page_before=None`，`enforce` 逐字节走原路径。
- `page_before` 在写前取一次、贯穿所有自愈轮（自愈轮也比对**最初的** before——自愈本身不该再丢源；若自愈轮仍漏并
  某 slug，门禁继续追究，符合预期）。`enforce_write_result` 透传 `page_before` 给其**成功路径**的 `enforce`
  （agent 失败路径不跑 check、不算源回退，`gate.py:277` 语义不动）。
- **call site 全不改**：`ingest.py:72`/`query.py:80` 调薄壳（默认开）、`heal.py:419` 调核心（默认关）。heal 若将来想
  守护可显式传 `page_guard=True`，但本阶段不开（决策P2.1-10）。

`gate.py:45` 的 `REPAIR_PROMPT` 现只覆盖 frontmatter/sources/aliases 修复。增补一句（仍属「只修门禁列出的、不新增

### 2.4 `skills/guanlan-wiki/SKILL.md`：合并纪律（引擎侧正向引导，零代码）

ingest 工作流 step 3「更新既有页」处补**合并纪律**（正向引导，门禁是兜底；llm_wiki「代码替 Agent merge」**不抄**，
决策P2.1-4）：

> **更新既有页 = 合并、不是覆盖**：
> - `sources`/`tags`/`aliases` 取**并集**（保留原有，追加本次新增）——**绝不**因本次只看一个源就把 `sources` 写成
>   `[本次源]`，那会丢掉旧来源（门禁 `sources.dropped` 会阻断 + 回喂你自愈）。
> - 正文**增补融合、不整段重写**：保留原有论断与 `## ⚠️ 矛盾与存疑`，新信息追加或就地融入。把一张富页洗成只反映
>   本次源的薄页是**腐蚀**（门禁 `body.shrank` 会警告，请自查是否误删）。
> - `type` 不随手改（既有页的分类是稳定身份）。

并在「写工作流收尾（共用）」的「会让本轮失败的硬错误（速查）」补一条阻断项 + 一条警告项：

> - **既有页丢失原有 `sources` slug**（覆盖而非合并）→ `sources.dropped` 阻断 + 自愈：ingest 只增不减来源。
> - （警告非阻断）**既有页正文骤缩** → `body.shrank`：疑似覆盖重写，自查是否误删旧内容（确属有意精简则放行）。

## 3. 行为细则

### 3.1 源不回退（`sources.dropped`，阻断 + 自愈，决策P2.1-2）

- **不变量**：对写前存在、写后仍存在（同相对路径）的页 P，`sources_after(P) ⊇ sources_before(P)`。任一写前有过的
  slug 在写后消失 → `sources.dropped`（每丢一个 slug 一条，便于自愈精确定位）。
- **为何阻断而非警告**：provenance 静默丢失正是写门禁存在要防的那类硬错误——与既有 `sources.unresolved`（来源指向
  不存在的 source 页）同级、互补：unresolved 防「指向虚空」，dropped 防「来源蒸发」。它**确定、近无歧义**：guanlan
  ingest 的既定行为是「更新既有页」（`SKILL.md` step 3），**从无**「合法删既有页来源」的工作流（guanlan 无删除路径）。
- **自愈闭环**：进 `check_failed` → 既有有界自愈环（`gate.py:340`）回喂 Agent，按 §2.3 增补的指令并回 slug。修复 =
  一次 frontmatter 编辑，最省事；2 轮内未修 → `EXIT_CHECK_FAILED`，改动留盘待人工。
- **智能仍归 Agent**：门禁**只检测**「源没了」，**不**替 Agent 写回（不照搬 llm_wiki「代码 union 落盘」）——并回哪个
  slug、正文如何因此调整，由 Agent 在自愈轮判断。工具守不变量，Agent 做合并（决策P2.1-4）。

### 3.2 正文骤缩（`body.shrank`，警告非阻断，决策P2.1-3）

- ingest **合法地**改既有页正文（合并即改），故正文变化**不能**当违规（与 heal 不同，heal 禁正文改）。但「腰斩级
  骤缩」是覆盖重写的强信号：`before.body_len ≥ SHRINK_FLOOR(200) 且 after.body_len < 0.5 × before.body_len`。
- **故只报警告**（非阻断、不自愈、不影响退出码），与 `wikilink.broken` 同通道（`GateResult.warnings`），经
  `report_outcome` 的警告汇总行提示（§5）。fuzzy 信号宁可漏不可误杀：合法精简（删冗余、合并重复段）也可能骤缩，
  由人/Agent 自判，工具只点出疑点——这与 guanlan「断链作警告、让自然增长消解」一脉（决策P2.1-3）。
- **阈值取舍**：0.5（比 llm_wiki 0.7 更宽）+ 200 字底线，专抓显著腐蚀、压低噪声；常数具名（`SHRINK_RATIO`/
  `SHRINK_FLOOR`），便于按真实库调。

### 3.3 合并纪律是引擎侧正向引导，门禁是确定性兜底（决策P2.1-9）

两侧配对，缺一不可，且**职责不混**：

- **`SKILL.md`（引擎侧，LLM）**：教 Agent「合并不覆盖、并集来源、增补不重写」——正向、在写之前生效，避免大多数腐蚀
  在第一轮就发生。
- **`gate.py`（门禁侧，零 LLM）**：写后确定性**检测**漏网的腐蚀，阻断性的回喂自愈、警告性的报告。

这正是仓内既有结构的复刻：`lint.missing_entity`（检测）↔ `heal`（修复引导）、`check`（门禁）↔ `SKILL.md`（写作引导）。
P2.1 **不**把合并逻辑写进 `gate.py`（那会让工具携带「如何合并页面」的 business intelligence，违反第一铁律）——
gate 只判「源没了 / 正文腰斩」这种**纯结构事实**。

## 4. 决策（沿用仓库 `决策PX-N` 体例）

- **决策P2.1-1（落 gate.py、属 P2 写时门禁；不在 check/lint）**：源不回退是**两态比较**，无状态的 `check`（P2）/
  `lint`·`health`（P3）原理上测不了。唯一持写前 before 快照的是 `run_guarded_write_result`（`gate.py:327`）。判别
  尺子「写时→P2 / 事后→P3」把它钉在 P2，编号为 **P2.1（P2 首个半阶段）**。
- **决策P2.1-2（`sources.dropped` 阻断 + 自愈，per-page、present-before-and-after）**：provenance 静默丢失是门禁该
  防的硬错误，与 `sources.unresolved` 同级互补。判据 = 写前写后都在的页其 `sources` 集**只增不减**；丢失即阻断、进
  既有有界自愈环并回。guanlan 无「合法删既有页来源」工作流，故此不变量近无歧义。
- **决策P2.1-3（`body.shrank` 警告非阻断，阈值 0.5 + 200 字底）**：ingest 合法改正文，故正文变化不可当违规；只把
  「腰斩级骤缩」当**疑点**报警告（与断链警告同通道、同哲学：宁漏不误杀，人/Agent 自判）。阈值比 llm_wiki 的 0.7 更宽
  以压噪，常数具名可调。
- **决策P2.1-4（工具只验证不变量、不替 Agent 做合并；明确不抄 llm_wiki「代码 merge / 代码 substitution」）**：
  llm_wiki 用确定性代码 union frontmatter、用代码落地 LLM 的链接 substitution——那是因为它的 ingest 是模板化 LLM 调用、
  不可靠。guanlan 的 ingest 由 Agent 持文件工具自主读写，**合并是 Agent 的职责**；把合并搬进 `gate.py` 会让工具携带
  business intelligence，破第一铁律。故 P2.1 只在门禁**检测**违反、回喂 Agent 自修；正向引导放 `SKILL.md`。这是
  反向评审的核心结论落地点。
- **决策P2.1-5（无新退出码）**：`sources.dropped` 复用 `EXIT_CHECK_FAILED`（属 `check_failed` 一类新阻断违规）；
  `body.shrank` 警告不影响退出码。`raw_mutated`/`agent_error` 路径逐字不动。沿用半阶段不引新码纪律。
- **决策P2.1-6（type 漂移本阶段显式不做）**：曾考虑「既有页 `type` 写后不得变」。否决：①设为阻断会**过约束**合法
  重分类（entity→concept）并在自愈环里死循环；②设为警告则价值边际、徒增噪声。guanlan 既有必备键无 `created`，可锁的
  身份字段实只 `type`，分量不足以单列机制。留待真有需要时再议（体现反向评审「不往工具塞策略」纪律）。
- **决策P2.1-7（不复用、不重构 heal 的 `_snapshot_wiki`；gate 自带轻量快照）**：heal 禁既有页正文改（`unexpected_write`）、
  ingest 允正文改，两者写入面语义相反，强行共用一份快照/审计反增耦合与 heal 回归面。gate 自带**更轻**的
  `snapshot_page_meta`（只取 `sources` 集 + `body` 长度），复用 `pages.iter_pages`/`load_page` 容错原语即可。heal 的
  写集审计是**同构先例、非共享代码**（诚实划界）。
- **决策P2.1-8（仅判 before∩after 的页；建页/删页/页拆分不在此裁决）**：`sources.dropped`/`body.shrank` 只比对写前
  写后**都存在**的页（in-place 更新腐蚀，ingest 常态）。写前无的新页天然无回退可言；写后没的删页本阶段不裁决（guanlan
  无 ingest 删页工作流；删页若该禁，属另一关注域，勿在 P2.1 顺手扩成 heal 式全写集审计）。**边界声明**：若未来引入
  「页拆分/重构」式 ingest（合法把某源从 X 移到 X2），per-page 源不回退需复议——届时改为「库级源引用不回退」或加白
  名单；v1 不为这一尚不存在的行为预留复杂度。
- **决策P2.1-9（合并纪律=引擎正向引导 + 门禁确定性兜底，职责不混）**：复刻仓内 `lint↔heal`、`check↔SKILL.md` 的
  「检测/引导 + 兜底」双层结构。`SKILL.md` 教 Agent 合并（写前、LLM）、`gate.py` 检测腐蚀（写后、零 LLM）。gate 不
  携带「如何合并」的知识，只判纯结构事实。
- **决策P2.1-10（作用范围 = ingest/backfill；heal 显式排除；开关键落 `page_guard` 缺省值）**：开关键在 `page_guard`
  缺省——**薄壳 `run_guarded_write` 默认 True**（其唯一两个调用方 `ingest.py:72`/`query.py:80` 均要守护，**call site
  不改**）、**核心 `run_guarded_write_result` 默认 False**（`heal.py:419` 直调核心，**不改**、逐字节不变）。**heal 排除
  既是工程也是原则**：①heal 自身写集审计（`_writeset`）已禁既有页正文/非别名 frontmatter 改动——其安全别名收编豁免
  （`_is_safe_alias_collection`）要求 body 与除 `aliases/last_updated` 外的 frontmatter（含 `sources`）**字节不变**，故
  heal 契约下源回退本不可能，P2.1 守护**冗余**；②heal 的语义是「新建实体/概念页 + 纯追加别名」，**从不更新既有页知识**
  （那是 ingest 的活），合并不覆盖之忧与它无关；③若覆盖 heal，其 `GateResult` 会多 `sources.dropped`/`body.shrank`
  kind → 破 heal `--json` 字节契约与可能的退出码。故 heal 排除是正确分层，非图省事。**JSON/SSE 不变性**亦由此成立
  （见 §0 头与 §7）。
- **决策P2.1-11（after 不可信解析 → `sources=None` → 跳过 dropped，去重 check）**：`PageMetaFingerprint.sources` 取
  `frozenset[str] | None`。当某页 frontmatter 坏/缺块、缺 `sources` 键、或 `sources` 类型非法（非字符串列表）时记
  **None**——这三类**已是 `check` 的阻断违规**（unparsable/missing_key/bad_type）。`_check_source_regression` 在
  before 或 after 任一为 None 时**跳过**该页的 `sources.dropped`，把 frontmatter 错误**单独**留给 check、不重复记账
  （否则 `[a,b]`→坏块会额外冒出两条 dropped，与 check 的一条 frontmatter 违规打架）。**空 `[]` 是可信的 `frozenset()`、
  不是 None**：`[a,b]`→`[]`（合法空列表、check 放行）仍判真回退——这恰是要抓的「洗光来源」clobber。`body.shrank`
  与 sources 可信度无关（body_len 总可算），不受此影响。

## 6. 边界（明确不做）

- **不抄 llm_wiki「代码 union frontmatter / 代码落地 link substitution」**——工具不替 Agent 合并，只验证不变量
  （决策P2.1-4）；
- **不引入 type 漂移检测**（决策P2.1-6）；
- **不覆盖 heal**——heal 直调核心、`page_guard` 默认关，其写集审计已更严，源回退本不可能（决策P2.1-7/10）；
- **不裁决 ingest 删页 / 页拆分**——只判 before∩after 的 in-place 腐蚀（决策P2.1-8）；
- **不把 `sources.dropped`/`body.shrank` 加进独立 `check`/`lint`/`health`**——它们无状态、测不了回退，逐字节不变；
- **不新增命令 / 参数 / 退出码 / 依赖 / SSE / JSON 契约**；
- **不重构 heal 的写集审计**（同构先例、非共享代码，决策P2.1-7）；
- **不改 `raw/` 快照、不改有界自愈轮数（仍 2）、不改 `enforce` 失败路径语义**。

## 7. 退出码（无新码）

| 情形 | 退出码 |
|---|---|
| 写入干净（含 `body.shrank` 警告） | `EXIT_OK(0)` |
| 既有页源回退且 2 轮自愈未并回（`sources.dropped`，新阻断违规） | `EXIT_CHECK_FAILED(3)`（复用） |
| `raw/` 被改 / agent 失败 | `EXIT_RAW_MUTATED` / `EXIT_AGENT_ERROR`（逐字不动） |

`body.shrank` 永不改变退出码；`sources.dropped` 复用既有 `check_failed → EXIT_CHECK_FAILED` 通道（决策P2.1-5）。

**机器 JSON 契约不变**（决策P2.1-10）：heal `--json`（`page_guard` 关、`GateResult` 不含新 kind）逐字节不变；
ingest/backfill 无暴露门禁 kind 的机器 JSON（门禁结论走 `report_outcome` 人读输出 / web backfill `result` 仍 null），
`sources.dropped`/`body.shrank` 仅现于**人读**门禁报告。

## 8. 测试要点（新增/扩充 `tests/test_gate.py`；回归 `tests/test_ingest.py`）

以 mock `AgentRunner` 制造「写后某既有页 `sources` 回退 / 正文骤缩」的 wiki 末态，直接验门禁裁决（不打真实 LLM）：

- **源回退 → 阻断 + 自愈**（决策P2.1-2）：写前页 X `sources:[a,b]`；mock runner 写出 `sources:[a]` → 首轮门禁
  `check_failed` 含 `Violation(X, "sources.dropped", …b…)`、`exit_code==EXIT_CHECK_FAILED`；REPAIR_PROMPT 含「并回」
  指令；mock 自愈轮并回 b → 门禁通过、`EXIT_OK`。
- **源只增 → 放行**：写出 `sources:[a,b,c]`（并集扩张）→ 无 `sources.dropped`、通过。
- **多 slug 回退 → 每个一条**：`[a,b]`→`[]` 产两条 `sources.dropped`（a、b 各一），(page,slug) 稳定排序。
- **新建页不误判**：写前无的页写后带 `sources:[a]` → 不在 before∩after，无 `sources.dropped`（决策P2.1-8）。
- **删页不裁决**：写前有、写后无的页 → 不报 `sources.dropped`（本阶段不判删页，决策P2.1-8）。
- **基线不误压制**（§2.1）：`sources.dropped` 必判为「本次新引入」、永不被 `check_baseline` 过滤掉（构造写前已有
  其它历史阻断违规，确认源回退仍被追究）。
- **正文骤缩 → 警告非阻断**（决策P2.1-3）：写前正文 1000 字、写后 300 字（< 0.5×1000）且 before≥200 →
  `body.shrank` 进 `warnings`、`gate.ok is True`、`EXIT_OK`；`report_outcome` 输出含骤缩提示行。
- **骤缩阈值边界**：①写后 600 字（> 0.5×1000）→ 不报；②写前 150 字（< SHRINK_FLOOR）骤缩到 10 字 → 不报（桩页豁免）。
- **骤缩与源回退正交**：一页同时源回退 + 骤缩 → 既出阻断 `sources.dropped` 又出警告 `body.shrank`，互不吞没。
- **坏 frontmatter 去重 → 不重复记账**（决策P2.1-11，**关键**）：before 合法 `sources:[a,b]`、after frontmatter
  坏块 / 缺 `sources` 键 / `sources` 类型非法 → `snapshot_page_meta` 记 `after.sources=None` → `_check_source_regression`
  **跳过** dropped，**不**报 a/b 丢失；该页只由既有 `check` 出**一条** frontmatter 类违规（断言**无** `sources.dropped`）。
  before 本就坏（`sources=None`）→ 无可信基线、同样跳过。`snapshot_page_meta` 全程绝不抛（决策P3-8 口径）。
- **空 `[]` 是真回退、非不可信**（决策P2.1-11）：after `sources:[]`（合法空列表、check 放行）→ `frozenset()` 可信 →
  before `[a,b]` 比出 a、b 两条 `sources.dropped`（与坏 frontmatter 的 None-跳过区分开）。
- **两个默认值各自钉死（决策P2.1-10，最高优先级硬回归门，签名 + 行为双断言）**：默认值错一个就会**误伤 heal 或漏守
  ingest/backfill**，故两层都测、不可只测其一：
  - **签名层（belt）**：`inspect.signature(run_guarded_write).parameters["page_guard"].default is True` **且**
    `inspect.signature(run_guarded_write_result).parameters["page_guard"].default is False`——直接锁缺省值，防有人
    日后顺手翻默认。
  - **行为层（suspenders）**：①`run_guarded_write(...)` **不显式传** `page_guard` → 制造源回退的 mock 末态 → 门禁
    **出** `sources.dropped`（守护确实默认开）；②`run_guarded_write_result(...)` **不显式传** → 同样末态 → `GateResult`
    **不含** `sources.dropped`/`body.shrank`（heal 路径确实默认关）；③heal 端到端（`run_heal`，内部走核心）→ 回执/
    `--json`/退出码逐字节不变。
  - **反向各一**：`run_guarded_write(..., page_guard=False)` 能关、`run_guarded_write_result(..., page_guard=True)`
    能开——确认开关本身可双向覆盖（heal 将来若要守护有路可走）。
