# P3.4 索引回填 `guanlan reindex`（零 LLM）

> 半阶段增强，不是新里程碑（P5 仍 = 多格式）。给 `health.index_missing_page` 配一个**确定性修复器**，
> 补齐 P3「检测有了、缺确定性修复」的空档——正如 P3.2 的 `heal` 之于 `lint.missing_entity`。

## 0. 为什么能零 LLM —— 以及唯一的损耗点

`health.index_missing_page`（磁盘有页、`index.md` 无登记行，见 `health.py:82`）的修复**几乎全是确定性的**：

- **分区**由页所在目录决定（`entities/` → `## Entities`）；
- **链接路径**由文件位置决定（`wiki/entities/DeFi.md` → `entities/DeFi.md`）；
- **锚文本**取 frontmatter `title`（容错档 `page_title`，坏/缺则退化用 stem）；
- **别名注记**取 frontmatter `aliases`。

以上都不需要 LLM。**唯一需要"生成"的是行尾那句 `— 一句话`**（人读摘要 + query 的 2-gram 召回面）。
这是 LLM 的活，零 LLM 给不出。所以 `reindex` 的明确立场是：

> **确定性地把页"登记可见"，把人话摘要留空给后续 `ingest` / 人工补。**

这对 `health` 是充分的——`index_md_links`（`pages.py:281`）只认链接目标在不在，不看摘要。`reindex` 因此
成为 `health.index_missing_page` 的**确定性配对修复器**（区别：`heal` 要造正文故走 LLM；`reindex` 只
登记既有页、无须生成，故零 LLM）。

## 1. 命令契约

```bash
guanlan -C <kb> reindex            # 把"磁盘有、index 无"的内容页登记进 index.md 对应分区（默认应用）
guanlan -C <kb> reindex --dry-run  # 只打印 worklist，不写盘（纯读、零 LLM）
guanlan -C <kb> reindex --prune    # 额外删除 index 里指向不存在文件的悬空行（index_dangling）
guanlan -C <kb> reindex --json     # 结构化回执 {ok, pages_checked, added, pruned}
```

- **默认是"修复"语义**（像 `heal`，不像 `health`/`lint` 的"只报告"）：跑完即写。
- **无 `--strict`、无新退出码**：成功恒 `EXIT_OK(0)`；非知识库根等用法错 `EXIT_USAGE(1)`。沿用「半阶段不引新退出码」纪律。
- **只写 `wiki/index.md`**，不碰 `raw/`、不起 Agentao、无写门禁/快照（与 `graph` 写 `graph/` 同性质，
  `require_kb_root(writable=False)` 即可——只读 `wiki/`、写其下 config catalog）。

## 2. 模块与归口（先把检测下沉，杜绝口径漂移）

index↔磁盘同步逻辑原本**只在** `health._check_index_sync`（`health.py:78`）。`reindex` 必须用**完全相同**
的判定，否则会出现"health 报 missing、reindex 看不到"的漂移。

**做法**：把存在性同步下沉到 `pages.py` 单一归口（决策P3.4-4）：

```python
# pages.py（health 与 reindex 共用）
def index_sync_state(wiki: Path, pages: list[Path] | None = None) -> tuple[list[Path], list[str]]:
    """返回 (missing_pages, dangling_targets)：
       missing  = pages 里 rel 不在 index_md_links 的内容页（绝对 Path，输入序）；
       dangling = index 链接里越界或磁盘无对应文件的目标（相对 wiki/ 字符串，sorted）。
       pages 可传调用方已遍历的内容页列表，免重复 walk，并让 health 的 missing 判定与桩页检查
       共用同一份快照（消两次 walk 间磁盘变动的 TOCTOU 漂移）；缺省则内部 iter_pages 一次。"""
```

- `health._check_index_sync` 改为调用它再包成 `Finding`（行为字节不变，原 health 测试全绿）；
- `reindex` 调用它拿 `missing` 去登记、`dangling` 供 `--prune`；
- 越界守卫 `_target_within_wiki` 一并从 `health.py` 迁入 `pages.py`（同归口）。

新增模块 `guanlan/reindex.py`，复用 `pages.py` 的 `iter_pages / index_sync_state / index_md_links /
load_page / page_title`，对外 `run_reindex(...)` + `reindex_entrypoint(...)` + `main`（与 health/lint 同形）。

## 3. 行为细则

### 3.1 分区路由（确定性）

```python
_DIR_TO_SECTION = {"sources": "Sources", "entities": "Entities",
                   "concepts": "Concepts", "syntheses": "Syntheses"}
```

页的分区 = `path.relative_to(wiki)` 的**首段目录名**。落在四目录之外的杂散内容页（理论上不该有）→
**跳过并告警**，不猜分区。

### 3.2 行构造（零 LLM，全部来自 frontmatter / 文件位置）

- 锚文本：`page_title(meta, stem)`——`title` 是非空字符串就用它，否则退化 `path.stem`。
- 链接目标：`path.relative_to(wiki).as_posix()`（如 `entities/DeFi.md`，与现存行口径一致）。
- 别名注记：frontmatter `aliases` 非空时，确定性追加 `— （别名：a/b）`（conventions 明确要求 index
  行附别名以喂 2-gram 召回；这是零 LLM 能拿到的真实信息，决策P3.4-5）。
- 摘要：**默认留空**（无人话 `一句话`，见 §0、决策P3.4-1）。

例：
```
- [DeFi](entities/DeFi.md) — （别名：defi）        # 有 aliases
- [图灵](entities/图灵.md)                      # 无 aliases
```

### 3.3 插入位置（最小扰动、可幂等）

- 把 `index.md` 按 `## ` 标题切块；新行**追加到目标分区末尾**（现有 HTML 占位注释与已有行原样保留，
  新行接其后——即"分区内最后一条非空行之后"）。
- **不全局重排**已有行（重排会搅动人工/历史顺序、制造无谓 diff）；同批新行按 `iter_pages` 稳定序追加。
- 目标分区标题缺失（被人删了）→ 在文末补建 `## <Section>` 再追加。
- 保留原文件 EOL（检测 `\r\n`/`\n`）与尾换行存在性，其余字节不动。

### 3.4 幂等

天然幂等：判据复用 `index_sync_state`，第二次跑 `missing` 即空 → 无改动、不写盘。`--dry-run` 只打印。

### 3.5 `--prune`（可选，删除悬空行）

- 删掉 `dangling` 对应的 index 行：逐行抽 `index_md_links`，整行链接目标**全部** ∈ dangling 集才删
  （单行单链接，精确匹配、不误删含正常链接的行）。
- **默认关**（决策P3.4-3）：删行可能误删"文件被改名/移动后人想保留的行"，破坏性高于追加，须显式开启
  （呼应 heal 默认非自动、init 不覆盖的保守纪律）。

## 4. 决策（沿用仓库 `决策PX-N` 体例）

- **决策P3.4-1（摘要留空 vs 占位符）**：默认**留空**（`- [title](path)`），最干净、不伪造内容；不写
  `— ⟨待补⟩` 之类占位（会污染 2-gram 召回，且 health 本就不需要）。要可 grep 出"待补摘要"者，后续可
  加 `--placeholder`——不设为默认。
- **决策P3.4-2（追加而非重排）**：见 §3.3，保人工序、最小 diff。
- **决策P3.4-3（prune 默认关）**：见 §3.5。
- **决策P3.4-4（检测单一归口）**：见 §2，health/reindex 共用 `index_sync_state`，守住"检测与修复同口径"。
- **决策P3.4-5（别名零 LLM 入行）**：frontmatter `aliases` 直接进行尾注记，既零 LLM 又补召回。
- **决策P3.4-6（不写 log.md）**：纯确定性维护（无 LLM 写作、无 raw/ 风险），不向 `log.md` 留痕——与
  `graph` 一致（heal 因走 LLM 写门禁才追加 log，语境不同）。

## 5. JSON 契约 / 退出码

```jsonc
{ "ok": true, "pages_checked": 795,
  "added":  [{ "page": "wiki/entities/DeFi.md", "section": "Entities",
               "line": "- [DeFi](entities/DeFi.md) — （别名：defi）" }],
  "pruned": [] }              // --prune 时填被删的整行原文；否则恒空
```

退出码：成功 `EXIT_OK(0)`；用法/IO 错 `EXIT_USAGE(1)`。**不引入新码。**

## 6. 边界（明确不做）

- 不生成人话摘要（§0）；
- 不改任何**内容页**（只动 `index.md`）；
- 不校验分区与 frontmatter `type` 是否自洽（那是 `check`/`lint` 的事，`reindex` 只做存在性登记）；
- 不碰 `log.md`（决策P3.4-6）、不碰 `raw/`、不起 LLM。

## 7. 测试要点

- missing 页被登记到正确分区、行格式符 conventions；登记后 `health` 该项归零。
- 幂等：连跑两次第二次零改动。
- frontmatter 坏/缺 title → 退化用 stem，不抛。
- 有别名 → 行尾注记；无别名 → 无尾注。
- `--dry-run` 不写盘；`--prune` 精确删悬空行且不误删正常行。
- 分区标题缺失时补建。
- 与 `health._check_index_sync` 共用归口后，原 health 测试全绿（行为字节不变）。
