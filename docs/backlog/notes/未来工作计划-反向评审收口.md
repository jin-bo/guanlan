### 轨 C · E 轨（双重佐证"该推迟"，非现在）

两篇评审最大的共识：**gbrain 与 OpenKB 本身就是 E 轨的参考实现**，而它们恰恰**反证观澜推迟 E 轨是对的**：

- **E1 向量 / rerank / 持久索引**：gbrain hybrid 检索 + Postgres、OpenKB PageIndex 长文树。哲学贴合（"markdown 真相、索引可重建"一字不差），但量级是另一回事——P5.0 故意 stateless、无落盘派生物。记入 E1 选型，现在别动。
- **E2 多租户 / 远程**：gbrain OAuth/HTTP MCP + 多脑租户。观澜 P4.9 reader + P4.10 MCP 已是本地只读前哨；source 级作用域属真正 E2，动数据模型，排 P 轨之后。
- **E3 夜间富化**：gbrain autopilot 22 相位 dream cycle 是样板，但它把零-LLM 结构修复（lint --fix / extract）和 LLM 合成（synthesize / consolidate）**揉进同一相位链**——正是观澜要避免的（**带刺的反面参照**）。排 E3 时参照其"先修文件后建索引"依赖序、但**守零-LLM/LLM 分档**。

外加一条早画好的后置半阶段：**Web 图可视化 `P4.12`**（[`next-milestone-and-graph-viz.md`](next-milestone-and-graph-viz.md) §3 定的可选后置 P4.x；原拟 `P4.11` 已被信任边界占用故顺延），触发条件"语料规模上来后"，复用现成 `graph.json`（已含 community/bridges/cut_vertices），vendored 零-CDN 力导向、不碰 `graph.html` 零-JS 承诺。**非下一里程碑。**

## 3. 关键洞察：一个决策解锁两件事

最高杠杆的发现——**`P3.9 源撤回` 与 `P3.7 语义审计` 卡在同一道架构决策上**：**宿主（wrapper/host）能否写 / 删 wiki *内容*页**。

- **P3.7-3a**：wrapper 要不要在内容页 stamp `raw_digest`（**头一回 wrapper 碰内容页**——以往只 verify，仅 `reindex` 写过 `index.md` 那种 *config* 页）。
- **P3.9**：`remove` 要不要删空 wiki *内容*页（越"Agent 写 wiki"线比 P3.7 更远——P3.7 只 stamp、不删）。

两条 backlog 未决笔记**都明说"建议一并拍"**。这道边界拍清楚，**同时解锁 P3.7 和 P3.9**——是整张地图上回报最高的单点决策。共同前提已稳：纯 provenance / 人发起纠正都是 Agent 结构上做不到或不该做的（LLM 抄不准 sha256；撤回是人意图），落点类比 `reindex` 写 config 页的先例。

1. **轨 A 随手补强**（无决策）：`finding 因果排序`（不单列，**已实现**）→ `链接归一 P3.8`（**已实现**）→ `backlink 重排 P5.3`（**已实现**），穿插 `schema detect P3.10`（**已实现**）。低风险、可增量。<br>**轨 A 已全部清空**：末项 `检索质量回放 fixture` 也已落 `tests/test_search_quality.py`（**已实现**）。
2. **拍那道边界决策**（宿主写 / 删内容页 = 未决P3.7-3a + P3.9 一并拍）。← **最高杠杆**
3. **`P3.9 源撤回`**（零-LLM、运营刚需）先落——比 P3.7 轻，决策一拍即可动。
4. **`P3.7 语义审计`**——首个 LLM-in-维护里程碑，raw_digest 收敛后进实现。
5. **`P6 技能蒸馏`**——拍 未决P6-1（skill 归属）后动，生成层终点。

> 排序原则：先清**零成本零决策**的轨 A，再用**一道决策**解锁轨 B 的两件事，E 轨整体后置。与两篇评审各自 §排期（gbrain §10 / openkb §8）一致，本笔记只做**合并去重 + 标出共享决策**。

## 5. 明确不做（红线对照）

LiteLLM 多 provider / 跨模型方差评测（绑 agentao）· OpenKB `recompile`（覆盖人工编辑、悖增量 ingest）· gbrain DB 外键"宁缺勿悬空"（扼杀前向引用生长信号）· OpenKB strip-无匹配链接（删前向引用占位，撞决策8）· gbrain schema mutate/migrate/retype 回填 + calibration/takes/Brier 评分（过重 / 绑 DB / gbrain 预测域专属）。

## 6. 观澜已领先（不该反向借，作信心锚）

无状态图重建（无残留陈旧边——gbrain 自认 add-only 留 stale edge 是缺陷）· 拓扑 lint（桥/割点/社区——两参照都只把图用于检索加权、不做健康度）· `raw/` 不可变 + 快照门禁（`EXIT_RAW_MUTATED`，两参照皆无等价源不可变闸）· 零-LLM/LLM 干净分档 · 薄壳（wrapper 不携带业务智能）。这五点是观澜区别于两个蓝本的结构优势，换线时不可丢。
