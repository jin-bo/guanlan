## 拍板结论（先放结论）

**取 Option A——wrapper 门禁后直写 source 页 `raw_digest`，不交 Agent。** 落进 [`../../P3.7-语义审计.md`](../../P3.7-语义审计.md) 决策P3.7-3a。理由：指纹是不可变 raw 的 sha256，**纯 provenance、非内容创作**；Agent **结构上做不到**（`INGEST_PROMPT` 禁 shell + LLM 抄不准 64 位 hex）；越线幅度极小且有 `reindex` 先例，相比 Option B「LLM 抄 hash + 回读兜底」一整层易腐机制更简单、零腐败。**可靠性压倒"Agent 是 wiki 唯一写者"的纯粹性。**

> **判例边界（窄）**：A 越的这一小步只是「宿主对内容页做**纯 provenance 的确定性写**」——它**不等于**"宿主可任意写/删 wiki 内容"。P3.9 源撤回那种**删除**内容页仍是更重的一步，不由本决策自动获批，须各自评估（见 [`未来工作计划-反向评审收口.md`](未来工作计划-反向评审收口.md) §3：两者曾被列为"同一道边界、建议一并拍"，现**两半均已拍板取 A 并实现**——P3.7 `guanlan audit` / P3.9 `guanlan remove`）。

## 曾经的未决项与取舍（存档）

两个选项，共同前提都成立：**只 stamp 本次摄入那一张 source 页，绝不顺手刷新别页**（否则抹平别处真实漂移，致命）。分歧只在写入主体：

- **Option A（采纳）—— wrapper 门禁后直写 source 页**
  - 优：值由代码算、零腐败、零 LLM、幂等确定。
  - 缺：**头一回让 wrapper 碰内容页**——以往 wrapper 只 verify，仅 `reindex` 写过 `index.md` 那种 *config* 页；这越过"Agent 写 wiki、wrapper 只验"默契一小步。
  - 正当性：纯 provenance（不可变 raw 的 hash）、Agent 结构上做不到（`INGEST_PROMPT` 禁 shell + LLM 不会/抄不准 sha256）、类比 reindex。

- **Option B（否决）—— wrapper 算值塞进 prompt、Agent 文件工具写 + wrapper 回读校验**
  - 优：守住"Agent 是 wiki 唯一写者"。
  - 缺：要 LLM 逐字抄 64 位 hex，**易腐**；须加一层"写后回读比对、不符则 wrapper 兜底改正或报错"。

**拍板取 A（可靠性优先）。** A 越"Agent 写 wiki"默契仅一小步、且有 reindex 先例；B 的「LLM 抄 hash」是结构性易腐点，回读兜底一层不偿失。
