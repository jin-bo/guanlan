## 1. 目标与范围

**P4.4 = 把 CLI 交互台的常用斜杠命令搬进浏览器对话框，全部只读——看状态/上下文/技能/工具、开关会话，绝不改写任何文件、不切换姿态。**

现状（P4 + P4.1/P4.2/P4.3）：浏览器能投喂、ingest、heal、只读多轮问答、历史会话落盘续聊，但**对话框只能发消息给 LLM**——无法像 `agentao` 交互 CLI 那样用 `/status` 看上下文用量、`/skills` 看技能、`/tools` 看工具、`/new` 开新会话。P4.4 用一条**前端命令解析层** + 一对**只读自省端点**补齐它。

参照实现是 **agentao 自己的 `cli/input_loop.py`**（已核对源码）：它把 `/help /clear /new /status /skills /context /mode /tools …` 逐条分发到 agent 的文档化自省面。P4.4 在浏览器里**镜像其中 8 条**（read-only 语义），背后复用同一组 agentao 嵌入面，Web 层零业务智能（「收命令 → 调既有自省函数 → 序列化」）。

### 纳入 P4.4

- **前端命令解析层**：输入以 `/` 开头时 `app.js` 本地解析 → 路由到「纯客户端渲染」或「只读自省端点」，**斜杠命令绝不进 `/api/chat` 喂给 LLM**（镜像 CLI：命令不消耗对话轮次，决策P4.4-1）。
- **8 条命令**（§2）：`/help`（纯客户端）、`/new` `/clear`（会话生命周期）、`/status` `/context` `/skills` `/tools` `/mode`（只读自省/报告）。
- **一对只读自省端点**：`GET /api/info`（app 级，无会话也能答）+ `GET /api/chat/{id}/info`（会话级，一处喂 `/status /context /skills /tools /mode`；**对冷会话出不建 agent 的部分信息**，决策P4.4-7）。

### 不纳入 P4.4

- **`/mode` 切换姿态 / 任何可写能力** → 推后到 **P4.5「可写 Web 工作会话」**（§7）。P4.4 的 `/mode` **仅报告**当前姿态（恒 `read-only`）+ 说明可写在 P4.5，**不接受 `read-only` 以外的值、不翻任何置位**（决策P4.4-6）。
- **`/compact`** → 不实现、**也不列入 `/help`**（决策P4.4-5）：它打 LLM + 改写历史 + 需会话锁/落盘，是这批最重的一条；自动压缩已在阈值触发（`context_manager.needs_compression`）。等真正实现时再开放，避免在「只读小设计」里挂未实现命令。
- **`/skills activate|deactivate`、`/tools <name>` schema、`/context limit <n>`、`/model` `/provider` `/temperature` `/memory`** → 不在本相位（决策P4.4-3）。自省命令**只列不改**。

## 2. 命令清单与映射（`app.js` 解析层）

输入 `input.trim()` 以 `/` 开头即视为命令：`const [cmd, ...rest] = body.slice(1).split(/\s+/)`，`arg = rest.join(" ")`。未知命令 → 本地提示「/help 看支持列表」。**命令一律不进 `/api/chat`**（决策P4.4-1）。

| 命令 | 类别 | 实现 | 复用的 agentao 面 / 既有件 |
|------|------|------|---------------------------|
| `/help` | 纯客户端 | `#chat-log` 渲染 note：本表 8 条 + 一行说明 | —（静态） |
| `/new` | 会话生命周期 | == 既有「新会话」：清本地视图、`conversationId=null`、保留盘上历史 | 既有 `#chat-new` |
| `/clear` | 会话生命周期 | 开新会话 **且** 删当前会话持久快照（== 删除当前 + 新建）；无活动会话退化为 `/new` | `DELETE /api/conversations/{id}` + `#chat-new`（决策P4.4-4） |
| `/status` | 只读自省 | 有活动会话 → `GET /api/chat/{id}/info`；否则 → `GET /api/info` | `get_current_model` + `context_manager.get_usage_stats` + `skill_manager` |
| `/context` | 只读自省 | 活动会话上下文用量（估算/上限/占比/分项）；无/冷会话 → 提示 | `context_manager.get_usage_stats` + `estimate_tokens_breakdown` |
| `/skills` | 只读自省 | 列 available + active 技能（名/描述/是否激活）；**只列不改** | `skill_manager.list_available_skills`/`get_active_skills`/`get_skill_description` |
| `/tools` | 只读自省 | 列已注册工具（名/描述/需确认/只读姿态下是否被拦）；**只列不改** | `agent.tools.list_tools()`（`.name/.description/.requires_confirmation`） |
| `/mode` | 报告（只读） | 报告当前姿态（恒 `read-only`）+ 说明：「Web 问答只读；可写会话见 P4.5」 | `GET /api/info` / `GET /api/chat/{id}/info` 的 `mode` 常量 |

- **`/status` 与 `/context`/`/skills`/`/tools`/`/mode` 共喂一个会话级端点**（§3）：前端按命令各取所需字段渲染（一次请求多命令复用）。
- **无活动会话的优雅降级**：`/status` `/mode` 走 app 级 `GET /api/info`（不需 agent）；`/context` `/skills` `/tools` 报告的是 agent 级事实，无会话 → 「先提一个问题以开启会话」；**冷会话**（仅盘上、打开历史后续）→ info 出部分信息 + 「续聊一轮以恢复完整上下文/工具/技能」（决策P4.4-7）。
- **渲染**：命令输出走 `addMsg("note", …)`（新增轻量 `note`/`cmd` 样式），落对话流、**不开浮层**（与右栏 wiki / `#overlay` 报告作业分离，决策P4.4-2）；字符串经 `escapeHtml`/`textContent` 防注入。

## 3. 端点契约（`guanlan/web/app.py`，只读自省）

| 方法 路径 | 入参 | 复用 | 返回 | 错误 |
|------|------|------|------|------|
| `GET /api/info` | — | `app.state`（root/model）+ `conversations` 计数 | `{kb_name, model, mode:"read-only", persist, conversations, max_conversations}`（无会话也能答；零 LLM） | —（恒 200） |
| `GET /api/chat/{id}/info` | `id`（路径） | 内存命中 → `Conversation.info()`；冷会话 → `ConversationStore.cold_info(id)`（不建 agent，决策P4.4-7） | 见下 info 体（`live:true` 全量 / `live:false` 部分） | `404` 未知且非盘上 Web 会话 |

**info 体**（`Conversation.info()` 全量；`cold_info` 出标 `*` 的部分项、其余 `null`）：

```jsonc
{
  "id": "…", "title": "…", "turns": 3, "live": true,   // cold_info: live=false
  "model": "…",                          // 内存:get_current_model() / 冷:load_session 的 model*
  "mode": "read-only",                   // 恒常量（P4.4 Web 只读）；P4.5 起才有别的值
  "messages": 12,                        // 内存:len(agent.messages) / 冷:catalog message_count*
  "context": { … } | null,               // get_usage_stats + estimate_tokens_breakdown；冷会话 null
  "skills": { "active":[…], "available":[…] } | null,   // skill_manager；冷会话 null
  "tools":  [ {"name","description","requires_confirmation","blocked"}, … ] | null  // list_tools；冷会话 null
                                                                                      // blocked ∈ {true,false,"unknown"}（静态判定，见 §3）
}
```

- **`info()`/`cold_info()` 落在 `chat.py`，`app.py` 只序列化**（决策P4.4-2，同 P4.3 `heal_result_dict` 归 `heal.py`）：读 agentao 自省面（`get_current_model`/`context_manager`/`skill_manager`/`tools.list_tools`/`tool_runner` 只读态）的逻辑归 `Conversation`/`ConversationStore`，维持「Web 层零业务智能」。
- **`blocked` = 只读姿态下该工具是否被 `tool_runner` 拦**（写/shell 类 `true`、读类 `false`）：让 `/tools` 如实显示「这些工具在只读 Web 下被禁」。判定**优先用 agentao 元数据**（工具自带的写/shell 类别或只读放行标记，若有）；**无元数据时按已知工具名/类别静态判定**（`write_file`/`replace`/shell 等 → `true`，`read_file`/`grep` 等读类 → `false`），**未知工具显示 `unknown`、不执行工具探测**（不为求 `blocked` 真去跑/试调工具）。§5 留测试兜底真实工具名。**P4.4 姿态恒只读，`blocked` 是静态展示、不随请求变。**
- **冷会话不懒恢复**（决策P4.4-7）：恢复要建 agent（重），纯自省不值当。`cold_info` 经 `load_session`/即时 catalog 取 `title`/`model`/`message_count`（过 P4.2 §2.2 两道精确闸：规范 UUID + `_disk_session` 作用域），`context`/`skills`/`tools` 置 `null`。真正恢复仍由 `POST /api/chat`（续聊一轮）走 `restore`。
- **零 LLM、`to_thread` 卸载**（`get_usage_stats` token 估算可能不廉价，按 P4 决策P4-2 卸离事件循环）；**不新增退出码、不新增 SSE**（决策P4.4-8）。

## 4. 自省落 `chat.py`（`Conversation.info()` / `ConversationStore.cold_info()`）

P4.4 **不改 `Conversation` 的构造与只读两点置位**（P4 现状全只读不动），只**加两个只读方法**：

```python
# chat.py
class Conversation:
    def info(self) -> dict:
        """会话级只读自省：模型 / 上下文用量 / 技能 / 工具（全量）。不改任何 agent 状态。"""
        cm = self.agent.context_manager
        stats = cm.get_usage_stats(self.agent.messages)
        bd = cm.estimate_tokens_breakdown(...)            # 见 agentao get_conversation_summary 的取法
        return {
            "id": self.id, "title": self.title, "turns": self.turns, "live": True,
            "model": self.agent.get_current_model(),
            "mode": "read-only",                          # P4.4 恒只读
            "messages": len(self.agent.messages),
            "context": {... stats ..., "breakdown": bd},
            "skills": {"active": [...], "available": [...]},  # skill_manager
            "tools": [{"name": t.name, "description": t.description,
                       "requires_confirmation": t.requires_confirmation,
                       "blocked": _blocked_in_readonly(t)} for t in sorted(self.agent.tools.list_tools(), key=...)],
        }
        # _blocked_in_readonly(t) -> True | False | "unknown"：优先读 agentao 工具元数据（写/shell 类别或
        # 只读放行标记）；无元数据则按已知工具名/类别静态映射；都不命中返回 "unknown"。纯静态、不试调工具。

class ConversationStore:
    def cold_info(self, cid: str) -> dict | None:
        """盘上-only 会话的部分自省（不建 agent）：title/model/message_count；context/skills/tools=None。
        过 P4.2 §2.2 两道精确闸（规范 UUID + _disk_session）；非 Web/未知/持久化关 → None（端点 404）。"""
        ...
```

`app.py` 端点只做内存命中 → `conv.info()` / 冷 → `store.cold_info()` / 否则 404 的分流 + 序列化。

## 5. 前端（`guanlan/web/static/`）

- **命令解析前置**（`sendChat` 之前）：`if (msg.startsWith("/")) { handleSlash(msg); return; }`——命令不走 SSE、不占轮次（决策P4.4-1）。
- **`handleSlash(msg)`** `switch`：
  - `help` → 静态 8 条 note。
  - `new` → 复用 `#chat-new`。
  - `clear` → 有 `conversationId`：`DELETE` 后清 log + 置 null；否则同 `new`。
  - `status`/`context`/`skills`/`tools`/`mode` → 取 info（有会话 `GET /api/chat/{id}/info`，否则 `GET /api/info`），按命令渲染；冷会话 info（`live:false`、context/skills/tools 为 null）→ 部分信息 + 「续聊一轮以恢复」提示（决策P4.4-7）；`/mode` 报「read-only（Web 问答只读；可写会话见 P4.5）」。
  - 其他（含 `/compact`、未知）→ 「未知命令，/help」note（`/compact` 不在 /help、按未知处理，决策P4.4-5）。
- **渲染**：`addMsg` + `note`/`cmd` 样式。`/status` 渲染模型/姿态/轮次/上下文占比；`/context` token 估算条；`/skills` 列技能；`/tools` 列工具（`blocked` 项灰显标「只读禁」）。`#chat-input` placeholder 追加「输入 / 看命令」。**不改两栏布局、不引新组件**（决策P4-3 克制）。

## 6. 决策记录

- **\[决策P4.4-1] 斜杠命令前端解析、绝不进 `/api/chat`。** 命令是宿主交互、非给 LLM 的消息：不消耗轮次、不打 LLM、不入历史。展示类（`/help`）客户端渲染，自省类调只读端点，生命周期类复用既有逻辑——SSE 对话路径保持干净。
- **\[决策P4.4-2] 自省逻辑落 `chat.py`，`app.py` 只序列化。** 一个会话级 `GET /api/chat/{id}/info` 喂 `/status /context /skills /tools /mode`；`Conversation.info()`/`ConversationStore.cold_info()` 读 agentao 自省面，维持「Web 零业务智能」（同 P4.3 `heal_result_dict` 归 `heal.py`）。端点零 LLM、`to_thread` 卸载。
- **\[决策P4.4-3] Web v1 自省命令只列不改。** `/skills` `/tools` `/context` 只读展示，不做 CLI 的 `activate/deactivate`、`/tools <name>` schema、`/context limit`、`/model`/`/provider` 等。Web 是单用户本地台，运行时改技能/上限/模型价值低且引可变状态。
- **\[决策P4.4-4] `/new` 保留盘上历史，`/clear` 删当前会话持久快照、不触库级记忆。** `/new` == 既有「新会话」（仅清本地视图、盘上历史可在「历史会话」再开）；`/clear` == `DELETE /api/conversations/{id}` 级联删盘 + 开新会话；无活动会话退化为 `/new`。不调 `memory_manager.clear`（越权、Web 无记忆 UI），与 CLI `/clear` 清记忆刻意不对齐。
- **\[决策P4.4-5] `/compact` 不实现、不列入 `/help`。** 唯一打 LLM 且改写历史的命令（`compress_messages` + 会话锁 + 落盘），最重；自动压缩已在阈值触发。在「只读小设计」里挂未实现命令是产品噪音——按未知命令处理，留实现时再开放（届时对标 P4.3 写作业骨架：会话锁内 off-loop 跑、成功后落盘）。
- **\[决策P4.4-6] `/mode` 仅报告、Web 保持只读、可写推 P4.5。** P4.4 不引入 `--mode`、不翻任何置位、不接受 `read-only` 以外的值；`/mode` 报当前姿态（恒 `read-only`）+ 指向 P4.5。理由：可写 `/mode workspace-write` 会把「只读问答」升级成「可写 agent 会话」，连带写边界守卫（`raw/` 不变性、wiki 写门禁、config 守卫、可写会话恢复），是独立相位的体量，不塞进只读小设计（§7 前瞻）。
- **\[决策P4.4-7] 冷历史会话：info 出不建 agent 的部分信息。** 打开历史后前端持 disk-only `conversationId`。`GET /api/chat/{id}/info` 对冷会话走 `cold_info`（`load_session`/catalog 取 title/model/message_count，`context`/`skills`/`tools` 为 `null`、`live:false`），**不建 agent**（纯自省不值当一次重恢复）；前端据此提示「续聊一轮以恢复后查看完整信息」。真正恢复仍由 `POST /api/chat` 续聊走 `restore`。
- **\[决策P4.4-8] 不新增退出码、不新增 SSE、不动既有写线/只读姿态。** 自省是同步 GET（无作业、无流）；命令解析在前端（无新协议）；`Conversation` 的只读两点置位（决策P4-8）原样不动。ingest/heal/投喂单写者 FIFO、`/api/chat` SSE 全不变。

## 7. 前瞻：P4.5「可写 Web 工作会话」（不在 P4.4，仅记前置）

> 已单独成文：[`P4.5-可写Web工作会话.md`](P4.5-可写Web工作会话.md)。下表为其前置清单的摘要。

把 `/mode workspace-write`（人在浏览器里驱动 agent 写工作区）作为**独立相位**，前置须闭合的安全边界（评审结论）：

- **写边界事实**：agentao `PathPolicy` 把文件写工具 contain 在 `working_directory` = **kb 根**（不细分 `workspace/`）、**且不查 shell 参数**（`security/path_policy.py`）。故裸 `workspace-write` 能写 `raw/`/`wiki/`/`config`——必须先补确定性守卫。
- **`raw/` 硬只读**：① 结构化写工具 `write_file`/`replace`（参数名 **`file_path`**）的 permission deny-rules（经 `permission_engine.add_run_rules(deny=…)`，进 `_run_scope_rules` 每模式先评估）；② 每轮可写 turn 套 P2 `raw/` 前后快照兜 shell 旁路。
- **`wiki/` 写门禁**：chat 直写 `wiki/` 须过 ingest 的 `check.py`（frontmatter/断链）等价校验，否则破 P2/P4 gated 写线。
- **config 守卫对称**：`AGENTAO.md`/`SCHEMA.md`/`index.md`/`log.md`/`overview.md` 与 `raw/` 一样纳入快照/hash，避免「raw 硬只读、config 尽力」的不对称边界。
- **姿态值对齐枚举**：agentao `PermissionMode` = `read-only`/`workspace-write`/`full-access`/`plan`（**无 `full`**）；Web 只用 `read-only`/`workspace-write`，**绝不 `full-access`/`plan`**。
- **可写会话恢复语义**：冷会话恢复后的姿态、`/mode` 翻转（两点置位翻转 + deny-rules 永久注入、`set_mode` 不装卸——`PermissionEngine` 无公开 remove/clear）、持会话锁与在飞 turn 串行。
