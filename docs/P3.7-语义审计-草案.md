## 0. 一句话

`guanlan audit`：一个**确定性粗筛（零 LLM）→ LLM 复核（复用 P2 写门禁）**的两层命令，对既有 wiki 做语义体检——先用结构信号圈出**疑点页**（零 LLM、可 `--dry-run` 纯读预览），再只对疑点交 Agentao+skill 做语义判断，确认则就地标 `## ⚠️ 矛盾与存疑` / 更新过期论断，驳回则记一笔。**精确复刻 P3.2 的 `lint.missing_entity`（检测）↔ `heal`（物化）配对**，只是把"检测"从结构升到语义触发、把"物化"换成"复核标注"。

## 1. 目标与范围

### 为什么不照搬 swarmvault

swarmvault 的语义层建在一张 **claim 图**上：每条抽取的论断带 `polarity ∈ {positive,negative,neutral}` + `confidence`，矛盾检测先零-LLM 用 polarity + token Jaccard≥0.3 粗筛候选、再交 LLM 复核（`analysis.ts` / `vault.ts:detectContradictions`）。这套有两个对 guanlan 致命的问题：

1. **它要在 wrapper 里重建一个 claim/polarity 数据模型** —— 直接违反 guanlan「`guanlan/` 不携带业务智能」的红线（CLAUDE.md 架构 §1）。论断抽取、极性判定本身就是语义活，本该归 LLM，不该塞进确定性脚本。
2. **它的极性检测器是英文正则**（`analysis.ts:detectPolarity`，`return "negative"` 命中英文否定式）。搬到 guanlan 的 CJK + 2-gram 语料上只会**高噪**——一个天天误报的 advisory 会被训练成噪音被无视（同 ESLint 噪音规则的命运）。

所以本设计**只借形状、不借实现**：双层分诊里 **Layer-1 只做最廉价的结构粗筛**（不重建 claim 图），**所有语义判断下放给 Layer-2 的 LLM**（Agentao + skill）。这正是 guanlan 在 P3.2 已经验证过的分工——`missing_entities()` 只做结构聚合，"这页该写什么"全交给 `heal` 的 LLM。

### 纳入 P3.7（本草案落地的）

- **`guanlan audit`** 命令：worklist（零 LLM）+ LLM 复核（复用门禁），契约对齐 `heal`。
- **第一类语义疑点：过期论断（source-drift）** —— 作为本阶段**唯一跑通的 worked example**（决策P3.7-2）。触发信号是结构的、零-LLM、低噪：页所引 `raw/` 源的内容指纹相对**建页时记录的指纹**已变 → 该页论断**可能**过期，交 LLM 复核。
- **一项新 provenance 约定**：**wrapper** 在 ingest 过完门禁后，把所摄 raw 文件的内容指纹 stamp 进对应 `wiki/sources/<slug>.md` 摘要页的 `raw_digest` 字段（决策P3.7-3，**非 Agent 写**）。这是整个阶段的**拱心石**——没有"建页时的源版本"这个锚，源漂移无从判定。

### 不纳入 P3.7（同框架的兄弟，留给 P3.7.x）

- **矛盾复检**：高价值、但 Layer-1 触发信号更难做到低噪——**已在 §11.1 展开为 P3.7.1 兄弟小设计**（复用同一 `audit` 命令、同一源指纹 provenance，主信号"多源页+源漂移"零新成本），按本草案放在 source-drift 跑通**之后**接入。
- **资料缺口**：与既有 `lint.missing_entity` / `health.stub_page` 大面积重叠，价值增量小，暂不单列。
- 不引入 claim/polarity 数据模型、不引入 confidence 数值、不引入衰减分写回页面（swarmvault `decayScore` 那套要求宿主往页写派生态，见 §10 风险）。
- 不新增退出码、不碰 `check` 门禁语义、不动 `raw/`。

## 2. 双层分诊架构（对齐 P3.2，不另起炉灶）

```
                既有 P3.2                      本草案 P3.7
  ┌─────────────────────────┐      ┌─────────────────────────────┐
  │ Layer-1  零 LLM 归口      │      │ Layer-1  零 LLM 归口          │
  │ missing_entities()       │  ⇒   │ audit_candidates()           │
  │ ＝ 结构聚合断链           │      │ ＝ 结构比对源指纹 → 疑点页    │
  └───────────┬─────────────┘      └───────────┬─────────────────┘
              │ 共享 worklist                    │ 共享 worklist
  ┌───────────┴─────────────┐      ┌───────────┴─────────────────┐
  │ Layer-2  LLM via 门禁     │      │ Layer-2  LLM via 门禁         │
  │ heal: 物化缺失页          │  ⇒   │ audit: 复核并就地标注/更新     │
  │ run_guarded_write_result │      │ run_guarded_write_result     │
  └─────────────────────────┘      └─────────────────────────────┘
```

复用点（逐一对齐 `heal.py`）：

- **零-LLM 归口被两处共享**：`heal` 里 `missing_entities()` 同时喂 `lint`（advisory 展示）与 `heal`（worklist）。本草案 `audit_candidates()` 同时喂 `audit --dry-run`（advisory 预览）与 `audit`（worklist）。**单一口径、永不分叉**（决策P3.2-1 的复用）。
- **写路径复用 P2 单写者门禁**：`audit` 的 LLM 写走 `gate.run_guarded_write_result`（与 `ingest`/`heal`/`backfill` 同一条编排核心、同一 `raw/` 快照前后比对、同 P2.1 源不回退闸）。**不另开门禁**。
- **结构化核 + 薄壳**：`run_audit_result(...) -> AuditRun`（不打印的编排核，仿 `run_heal_result`）+ `run_audit(...)` CLI 薄壳 + `audit_entrypoint`。`--dry-run` 纯读零-LLM 不进核（仿决策P3.2-5）。
- **写后确定性回执**：复核完成后零-LLM 复查（§7），回答"这页到底被标了 / 被更新了没"，不靠 LLM 自述。

## 3. Layer-1：确定性粗筛 `audit_candidates()`（零 LLM）

签名（仿 `missing_entities()` / `compute_worklist`）：

```python
@dataclass(frozen=True)
class AuditCandidate:
    page: str                       # 相对库根 posix（source 摘要页 或 引用它的页）
    reason: str                     # "source-drift"（直接） / "cites-drifted-source"（传播）
    drifted_slugs: tuple[str, ...]  # 触发本候选的、已漂移的 source slug（升序）

def audit_candidates(wiki: Path, raw: Path) -> list[AuditCandidate]:
    ...
```

判定分**两步**（全程读文件 + 比 hash，零 LLM、确定性、可字节稳定排序）：

**第一步——找漂移的 source 摘要页**：遍历 `wiki/sources/*.md`，对每张带 `raw_digest` 的页：
- 解出其 `raw_digest` 的 `<raw路径>@sha256:<记录值>`，读该 raw 文件现字节、算现 sha256，比对：
  - 不等 → 该 source slug **已漂移**，进漂移集 `S`；摘要页本身记一条候选 `reason="source-drift"`。
  - raw 文件已不存在 → **不在此报**（那是 `check.sources.unresolved` / 缺源的活，正交不抢口径）。
  - 页**无** `raw_digest`（旧页 / P3.7 前建）→ **跳过不罚**（向后兼容，决策P3.7-5；同 swarmvault `lastConfirmedAt` 缺失不罚）。

**第二步——沿 `sources:` slug 图传播到引用页**：`iter_pages(wiki)` 遍历非 config 页（同 `health`/`check` 口径），对每页读 `sources:`（容错档 `load_page`），若其 slug 集与 `S` 有交 → 记一条候选 `reason="cites-drifted-source"`、`drifted_slugs` = 该交集。

输出按 `(page)` 升序稳定。**指纹只在 source 页比一次**（不是每张引用页各算各的），引用页只做廉价的 slug 集合求交——这正是把指纹收在 source 页一份的红利。

**为什么这个信号配做 Layer-1**（反向评审的唯一幸存者）：纯**结构**——只比 hash + 求 slug 交集，无 NLP、无 claim 图、对 CJK 无偏。低噪：只有 raw 真被替换过的源才触发；没被覆盖过的库 `audit` 一条不报（§4.4）。召回够用：一个源一漂移，它的摘要页 + 所有引它的页都被点名，逐页交 Layer-2 判"到底过期没"。

## 4. 拱心石：`raw_digest` provenance（**已定稿**的约定）

读 `check.py` / `conventions.md` / `ingest.py` 后，上一轮草案这节有三处错，逐一校正后定稿如下。

### 4.0 先纠三处错（只有读码才看得出）

1. **`sources:` 列的不是 raw 路径，是 `wiki/sources/<slug>.md` 摘要页的 slug**（`check._check_sources`：每个 slug 必须解析到 `wiki/sources/<slug>.md`；conventions §frontmatter：「列 source 页的 slug，不含路径/扩展名」）。真实链条是 `entity/concept 页 → sources:[slug] → wiki/sources/<slug>.md 摘要页 → 摘 raw/<file>`。
2. **故源指纹不该复制到每张引用页**（上一轮 `source_digests:{slug:hash}` 既冗余、又会多页各记一份互相打架）——它该**只落在 source 摘要页一份**，那里与 raw 文件 1:1 绑定。漂移经既有 `sources:` slug 图**传播**到引用页，不必各记。
3. **不能让 Agent 写指纹**：`ingest.py:INGEST_PROMPT` 明令 Agent「**不要运行 shell 命令**」，而 LLM 不可能可靠地算/逐字抄一条 sha256。指纹**只能由 wrapper 确定性算**。

### 4.1 字段形态（定稿）

`source` 型页（`wiki/sources/<slug>.md`）增一个**可选标量** frontmatter 字段：

```yaml
raw_digest: 'raw/2024-财报.md@sha256:1a2b3c…'   # 本摘要页所摘 raw 文件 + 被摘时的内容指纹
```

- **单标量**（`<raw相对路径>@sha256:<hex>`）、单引号、路径+hash 自含；**一张 source 页一条**。entity/concept/synthesis 页**不带**此字段。
- 选标量而非 mapping：YAML 最稳（conventions 力推单引号标量），且 source↔raw 恒 1:1、无需 mapping。含路径是为稳健——对 `convert` 改名 / CJK slug 不靠"猜 raw 名"。

### 4.2 谁来写指纹：⚠️ 未决（OPEN，见 未决P3.7-3a）

> **此小节是全文唯一未拍板项。** 下文为求行文具体**暂按 Option A（wrapper 直写）**铺陈，§3/§5/§7/§9 同此假设；若最终取 Option B，这些处的"wrapper stamp/刷新"需相应改为"wrapper 供值、Agent 写 + 回读校验"。两个选项与取舍见 未决P3.7-3a。

（以下按 Option A 描述，**非定论**）

- ingest 成功**过完 P2 门禁后**，wrapper 对本次摄入的 `raw/<stem>.md` 算字节 sha256，stamp 进 `wiki/sources/<stem>.md`（按 conventions 既有 slug==rawname 约定定位；定位不到则跳过、记一句降级，不报错、不阻断）。
- **只 stamp 本次摄入那一张 source 页**——**绝不**顺手刷新别的 source 页指纹（否则会把别处真实漂移一起抹平，是致命 bug）。**此条两个选项都成立**。
- Option A 的取舍：零 LLM、幂等、确定性，但**新增一条 wrapper 对内容页的确定性写**——这正是 未决P3.7-3a 要权衡的越线点。
- `audit` 复核**确认页仍准**后，把该 source 页 `raw_digest` 刷成现值（= swarmvault `resetDecay` 的 provenance 版，零数值衰减），使幂等成立、该页离开候选。**由谁刷新同样取决于 未决P3.7-3a。**

### 4.3 它是 provenance、不是派生索引（守住不变量）

记的是"本摘要页建自该 raw 的**这个版本**"——历史事实，性质同学术引用版次、同 `convert` 写进 raw frontmatter 的 `origin`。它**不需**从当前 markdown 可重建（不是对当前内容的派生），故不违反「markdown 唯一真相 / 派生物可幂等重建」。

### 4.4 raw 不可变，漂移从哪来？（必须讲清的前提）

guanlan 的 raw **只在 agent 写期间被快照硬锁**；但**人手替换 / `convert --overwrite` / Web 投喂覆盖**可在两次 ingest 之间换掉一个 raw 文件。`source-drift` 抓的正是这个事件：**「重转/替换了源、但 wiki 页还没重新综合」**。典型用例:拿更新版 PDF `convert --overwrite` 盖掉旧 `raw/`，`audit` 即告诉你哪些 wiki 页该重摄。若一个库的 raw 永不被覆盖，`audit` 永远空跑——这是**正确**的（无事件即无候选），不是缺陷。

### 4.5 check 契约不变 + 向后兼容

- **`check` 完全不动**：`raw_digest` 是可选未知字段，`check` 既不要求、也不校验它——**不新增 Violation、门禁字节稳定**（决策P3.7-3b）。合法性由 `audit` 自己宽松读（缺/格式坏 → 当无信号跳过），不进门禁。
- 无 `raw_digest` 的旧 source 页 → 无漂移信号，下次 ingest 触及时由 wrapper 自然补 stamp。**安全退化：少报不误报**（决策P3.7-5）。
- `conventions.md` / `examples/SCHEMA.md`：把 `raw_digest` 记为 source 页上**wrapper 托管、请勿手改**的 provenance 字段；Agent 知其存在但不写、不读它做判断。

> 决策P3.7-3 是本阶段**唯一动数据模型**的改动。**已定稿**要点:字段在 source 页、单标量含路径、只 stamp 本次那一张、check 不可见、向后兼容跳过。**唯一未定**:指纹由谁写（wrapper 直写 vs Agent 写）——见 未决P3.7-3a。

## 5. Layer-2：LLM 复核（复用 P2 门禁 + skill 工作流）

`AUDIT_PROMPT`（仿 `HEAL_PROMPT`，薄；真步骤在 skill）：

```
请按 `guanlan-wiki` skill 的 audit 工作流，逐一复核下列「所引的源已变更、论断可能过期」的页面：
{targets}
对每页：读该页正文与它所引 `sources:` 中**已漂移源**对应的 `raw/` 源**现版本**，判断页中论断是否仍被现源支持。
 - 仍准：无需改正文（指纹刷新由 wrapper 在你返回后自动处理，**你不要碰 raw_digest**）。
 - 已过期 / 现源与页冲突：就地按 conventions 标 `## ⚠️ 矛盾与存疑`，或最小化更新该论断（其余正文一字不动）。
 - 向 `log.md` 追加一条 `## [<日期>] audit | <标题>`。
遵循 AGENTAO.md 硬约束。**永不修改 `raw/`。** 不要运行 shell 改文件、不要自跑 check（wrapper 会强制校验）。
完成后用一两句说明哪些页确认、哪些标了存疑。
```

- **指纹刷新不进 prompt**：`raw_digest` 由 wrapper 在门禁后确定性 stamp（§4.2），**不交 Agent**——LLM 抄 hash 不可靠且 ingest 禁 shell。prompt 只让 Agent 做语义判断 + 标注/更新正文。
- 走 `run_guarded_write_result`：Agent 只能写 `wiki/`，`raw/` 由快照前后比对硬拦；既有 `sources:` slug 由 P2.1 源不回退闸保只增不减。
- 空 worklist 短路 `EXIT_OK`、不触 Agentao（仿决策P3.2-6）。
- 复核**就是 ingest 已会的动作**：`ingest.py:INGEST_PROMPT` 里"发现矛盾就地标 `## ⚠️ 矛盾与存疑`"已是现成行为——`audit` 只是把它从"摄入时顺手"变成"可定向、可批量、可复查"。**不教 LLM 新把戏**。

## 6. CLI 契约（逐项对齐 `heal`）

```
guanlan -C <kb> audit            # 复核全部 source-drift 候选（LLM 写，走门禁）
guanlan -C <kb> audit --dry-run  # 零 LLM：只打印候选页 + 漂移的源，不触 Agentao、必 EXIT_OK
guanlan -C <kb> audit --limit N  # 限本批复核页数（worklist 批量上限，仿 heal）
guanlan -C <kb> audit --json     # 稳定 JSON 契约（audit_result_dict，仿 heal_result_dict）
```

- 退出码：复用现有集合——`EXIT_OK` / `EXIT_AGENT_ERROR` / `EXIT_USAGE`（同 ingest/heal）。**不新增退出码**。
- `--dry-run` 是纯读零-LLM 预览，等价 `lint`/`health` 的 advisory 性质——可进 CI/nightly 做"有没有漂移页"的看门，但**默认不门禁**（语义判断不该卡 CI）。
- `audit_candidates()` 也可被 `health` 顺带展示一条 `health.source_drift`（零 LLM、advisory），让用户不跑 `audit` 也能看见漂移——但**复核动作只在 `audit`**（同 missing_entity 在 lint 可见、物化只在 heal）。

## 7. 写后回执 / 幂等 / 不变量

- **写后确定性回执**（零 LLM 可验证，仿 `_build_receipts`）：复核后 wrapper 刷新本批已确认 source 页的 `raw_digest`、再重算 `audit_candidates()`——已刷新的页应**离开**候选集；仍在集内者 = 未被处理，回执如实标"未复核"。不靠 LLM 自述触没触。
- **幂等**：连跑两次 `audit`，第二次 worklist 为空（指纹已由 wrapper 刷新）→ 短路、零写、零 Agentao。raw 再被替换才再进候选。
- **不破坏既有页**：走 P2.1 门禁——既有 `sources` 只增不减、正文骤缩告警全部继承，无需新写防护。
- **markdown 唯一真相**：本阶段零新派生态文件。`raw_digest` 是 source 页内 provenance（§4.3），随页走、随页重建，不构成需独立重建的索引。
- **零 LLM 脚本 vs LLM 工作流**：Layer-1（比 hash + 求 slug 交集）零 LLM；Layer-2（语义判断）全经 Agentao。指纹计算与 stamp 是 wrapper 确定性代码，**不持 LLM client**。红线全守。

## 8. 决策记录

- **决策P3.7-1**：语义 lint **不进 `lint`**，新开 `audit` 命令（LLM 档），与 `heal` 平级。理由：`lint.py:19` 自承 lint 零 LLM 只做结构；把 LLM 判断塞进 lint 会破其教义。`lint`/`health` 至多展示零-LLM 的**触发信号**（`health.source_drift`），复核动作归 `audit`。
- **决策P3.7-2**：本阶段**只跑通"过期论断 / source-drift"一类**作 worked example。理由：它是反向评审里唯一"触发信号纯结构、零 LLM、对 CJK 无偏、低噪"的幸存者；矛盾复检触发更难低噪（§11.1），留兄弟项。
- **决策P3.7-3（已定稿，本阶段唯一动数据模型）**：引入 `raw_digest` provenance——**`source` 摘要页**上的可选单标量 `'<raw路径>@sha256:<hex>'`，记本页所摘 raw 的版本指纹。理由：`sources:` 是指向 source 页的 slug（非 raw 路径），故指纹只该在 source 页一份、经 slug 图传播；它是出处事实非派生索引（§4.3），是 source-drift 可判定的拱心石。
  - **🔴 未决P3.7-3a（OPEN，需拍板才能进实现）**：`raw_digest` **由谁写**？无论哪个选项，都**只 stamp 本次摄入那一张 source 页**、绝不顺手刷新别页（否则抹平别处漂移）——这点已定。未定的是写入主体：
    - **Option A（本草案暂按此行文）——wrapper 门禁后直写 source 页**。优：值由代码算、零腐败、零 LLM。缺：**头一回让 wrapper 碰内容页**（以往 wrapper 只 verify；仅 `reindex` 写过 `index.md` 那种 *config* 页），越过"Agent 写 wiki、wrapper 只验"旧默契一小步。正当性类比 `reindex`、纯 provenance、Agent 结构上做不到（禁 shell + 不会 hash）。
    - **Option B——wrapper 算值塞进 prompt、Agent 用文件工具写 + wrapper 回读校验**。优：守住"Agent 是 wiki 唯一写者"。缺：要 LLM 逐字抄 64 位 hex，**易腐**，须加"写后回读比对、不符则 wrapper 兜底改正或报错"一层。
    - 倾向：A（可靠性优先）。但因其越线，**留作未决待你拍板**；定了再把 §4.2/§5/§7/§9 的"wrapper 写"措辞收敛到所选项。
  - **决策P3.7-3b（已定稿）**：`check` **完全不动**——`raw_digest` 对 check 不可见、不校验、不阻断；合法性由 `audit` 宽松自读。门禁字节稳定、不新增 Violation。
- **决策P3.7-4**：复用 `run_guarded_write_result` 写路径与 P2.1 闸，**不另开门禁、不新增退出码**。
- **决策P3.7-5**：无 `raw_digest` 的旧 source 页**跳过不罚**（向后兼容），随下次 ingest 触及自然补 stamp。
- **决策P3.7-6**：**不引入 claim/polarity/confidence/衰减分**。所有语义判断下放 LLM，wrapper 只比 hash。明确与 swarmvault `analysis.ts` 路线分道。
- **决策P3.7-7**：`audit` 确认页仍准时刷新该 source 页 `raw_digest`（= `resetDecay` 的 provenance 版），使幂等成立、避免同页反复进候选。**刷新由谁执行同受 未决P3.7-3a 约束。**

## 9. 测试要点（仿 `tests/test_heal.py` + `test_lint.py`）

- `audit_candidates()`：raw 改 1 字节 → 其 source 页（`source-drift`）+ 所有引该 slug 的页（`cites-drifted-source`）全进候选 / raw 未动 → 空 / source 页无 `raw_digest` → 跳过不报 / raw 文件被删 → 不抢 `check` 缺源口径 / 多源页只一源漂移 → `drifted_slugs` 只列变的那个 / 候选按 page 升序字节稳定 / **只对 source 页比一次 hash**（断言引用页不重复算 raw hash）。
- wrapper stamp：ingest `raw/X.md` 成功后 → `wiki/sources/X.md` 被写入 `raw_digest`、值 = 该 raw 字节 sha256；**别的 source 页 `raw_digest` 一字不动**（防"顺手刷新抹平漂移"回归）；source 页定位不到 → 跳过、不报错。

- 回执 / 幂等：mock 一次确认写 + wrapper 刷新指纹 → 复算候选该页离开；连跑两次第二次空。
- `--dry-run` 纯读零 LLM：不触 runner、必 `EXIT_OK`、不写任何文件（**不 stamp、不刷新指纹**）。
- `audit_result_dict` JSON 契约字段稳定（仿 `heal_result_dict`）。

## 10. 风险

- **指纹漂移≠语义过期**：raw 改一个错别字也会触发候选 → Layer-2 LLM 复核会判"仍准"、wrapper 刷新指纹，成本是一次 LLM 调用。缓解：`--limit` 批量上限；nightly 而非每次 ingest 跑。可选增"仅当 raw body 变化超 X%"的二阶过滤（但这又滑向语义，倾向不做，让 LLM 兜）。
- **新约定的采纳缺口**：旧库、手写 source 页没 `raw_digest` → 永不进候选（决策P3.7-5）。这是**安全**的退化（少报，不误报），但意味着 P3.7 的价值随 ingest 重写逐步累积，非一上来全覆盖。文档需讲清。
- **provenance 写入主体是未决项**（🔴 未决P3.7-3a）：Option A 让 gate 之后宿主再碰一次 *内容* 页（越"Agent 写 wiki"默契一小步），Option B 守默契但靠 LLM 抄 hash（易腐、需回读校验）。无论哪个，都需 `conventions.md` + `examples/SCHEMA.md` 把 `raw_digest` 标注为"托管字段、勿手改"、并在落地点小心只 stamp 本次那一张。比纯加法 CLI 重 + 含一项越线取舍，是它排在 §1"唯一需先拍板"的原因。

## 11. 后续工作（P3.7 之后，同框架兄弟）

### 11.1 矛盾复检（P3.7.1 — 高价值兄弟，与本阶段共命令共拱心石）

矛盾复检是反向评审认定的**最高价值**项，但 Layer-1 触发信号最难做到"低噪 + 零 LLM"。本节把它展开成一个**可落地的兄弟小设计**：复用 `audit` 同一命令、同一 `raw_digest` provenance、同一门禁，**只在 Layer-1 加一个零成本判别、在 Layer-2 加一条 prompt 分支**。

#### 11.1.0 先承认边界（决定了整套设计的形状）

必须诚实划清：**有一类矛盾，任何零-LLM 结构信号都抓不到——首次 ingest 时就并存、且当时 LLM 没标出来的"静态矛盾"。** 它们本质是语义的，没有任何结构痕迹。

- swarmvault 试图抓静态矛盾：靠一张 claim 图 + polarity + token Jaccard≥0.3 全库两两比。代价是**高噪**（英文正则极性、上限 5000 比较/100 结果）+ **一个 guanlan 红线禁止的 claim 数据模型**。
- guanlan 的**有意取舍**：**放弃**用确定性手段抓静态矛盾，把它留在两道已有防线——(a) ingest 时 LLM 就地标 `## ⚠️ 矛盾与存疑`（`ingest.py:23` 现成行为），(b) 人工/`query` 复核。`audit` 的矛盾复检**只负责变更驱动的那一类**：源变了 / 加了新源之后，新老论断之间**新产生**的冲突。

> 这正是反向评审推崇的姿态：**显式声明约束的代价**，而非假装全覆盖。`audit` 的矛盾复检是"变更触发的增量调和"，不是"全库矛盾扫描"。文档与 `--help` 都要这样讲清，避免给用户"跑完就没矛盾了"的错觉。

#### 11.1.1 三个候选 Layer-1 信号（逐一做噪/召回对抗分析）

- **信号 A —「多源页无矛盾标记」（否决）**：页引 ≥2 源且无 `## ⚠️ 矛盾与存疑` → 疑似"未审视的一致"。**否决**：召回虚高、噪声爆炸——绝大多数多源页本就无矛盾，等于把半个 wiki 报成候选。这就是第一轮我自己批过的"every multi-source page"反模式。

- **信号 C —「多源引用页命中漂移」（采纳，主信号，零新成本）**：复用 §3 第二步已经在产的 `cites-drifted-source` 候选——**当一张候选页本身 `sources:` ≥2 个 slug**时，不仅它论断可能过期，它**跨源综合的结论**也可能因某源被替换而与其余源**新生冲突**。
  - 召回：抓住"源变更引入的矛盾"这一变更驱动核心类。
  - 噪声：**与 source-drift 同档（已低）**——不新增任何候选页，只在既有 `cites-drifted-source` 候选上多读一个"是不是多源"的布尔，把多源的那些**路由到更狠的 prompt**。
  - 成本：**零新 bookkeeping**。`sources` slug 计数本就在 frontmatter，Layer-1 加一个 `multi_source: bool` 即可。这是它当主信号的决定性理由——它**搭 P3.7 拱心石的车，不再动数据模型**。
  - 局限：只覆盖"源**变化**引入"的矛盾，不覆盖首次并存的静态矛盾（见 §11.1.0，那本就不归它）。

- **信号 E —「既有矛盾标记 + 源漂移」（采纳，免费搭车）**：已带 `## ⚠️ 矛盾与存疑` 的页，其引用的源发生漂移 → 旧的存疑结论可能**已被现源解决或加剧**，值得复核"这条存疑是否还成立"。
  - 噪声：极低（**只在已标存疑且源又变的页**触发，集合很小）。
  - 成本：零——`audit_candidates()` 顺手 grep 该 ATX 标题即可。
  - 价值：补 source-drift 的盲区——存疑页的"维护"，让矛盾标记不会标完就烂掉。

- **信号 B —「新源未调和」（推迟，需新 provenance）**：页 `sources:` 相对上次 audit **新增了一个 slug**，但正文**未被该源 ingest 触及** → "加了料没重新综合"。
  - 噪声：低（结构性"未调和"，非"看起来像否定"）。
  - 成本：**需新 provenance**——ingest 要可靠地记"本次触了哪些页"。现状不够：`INGEST_PROMPT` 只保证 `log.md` 追一条 `## [日期] ingest | <标题>` 标题、"触及哪些页"只在**给用户的回执**里（自然语言、不落盘、不可靠解析）。要可靠就得结构化落盘，又一项约定改动。
  - 局限：guanlan **正常** ingest 流里 Agent 本就会更新实体页，"加源不动页"主要出现在**手改/半截 ingest**等非常规路径——召回不如表面高。
  - 结论：**推迟**。等真有非常规流的需求、或 touched-pages provenance 因别的原因落地了，再以同框架接入。当前 C+E 已无新成本地覆盖变更驱动主类。

#### 11.1.2 落地增量（在 P3.7 之上几乎全是加法）

- **Layer-1**：`AuditCandidate` 增两个布尔 `multi_source`（页 `sources:` ≥2）与 `has_conflict_mark`（正文含 `## ⚠️ 矛盾与存疑`）——都零成本读 frontmatter / grep 标题。**`reason` 仍是 §3 的 `{"source-drift", "cites-drifted-source"}`、不新增取值**；矛盾路由靠这两个布尔，不靠扩 reason（避免 reason 维度爆炸）。同一份 `audit_candidates()` 归口产出，**不分叉**。
- **Layer-2**：`AUDIT_PROMPT` 据布尔分支——`multi_source` 或 `has_conflict_mark` 为真的页追加一句："此外，对照该页 `sources:` 指向的源的现版本，判断被替换的源是否与其余源或页中既有综合**新生矛盾**；若是，按 conventions 就地标/更新 `## ⚠️ 矛盾与存疑`。" 单源页走原 §5 过期论断分支。**仍不教 LLM 新把戏**——只是把 ingest 已会的标注动作定向到"跨源对照"。
- **CLI**：无新命令、无新 flag、无新退出码。`audit --dry-run` 的候选行附 `reason` + 两布尔即可区分疑点类别。

#### 11.1.3 决策与测试增量

- **决策P3.7.1-1**：矛盾复检**只覆盖变更驱动类**，显式放弃用确定性手段抓静态矛盾（留给 ingest 内联标注 + 人工）。`--help`/文档必须讲清边界。
- **决策P3.7.1-2**：主信号取 **C（多源页 + 源漂移）**，因其**零新 bookkeeping、噪声与 source-drift 同档**；E 免费搭车；**B 推迟**（需 touched-pages provenance）。
- **决策P3.7.1-3**：**仍不引入 claim/polarity/Jaccard**（决策P3.7-6 的延续）。跨源矛盾判断整体下放 Layer-2 LLM；Layer-1 只贡献"哪些漂移页是多源 / 带存疑标记"的零-LLM 路由。
- 测试增量（仿 §9）：多源引用页命中漂移 → `reason="cites-drifted-source"`、`multi_source=True` / 单源页命中 → `multi_source=False` / 带 `## ⚠️ 矛盾与存疑` 且命中漂移 → `has_conflict_mark=True` / **无漂移的多源页 → 不进候选**（验证 C 不退化成信号 A 的全多源页洪水）/ Layer-2 prompt 按布尔正确分支（mock runner 断言多源候选的 prompt 含跨源对照句、单源候选不含）。

### 11.2 资料缺口
大面积与 `lint.missing_entity` / `health.stub_page` 重叠，仅在"被 ≥N 页引用且正文薄"叠加时有增量。倾向并入既有 finding 的阈值微调，不单列。

### 11.3 把 `audit --dry-run` 接入 nightly / `health`
零-LLM 的 `health.source_drift` 一行，让用户不跑 LLM 也看得见漂移面，是最便宜的接入点。
