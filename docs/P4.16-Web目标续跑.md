## 1. 目标与范围

**P4.16 = 让 Web 会话能挂一个长任务目标，由宿主拥有的 `while goal.is_active` 外层循环反复驱动 `Conversation.turn()`，
每轮注入续跑 prompt、host 侧记时间/轮数预算、撞预算跑恰好一次收尾轮；agent 仅能经注入的 `update_goal` 工具自报
`complete`/`blocked`，用户经 `/goal` 命令/端点 create/pause/resume/clear/edit/budget，host 循环置 `limit_reached`——
**最大化复用 agentao 已落地的 `GoalState`/`parse_duration`/`UpdateGoalTool`**，guanlan 只建 Web 编排层（异步续跑循环 +
SSE 帧 + 端点 + 前端），不重造状态机、不动 agentao 内核、不走 `force_continue`。**

### 1.1 现状（已核对 agentao v0.4.13 源码 + guanlan 源码）

**agentao 侧（已实现）**：`/goal` 是 CLI 的 host-owned 续跑循环（`cli/input_loop.py::run_goal_continuation`），由三件**可复用原语**搭成：

- `agentao.cli.goal_state.GoalState`：dataclass + 状态机（`mark_complete`/`mark_blocked`/`mark_limit_reached`/`pause`/
  `resume`/`reactivate_from_limit`）+ 谓词（`is_active`/`is_stopped`/`budget_tripped`/`status_label`）+ `to_dict`/`from_dict`
  + 显示 helper（`format_duration`/`budget_summary`）。**纯模块、零副作用**。
- `agentao.cli.duration.parse_duration` + `DurationParseError`：`90s|30m|2h|1h30m → 秒`，拒无单位裸数字。纯模块。
- `agentao.tools.goal.UpdateGoalTool(goal, on_change)`：agent 写 goal 状态的**唯一**工具，`is_read_only=True`，active-only
  守卫。**刻意 duck-type**（只要 goal 有 `is_active`/`mark_complete`/`mark_blocked`/`status_label`），「在 `tools/`、不依赖
  cli 层」——明摆着是给别的 host 复用的。经 `agent.add_tool(tool, replace=True)`（`agent.py:847`）注入、`remove_tool`
  （`agent.py:886`）摘除，**loop 作用域**（非构造期注入）。

**guanlan Web 侧（现状）**：对话路径**一次 HTTP POST = 恰好一轮**：

- `POST /api/chat`（`app.py:955`）建/恢复会话 → `_run_turn`（`app.py:1010`）`begin_turn` → `emit("start")` →
  `turn_task = ensure_future(conv.turn(...))`（`app.py:1030`）→ `await shield(turn_task)` → `emit("done")` → `end_turn`。
- `Conversation.turn()`（`conversation.py:408`）持 `self.lock` 跑**一次** `agent.arun()`（`conversation.py:500`），
  `agent.messages` 跨轮累积——这是「驱动多轮 turn」原语，guanlan **已在用**。续跑上下文注入 = 把字符串当 `msg` 传给
  `turn()`（已具备）；工具注入 = `extra_tools`（`conversation.py:148-151`，现注入 `guanlan_search`）或运行期 `add_tool`（已具备）。

**结论**：三件套齐备 + agentao 已把状态机/解析器/工具做成可复用件 ⇒ guanlan **只需把「一次请求一轮」升级为「一次请求一个
目标续跑流」**，加一份 per-conversation 持久化、一条异步续跑循环、一组命令/端点/SSE 帧。**零 agentao 改动**。

### 1.2 纳入 / 不纳入 P4.16

## 2. GoalState：复用 agentao 的状态机（不重造）

**复用 `agentao.cli.goal_state.GoalState` / `GoalStatus`**（§1.3）。字段（实测）：

```python
# agentao/cli/goal_state.py（复用，不在 guanlan 重定义）
objective: str
goal_id: str                          # uuid hex
status: GoalStatus                    # ACTIVE|PAUSED|COMPLETE|BLOCKED|LIMIT_REACHED
time_budget_seconds: int | None       # None = 无时间上限
max_turns: int | None                 # None = 无轮数上限
time_used_seconds: float              # 累计**活跃**墙钟（不含 paused）
turns_used: int                       # 续跑轮数
created_at: str; updated_at: str       # ISO（UTC）
```

状态机（agentao 实测，比初稿更全——`resume` 也复活 blocked、`reactivate_from_limit` 由 re-budget 触发）：

```
   active ──mark_complete──────▶ complete        (终态)
   active ──mark_blocked───────▶ blocked          (休眠；resume 复活)
   active ──mark_limit_reached─▶ limit_reached    (终态，至 re-budget/clear)
   active ──pause──▶ paused ──resume──▶ active
   blocked ────────resume────────────▶ active
   limit_reached ──reactivate_from_limit(re-budget 有余量)──▶ active
```

| 转移 | 谁 | guanlan 落点 |
|------|----|------|
| create / pause / resume / clear / edit / **budget** | **user** | `/goal*` 命令 → 端点（§8） |
| complete / blocked | **agent**（仅，且仅经 `update_goal`） | 复用 `UpdateGoalTool`（§3） |
| limit_reached | **host 循环**（自动） | `run_goal` 预检调 `goal.mark_limit_reached()`（§4） |

- **预算判定复用 `goal.budget_tripped()`**（首到先触发：`time_used_seconds >= time_budget_seconds` 或
  `turns_used >= max_turns`），guanlan **不**自己写预算比较。
- **挂在 `Conversation`**：`self._goal: GoalState | None`，与 `_confirm_mode`/`_pending` 并列；护以**两把 per-conversation 锁**——
  `self._goal_lock`（`threading.Lock`）串行 `GoalState` 的**一切内存读写**（`run_goal` 循环、`/goal*` 端点、`UpdateGoalTool`
  回调，§3 并发模型），`self._goal_io_lock` 串行 sidecar **文件写盘**（§10）。executor 线程（工具）、loop 线程（端点/循环）
  真并发，故二锁皆正确性所必需、非可选（决策P4.16-7/14/16）。
- **per-conversation 持久化**：见 §10（**不**用 agentao 的单文件 `save_goal`/`load_goal`）。

## 3. `update_goal` 注入工具：复用 `UpdateGoalTool`，loop 作用域注入

**复用 `agentao.tools.goal.UpdateGoalTool(goal, on_change)`**（§1.3）——agent 写 goal 状态的唯一通道，无需在 guanlan 重写。

- **签名**：`UpdateGoalTool(goal, on_change=persist)`。`on_change` = guanlan 的 per-conversation 持久化回调（§10）。
  工具 duck-type 只调 `goal.is_active`/`mark_complete`/`mark_blocked`/`status_label`——**正是这点让 guanlan 能塞一个
  lock-aware 代理进去**（见 §3.1）。
- **`is_read_only=True`（agentao 已内建）**：它不写盘/不跑 shell、只改内存 goal 状态。这使**只读会话**也能用——否则
  `tool_planning._decide`（`readonly_mode and not is_read_only → DENY`，镜像 `chat_support.py:70-84`）会把它 DENY，
  研究型 goal 失效（agentao 注释明示同款考量）。
- **active-only 守卫（agentao 已内建）**：终态/暂停后 `execute` 返回错误串、no-op——保护终态 `limit_reached` 不被 wrap-up 轮里
  迟到的 `update_goal` 覆盖。
- **注入方式 = loop 作用域 `add_tool`/`remove_tool`，不是构造期 `extra_tools`**（与初稿的重要纠正，对齐 agentao
  `run_goal_continuation`；prior 快照 + finally 还原/摘除见 §4 `run_goal`）。**为何 loop 作用域**：`update_goal` 只在 goal
  运行期对模型可见——非 goal 的普通问答不该看到它（否则模型可能在无目标时乱调）。`guanlan_search` 会话常驻、goal 工具
  goal 常驻，注入时机不同。

### 3.1 并发模型：`_GoalProxy` + 单锁（评审 Medium #4）

`UpdateGoalTool` 在 **executor 线程**（arun 期）直接改 `GoalState`，而 `/goal*` 端点在 **loop 线程**改同一对象——真并发。
若把裸 `self._goal` 传给工具（初稿），`_goal_lock` **只**串行了端点与循环、**没**串行 agent 的写，锁形同虚设。

**修正：传一个 lock-aware 代理**（利用工具的 duck-type 面）：

```python
class _GoalProxy:                       # 只暴露 UpdateGoalTool 调的 4 个面，各自持 _goal_lock
    def __init__(self, goal, lock): self._g, self._lock = goal, lock
    @property
    def is_active(self):
        with self._lock: return self._g.is_active
    def mark_complete(self):
        with self._lock: return self._g.mark_complete()
    def mark_blocked(self):
        with self._lock: return self._g.mark_blocked()
    def status_label(self):
        with self._lock: return self._g.status_label()
```

**单锁规则（不变量⑨）**：`run_goal`、端点、工具代理对 `GoalState` 的每次读写都在 `_goal_lock` 内。残留的窄 TOCTOU——
`UpdateGoalTool.execute` 内 `is_active` 检查与随后 `mark_complete()` 之间不是一个临界区（那是 **agentao 代码**、动不了）——
**是良性的**：所有转移都 active-only 守卫（`mark_*` 在非 active 时 no-op），故任何交错都**不产生非法状态**，只决定「agent 标
complete」与「用户同时 pause」谁赢；循环每轮收尾在锁内**重读** `g.status` 据实收场（§4），不依赖该检查的原子性。

## 4. host-owned 续跑循环（`Conversation.run_goal()`，镜像 `run_goal_continuation`）

把 async 续跑循环封进 `Conversation`、与 `turn()` 对称。**`run_goal` 是 `_run_turn` 单轮 shield 逻辑的多轮推广**——
它**复用现有 `turn()` 跑每内层轮**（锁/写门禁/P4.15 确认/落盘全继承），且**每内层轮必须照搬 `_run_turn` 的
`ensure_future + asyncio.shield` 不变量**（评审 High #1/#2），结构镜像 agentao `run_goal_continuation`：

```python
async def run_goal(self, first_msg: str, emit: Emit, meta_out: dict | None = None,
                   first_images: list | None = None) -> None:   # first_images：仅首轮（turns_used==0）带（§14 #10）
    from agentao.tools.goal import UpdateGoalTool
    from agentao.cli.goal_state import GoalStatus, budget_summary   # budget_summary 纯读字段、不取锁
    g = self._goal                                  # 活动期 g 恒为 live 对象：active 时 clear/替换一律 409（§8.2），
                                                    # pause/resume/budget 改的是同一对象（持 _goal_lock，§3）
    prior = self._snapshot_prior_update_goal()      # 见 §3：prior 快照
    # 工具收到 **_GoalProxy(g, _goal_lock)**，使 agent 的 mark_* 与端点 pause/budget 同锁串行（§3，评审 Medium）
    self.agent.add_tool(UpdateGoalTool(_GoalProxy(g, self._goal_lock), self._persist_goal), replace=True)
    idx = 0
    turn_task = None
    try:
        while True:
            with self._goal_lock:                   # 状态读 + 快照都在锁内（评审 Medium #1：绝不锁外读 g.*）
                if not g.is_active: break
                tripped = g.budget_tripped()
                if tripped: g.mark_limit_reached()
                objective, turns_used = g.objective, g.turns_used     # ← 快照，prompt/帧只用快照、不碰 g
            if tripped: self._persist_goal()        # _persist_goal 自取 _goal_lock 快照后再写盘（§10，绝不锁内 IO）
            prompt = (WRAP_UP_PROMPT(objective) if tripped
                      else objective if turns_used == 0 else CONTINUATION_PROMPT(objective))
            # 首轮（仅 turns_used==0、非 wrap-up）带附件；后续轮纯文本（对齐 agentao，§14 #10）。
            images = first_images if (turns_used == 0 and not tripped) else None
            emit("turn_start", {"idx": idx, **({"kind": "wrap_up"} if tripped else {})})
            t0 = self._clock()
            turn_task = asyncio.ensure_future(self.turn(prompt, emit, meta_out, images=images))  # ← 每内层轮显式 task
            try:
                await asyncio.shield(turn_task)     # ← shield：断线 cancel 不穿透进 turn()/arun（评审 High #1）
            except AgentCancelledError:             # 停止/令牌被 trip：inner turn 已收尾（done），自身 finally 释写锁
                with self._goal_lock:
                    if g.is_active: g.pause()
                self._persist_goal(); emit("turn_done", {"idx": idx, "stopped": True}); break
            except asyncio.CancelledError:          # 客户端断线：cancel 打到 run_goal 的 await（shield 处）
                with self._goal_lock:
                    if g.is_active: g.pause()
                self._persist_goal()                # 同步落盘，保证收为 paused（评审 High #2）
                # ★ 排空仍在后台的 inner turn，**再**让 finally 动工具注册表（评审 High：tool 增删须「无 in-flight 轮」时）
                await asyncio.gather(turn_task, return_exceptions=True)
                raise                               # 流已断，向上传播
            with self._goal_lock:
                g.time_used_seconds += self._clock() - t0
                done = False
                if not tripped:                     # wrap-up 轮只计时间、不计轮数（对齐 agentao）
                    g.turns_used += 1
                    done = g.status in (GoalStatus.COMPLETE, GoalStatus.BLOCKED)  # update_goal 本轮置位
            self._persist_goal(); emit("turn_done", {"idx": idx})
            if tripped or done: break               # 收尾轮跑完即停；agent 标终态即停
            idx += 1
    except Exception:                               # 轮内异常 → 暂停 active 目标再抛（对齐 agentao）
        with self._goal_lock:
            if g.is_active: g.pause()
        self._persist_goal(); raise
    finally:
        # tool 增删规则：**仅在无 in-flight 轮时**（评审 High）。CancelledError 路径已在上面 gather 排空；其余路径
        # turn_task 在离开 try 前 shield 已 resolve（done）。兜底再判一次 done，绝不在 inner turn 仍跑时碰注册表。
        if turn_task is not None and not turn_task.done():
            await asyncio.gather(turn_task, return_exceptions=True)
        self._restore_or_remove_update_goal(prior)  # 还原 prior / remove_tool（loop 作用域，§3）
    with self._goal_lock:                           # goal_done 帧的字段也在锁内快照（评审 Medium #1）
        final = {"status": g.status_label(), "summary": budget_summary(g)}
    emit("goal_done", final)
```

> **为何每内层轮都要 shield（评审 High #1/#2 的根因）**：`_run_turn` 现状用 `await asyncio.shield(turn_task)` 杜绝「客户端断线
> cancel **穿透**进 `Conversation.turn()`、而 `agent.arun()` 仍在 executor 线程跑」的竞态——若穿透，`turn()` 的 `async with
> self.lock` 会在 executor 线程还在改 `agent.messages` 时被释放，下一轮即与残线程并发（`conversation.py:1022-1029` 注释）。
> 初稿 `await self.turn(...)` **丢了这个 shield**，把竞态重新引入。修正：`run_goal` 每内层轮自建 `turn_task` 并 `shield`，
> 让 cancel 只打到 `run_goal` 的 await（其 `except CancelledError` 里 pause+**同步**落盘 → `await gather(turn_task)` 排空
> inner → 传播），inner turn 则在后台干净收尾、由 `turn()` 自身 finally 释写锁。**`AgentCancelledError`（停止/令牌 trip）
> 经 shield 正常上抛**（shield 只挡外部 cancel、不挡 inner task 自身异常）→ 走 `except AgentCancelledError` 分支。这样断线
> **必然**把 goal 落为 `paused`（修评审 High #2：初稿 `task.cancel()` 会让 `run_goal` 收到 `CancelledError` 而非
> `AgentCancelledError`、`pause` 落盘被跳过）。
>
> **排空先于动注册表（评审 High，本轮新增）**：`update_goal` 经 `add_tool`/`remove_tool` 增删，而 agentao 运行时**在两轮
> 之间增删、执行期按 live 注册表解析工具**——若 `finally` 在 shield 的 inner turn **仍在后台跑**时就 `remove_tool`，一个迟到
> 的 in-flight `update_goal` 调用会落空 / 与注册表竞态。故断线路径在 `pause`+落盘后**先 `await asyncio.gather(turn_task,
> return_exceptions=True)` 排空 inner turn**（其令牌已被 `request_stop` trip、promptly 收尾），`finally` 再 `remove/restore`；
> `finally` 另兜底「`turn_task` 未 done 则先 gather」。`gather(return_exceptions=True)` 一并消费 inner 终态异常，免「Task
> exception never retrieved」噪声（取代初稿的 `add_done_callback`）。

不变量（developer-guide part-4/8.2 四条 + guanlan 追加）：① 首轮 objective、续轮 continuation（gate 在 `turns_used == 0`）；
② 预算**轮前**检、撞顶产**恰好一次** wrap-up（wrap-up 只计时间不计轮数）；③ 注入工具是 agent 唯一写、active-only 守卫；
④ 工具 loop 作用域（finally 还原/摘除）、**且增删前 inner turn 必须 done（断线路径先 `gather` 排空）**（评审 High）；
⑤ 轮内中断（停止/断线）→ `pause()`；⑥ 轮内异常 → `pause()` active 目标再抛；⑦ **每轮 persist**；⑧ **每内层轮
`ensure_future + shield`**（Web 取消不变量，本节根因框）；⑨ **所有 GoalState 读写过 `_goal_lock`**（loop / 端点 / 工具代理
同锁、prompt 与帧字段先在锁内**快照**再用，§3、评审 Medium #1）；⑩ **`_persist_goal` 经专用 `_goal_io_lock` 串行「锁内快照
`to_dict()` → 释 `_goal_lock` → 临时文件 `os.replace` 原子写盘」**（绝不锁内 IO、绝不锁外读、**绝不两线程并发写同一 sidecar**，
§10、评审 Medium #3 + 本轮）。

- **逐轮取/放锁**：每内层 `turn()` 各自取/放 `self.lock` 与（workspace-write）`write_lock`，**绝不跨整个 goal 持锁**（§6）。
- **`begin_turn`/`end_turn` 只在最外层包一次**：整个 goal run 期 `_inflight=1`，使停止按钮能打到当前内层轮的 `_cancel_token`、
  idle 回收不淘汰正在跑 goal 的会话（§9）。`_run_turn` 的 goal 分支**只 `await conv.run_goal(...)`**——inner turn 的 shield/
  consume 全归 `run_goal`（它持 `turn_task` 句柄），`_run_turn` 自身不再另持单轮 `turn_task`。
- **首轮附件（决策P4.16-18，对齐 agentao `_staged_images_payload`）**：`first_images` 来自触发 `/goal` 的那次 `/api/chat`
  请求（guanlan Web 附件随请求体来，**非** CLI 的 `/image` 暂存），**仅首轮带**。**「成功才算消费」语义靠 `turns_used`**：
  首轮成功 → `turns_used` 0→1，下轮 `images=None`；首轮被中断/失败 → `turns_used` 仍 0，`/goal resume` 的首轮**又是 turn 0**、
  需再带附件 → **前端在 resume 时据 `info().goal.turns_used==0` 重新随 `/api/chat` 附上原附件**（与 agentao「中断首轮保留
  staged images 给 resume」同形）。附件 base64 **不入 GoalState/sidecar**（沿用 `_lean_messages` 不变量）。

## 5. 停止、取消、断线（Web 特有，已据 agentao 实测简化）

P4.16 引入「一次请求跨多轮」。停止/断线语义对齐 agentao（中断 → `pause()`），但映射到 guanlan 的取消令牌机制：

### 5.1 停止按钮断「整个 goal」——靠现有取消令牌，无需新标志（纠正初稿）

初稿臆造了 `_goal_stop_requested`。实测 guanlan 现有机制已够：

- **轮内停**：停止按钮 → `request_stop()`（`conversation.py:649`）cancel 当前 `_cancel_token` → `arun` 抛
  `AgentCancelledError` → `run_goal` 的 `except AgentCancelledError` 捕获 → `g.pause()` → break。**整个 goal 收为 paused。**
- **轮间停**（`_cancel_token` 已清、`_inflight>0`）：`request_stop` 置 `_stop_requested`（`conversation.py:669`）→
  **下一内层 `turn()` 进锁即自兑现**（`conversation.py:492-494`：`if self._stop_requested: token.cancel()`）→ `arun`
  **立即**抛 `AgentCancelledError` → 循环 break。即「下一轮秒取消」天然让循环停下，**无需独立 goal 停止标志**。
- 可选优化：`run_goal` 每轮顶部读 `self._stop_requested` 提前 break，省一个「秒取消」的空轮——锦上添花、非必需（决策P4.16-4）。

### 5.2 取消令牌复用 P4.15 骨架

内层每轮仍走 `turn()` 锁内装/卸 `_cancel_token`（`conversation.py:488-494`）。停止/超时/断线对**当前内层轮**的打断与
P4.15 **逐字一致**（confirm/ask 等待期短轮询读同一令牌，`conversation.py:208-225`）——goal 只在内层轮被取消后**额外
决定不再续**（`except` 里 `pause()` + break）。无新取消机制。

### 5.3 断线即停 goal（v1）

P4.15 已钉死断线执行序（`app.py:1083-1097`：有 pending 先 `request_stop()` trip 令牌、再 cancel 外层 task）。P4.16
**复用且扩展**：goal 运行期断线，**只要在 goal 中就 `request_stop()`**（cancel 当前轮 + 置 `_stop_requested`），再 cancel 外层。

```python
# event_stream finally（app.py:1093 起，P4.16 扩展）
if conv.in_goal() or conv.has_pending():   # goal 运行中 或 有未决 pending
    conv.request_stop()                    # trip 令牌：inner turn 抛 AgentCancelledError、自身 finally 释写锁
if not task.done(): task.cancel()          # cancel 外层 → 穿到 run_goal 的 await shield 处
await asyncio.gather(task, return_exceptions=True)
```

**「收为 paused」由谁保证（评审 High #2 的修正落点）**：**不**靠这里的 `request_stop()`+`task.cancel()` 顺序，而靠
`run_goal` **自己**的 `except (AgentCancelledError | asyncio.CancelledError)` 分支在**传播前同步** `pause()`+`_persist_goal()`
（§4 框）。原因：`task.cancel()` 会让 `run_goal` 收到 `CancelledError`（非 `AgentCancelledError`），若只在 `except
AgentCancelledError` 里 pause，断线路径就漏 pause——故 `run_goal` **两个 except 都 pause+落盘**。两条进入路径都收敛到
paused：① `request_stop` 先 trip → inner 抛 `AgentCancelledError` 经 shield 上抛 → `except AgentCancelledError`；
② `task.cancel` 先到 → `await shield` 抛 `CancelledError` → `except CancelledError`（消费 inner 终态异常 + pause + 落盘 + 传播）。

- **v1 停而非后台续**（决策P4.16-2）：后台 detached 续跑要把 `run_goal` 从 SSE 请求生命周期解绑 + 重连重挂 SSE 回放，
  是另一体量，留 v2（§14）。v1 语义清晰：**关标签页 = 暂停目标**，重开续聊点「恢复」即继续（§10）。与 agentao CLI 的
  Ctrl-C → pause → `/goal resume` 同形。
- 窄竞态由预算兜底（最坏多跑一轮即被顶部 `budget_tripped` 或下一轮取消拦下）。

## 6. 写锁、并发与层③（workspace-write goal）

**workspace-write goal 逐轮取/放写锁，绝不跨整个 goal 持锁**（决策P4.16-5）：

- 每内层 `turn()` 在 workspace-write 下**整轮持** `write_lock`（`conversation.py:454-521`，覆盖整个内层 `arun`），且
  层③ 在可写 turn 活跃期间 423 拒宿主写端点。故 ingest/heal/投喂/别的写者**不能在某一内层轮进行期插入**，**只能在两个
  相邻 goal 轮之间的窄缝**（`run_goal` 轮间不持 write_lock 的那一瞬）抢到锁。代价：长 goal 期写锁被**高频但非连续**占用，
  其它写者排队偏久——单用户本地可接受，远好于「跨整个 goal 持锁导致全站写死」。
- **`_reject_if_writable_active` 纳入「活跃可写 goal」、且口径为进程级（决策P4.16-20，§14 #12）**：现状（`app.py:957-993`）
  按「会写的 turn」拒嵌套可写 turn。P4.16 把「正在跑的可写 goal」也算 writable-active，且判定**扫全体会话**
  （`any(c.in_goal() and c.mode == "workspace-write" for c in conversations)`，**非**仅本 conv）——**进程内至多一个活跃可写
  goal**。否则会话 A 的 goal 轮间窄缝里，会话 B 可起第二个可写 goal、二者争 `write_lock` 破坏单写者语义。**只读 goal 不限**
  （不持写锁，多个并存无妨）。
- **层①②③逐轮生效**：每内层可写轮照拍 AGENTAO.md 快照（层②）、记写日志、写后 check、撤销可用性
  （`_finalize_writable`，`conversation.py:526`）。goal 每一轮都有 P4.5 全套写守卫——goal 不放宽任何边界（§9）。
- **read-only goal 不持写锁**：研究型目标每轮只占 `conv.lock`，比 workspace-write goal 轻得多。

## 7. SSE 帧与 `_run_turn` 分支

goal = **一条 SSE 流跨多轮**。在现有帧外加**轮边界帧**，让前端把每内层轮渲成独立气泡 + 一个 goal 横幅：

| 帧 `kind` | data | 时机 |
|------|------|------|
| `goal_start` | `{goal_id, objective, status, time_budget_seconds, max_turns}` | `run_goal` 开跑 |
| `turn_start` | `{idx, kind?: "wrap_up"}` | 每内层轮起跑 |
| `token` | （字符串 chunk，复用现有） | 内层轮流式 |
| `turn_done` | `{idx, answer_html?, check?, undo?, stopped?}` | 每内层轮收尾（含 P4.5 meta，§4） |
| `goal_done` | `{status, summary}` | 循环收场（complete/blocked/limit_reached/paused；summary=`budget_summary(g)`） |

- **`confirm_request`/`ask_request` 原样复用**（§9）：goal 里某可写轮触发确认气泡，与 P4.15 单轮**完全一样**，自动组合。
- **`_run_turn` 加分支**（`app.py:1010`）：会话有 active goal 且本次是 goal 触发 → 走 `conv.run_goal(first_msg, emit, meta)`；
  否则走原单轮路径。`begin_turn`/`end_turn`/哨兵/断线 finally（`app.py:1016/1059/1093`）**只在最外层包一次**。
- **`answer_html`** 每 `turn_done` 渲一次（复用 `render_markdown`，`app.py:1037`）。**零流式改造**（同决策P4.15-3）。

## 8. `/goal` 命令与端点（镜像 agentao 命令面）

### 8.1 命令面（镜像 `agentao/cli/help_text.py` + `cli/commands/goal.py` 实测）

| 输入 | 动作 |
|------|------|
| `/goal` 或 `/goal show` | 显示当前目标（status/objective/`budget_summary`：`3/10 turns, 12m/120m`） |
| `/goal <objective> [--for 30m] [--turns 10] [--unbounded]` | 设目标 → `POST /goal` → **触发续跑 SSE 流**；**active 目标存在 → 409 先 pause**（决策P4.16-15/19），paused/终态可直接替换 |
| `/goal budget [--for <d>] [--turns <n>]` / `/goal budget --clear` | 改/清 live 目标的上限；limit_reached 且新上限有余量 → `reactivate_from_limit` + 续跑 |

- **子命令分类对齐 agentao `_classify`**：`show/pause/resume/clear` 是**无参**子命令（`/goal clear out temp files` 当
  **objective**、非 clear）；`edit/budget` 取参；其余当 `set`（新 objective）。**flag 解析复用 `parse_duration`**（`--for`），
  `--turns` 解析正整数；guanlan 重写 ~20 行（不 import `cli.commands.goal`，§1.3）。
- **设目标/恢复驱动 SSE，其余瞬时**（决策P4.16-8）：`/goal <obj>` 与 `/goal resume`（及 re-budget 复活 limit_reached）
  **驱动续跑 SSE 流**（复用 `submitChat`，非 `noteLines`）——它们是「开始多轮目标」；`show`/`pause`/`clear`/`edit`/`budget`
  （非复活）是瞬时端点 + `noteLines`。受控偏离 P4.4-1（斜杠命令不进 chat）。

### 8.2 后端端点（瞬时、loop 线程、绝不取 write_lock/conv.lock）

```
POST /api/chat/{id}/goal          body: {objective, for?, turns?, unbounded?}  → 建/替换 GoalState；瞬返（active 时 409，见下）
POST /api/chat/{id}/goal/budget   body: {for?, turns?, clear?}                 → 改上限（reactivate_from_limit 若有余量）
POST /api/chat/{id}/goal/pause    → 置 paused **且** request_stop()（打断在飞轮 / 拦轮间起新轮，见下）
POST /api/chat/{id}/goal/resume   → paused/blocked → active；前端随后发 /api/chat 触发续跑流
POST /api/chat/{id}/goal/clear    → active 时 409（先 pause）；非活动时删 GoalState + sidecar
GET  内含在 /api/chat/{id}/info   → info() 加 "goal" 段（§8.3）
```

- **锁纪律同 P4.15**：goal 端点只取 `_goal_lock` 瞬改 GoalState，**绝不**取 `conv.lock`（在飞轮持有）/`write_lock`
  （否则与续跑 goal 死锁）——同 `/confirm`/`/answer` 的 loop 线程瞬返纪律（决策P4.15-5/9）。
- **写权限三分由端点/工具/循环分别落地**（§2 表）：端点 = user 面；`UpdateGoalTool` = agent 面；`run_goal` 预检 = host 面。
  端点**绝不**置 complete/blocked；工具**绝不**置 paused/limit_reached。
- **`pause` 必须同时 `request_stop()`（评审开放问题 #2 = 是）**：只置 `status=paused` 不够——在飞内层轮可能很长，单改状态要
  等本轮自然结束才生效。故 pause = 锁内 `g.pause()` **+** `request_stop()` 打断当前 `arun`：inner 抛 `AgentCancelledError` →
  `run_goal` `except` 见 `g.is_active` 已为 False（pause 在先）→ 不重复 pause、直接 break。轮间 pause 则下一轮顶部锁内
  `if not g.is_active: break`（§4）即拦下、不起新轮。
- **`clear`/替换 在 active 时一律 409（评审 High #3 + 开放问题 #1 = 否，不允许 active 时直接 clear）**：避免「端点删 `self._goal`，
  而 `run_goal` 仍持 live `g` 继续驱动/落盘」的悬挂引用。**用户须先 `/goal pause`（停循环）再 `clear`/设新目标**。这把
  「活动期 `self._goal` 不被换/删」变成硬不变量——故 §4 循环里的 `g` 恒等于 `self._goal`（pause/resume/budget 改的是同一对象、
  不换对象），无 stale-`g` 问题。终态/暂停目标（`is_stopped` 或 paused，即非 active）可直接 `clear`/替换。
  *（备选：active 时 clear 做 stop-and-drain——pause()+request_stop() 后由循环退出时删 sidecar；v1 不取，留 §14。）*
- **设/恢复 goal 时本会话不得有在飞 turn（决策P4.16-19，§14 #11）**：`/goal <obj>`·`/goal resume`·re-budget 复活这些
  **驱动 SSE 续跑流**的端点，若本会话 `_inflight > 0`（已有普通 turn 或 goal 在跑）→ **409**「先停当前再设目标」。否则同会话
  两条 SSE 流抢 `conv.lock`/串 `begin_turn`/`_inflight` 语义。瞬时端点（`show`/`pause`/`clear`/`edit`/`budget` 非复活）不受此限。
- **loop-blocked 守卫**（镜像 agentao `_loop_blocked_reason`）：`goal.enabled=false` → 拒；guanlan 无 plan 模式，故省那条；
  可写 goal 还须过进程级 `_reject_if_writable_active`（§6、决策P4.16-20）。

### 8.3 `info()` 自省加 `goal` 段

`Conversation.info()`（`conversation.py:693`）加一段（持 `_goal_lock` 读快照 `goal.to_dict()`），喂 `/goal show` 与
断线/重启重渲染：`"goal": null | {goal_id, objective, status, time_budget_seconds, max_turns, time_used_seconds, turns_used}`。

## 9. 与 P4.5 / P4.9 / P4.15 的关系（各管各，goal 不放宽任何边界）

| 既有机制 | 与 P4.16 的关系 |
|------|------|
| **P4.5 层①②③ + 写后 check + 撤销** | **逐轮生效**：goal 每内层可写轮照拍快照/记写日志/写后 check/登记撤销 token（§6）。goal **不**给 agent 开新写路径——「自动续跑」只是多跑几轮，每轮守卫一个不少（决策P4.16-6）。 |
| **P4.5 单写者写锁** | 逐轮取/放，绝不跨 goal 持锁（§6）。`_reject_if_writable_active` 纳入「活跃可写 goal」。 |
| **P4.9 多会话 + idle 回收** | 正在跑 goal 的会话 `_inflight>0`，`is_idle`（`conversation.py:389-395`）已排除 → **天然不被回收**，无需额外改。paused/终态 goal 会话回到可回收（其 GoalState 已落 sidecar，§10）。 |
| **P4.15 confirm/ask 人在环** | **完全复用、自动组合**：goal 内某可写轮触发 ASK shell → 弹 confirm 气泡（与单轮一致）；`confirm_mode=auto` 下 goal 一路静默放行（仍 workspace-write + 层①②③，非 full-access）。confirm 等待期 goal 循环自然阻塞在该内层轮的 `arun` 上。 |
| **P4.15 断线释锁** | 扩展：goal 运行期断线即 `request_stop()` 收为 paused（§5.3），不只「有 pending 才 trip」。 |

**一句话定位**：P4.16 是**宿主层外层续跑循环**，坐在 P4.5/P4.15 的每轮守卫之上——它决定「**跑几轮、何时停**」，
**不**决定「**能写到哪、要不要问人**」（那是层①②③与 confirm 的事，goal 一个都不松）。

## 10. 持久化（per-conversation sidecar——与 agentao 单文件的关键分歧）

**agentao 是每 KB 单文件** `<root>/.agentao/goal.json`（CLI 单会话），其 `save_goal`/`load_goal`/`goal_path` 硬编码该路径。
**guanlan Web 多会话**（`ConversationStore`，可并发多个 goal），故**不能用** agentao 的单文件 IO，须 **per-conversation**：

- **落 `<kb>/.agentao/goals/<conversation_id>.json`**（与 `.agentao/sessions/` 并列，已 `.gitignore`、扫描只看 `*.md`）。
  **复用 `GoalState.to_dict`/`from_dict`** + guanlan 自己的 ~5 行 read/write（`self._persist_goal` = `run_goal` 与
  `UpdateGoalTool` 的 on_change 回调）。`run_goal` **每轮** persist（§4，对齐 agentao）。
- **`_persist_goal` 的锁纪律（不变量⑩，评审 Medium #3 + 本轮 Medium）**：它被 **loop 线程**（`run_goal`/端点）与
  **executor 线程**（`UpdateGoalTool.on_change`）**双路**调用，故须 (a) 不读半改的 GoalState、(b) 不持 `_goal_lock` 跨 IO、
  (c) **不让两线程并发写同一 sidecar**（否则截断/交错/旧快照覆盖新快照——`_goal_lock` 快照**只**解了 (a)，解不了写竞态）。
  固定为「**专用 IO 锁串行整个『快照→写盘』 + 临时文件 `os.replace` 原子替换**」：

  ```python
  def _persist_goal(self) -> None:        # 调用方一律**不**持 _goal_lock / _goal_io_lock
      with self._goal_io_lock:            # ① 串行整个快照→写盘：杜绝并发 open 同一文件，且**写序==快照序**（无陈旧覆盖）
          with self._goal_lock:           # ② 锁内快照（避免半改 GoalState）；随即释 _goal_lock，绝不持它跨 IO
              if self._goal is None: return
              data = self._goal.to_dict()
          tmp = self._goal_sidecar.with_suffix(".json.tmp")   # 同目录临时文件（os.replace 须同文件系统）
          tmp.write_text(json.dumps(data, indent=2), encoding="utf-8")
          os.replace(tmp, self._goal_sidecar)                 # ③ 原子替换：reader（restore/cold_info）永不见半写文件
  ```

  三重保证：① **`_goal_io_lock`（per-conversation `threading.Lock`，独立于 `_goal_lock`）** 串行进程内两写者——无并发
  `open(...,"w")`、且因「快照也在 IO 锁内取」故写盘顺序与快照顺序一致（杜绝「老快照后写、盖掉新快照」）；② **`os.replace`
  原子替换**防 reader（懒恢复/`cold_info`，甚至跨进程）读到半写文件；③ **`_goal_lock` 只在快照那一瞬持有、绝不跨 IO**。
  锁序固定 `_goal_io_lock → _goal_lock`（仅 `_persist_goal` 碰 IO 锁，其余代码只碰 `_goal_lock`，无反序 → 无死锁）；
  `run_goal` 内所有 `_persist_goal()` 调用点都在 `with self._goal_lock` **块外**（§4 已核），故内层取 `_goal_lock` 不自死锁。
- **懒恢复**：`Conversation` 恢复时 reload goal sidecar；`cold_info` 读它出 goal 概要（不建 agent，同决策P4.4-7）。
- **重启策略（对齐 agentao `_resume_goal` 的「stranded active」语义）**：进程崩溃/重启后，sidecar 里若 goal 仍是 `active`，
  它**必定不在被驱动**（续跑循环绑在 SSE 请求生命周期、随进程消失）。**不**自动续跑——前端据 `info().goal` 把
  active/paused/blocked 都显示为「可恢复」，弹「检测到未完成目标『…』，恢复？」，用户点 `/goal resume` 才续（决策P4.16-7）。
  （agentao CLI 对盘上 `active` 的处理同此：`_resume_goal` 允许 resume 一个 stranded active，inline 重新驱动。）
- clear 删 sidecar；终态保留供 `/goal show` 回看，下次设新目标覆盖。

## 11. 默认值与开关（复用 agentao 默认，安全上限默认开）

对齐 agentao `cli/commands/goal.py` 的 `_DEFAULT_MAX_TURNS=25` / `_DEFAULT_TIME_BUDGET="120m"` 与 settings 块：

```jsonc
// .agentao/settings.json（agentao 实测块，guanlan 同读）
"goal": { "default_max_turns": 25, "default_time_budget": "120m", "enabled": true }
```

- **预算解析复用 agentao 逻辑**（`_resolve_budget`：`--unbounded` → 双 None；省略则套 settings 默认）。guanlan 可重写
  这 ~15 行（复用 `parse_duration`）或前端套默认 + 服务端校验。`guanlan web --goal {on,off}` 一键禁用（映射 `goal.enabled`）。
- **两轴互不遮蔽**（developer-guide part-4/8.3）：`--turns` 是**主防跑飞**（一轮 = 一次完整 `arun`、内层至
  `max_iterations`，25 轮已是大量工作）；`--for` 只防**墙钟病态**，故设在轮数正常跑完点**之上**（120m），免在慢迭代目标上
  时间默默抢先、架空轮数守卫。`--unbounded` 显式退出。
- **`--turns` ≠ `max_iterations`（关键正交）**：`--turns` 界**外层续跑轮数**；`max_iterations`（`run_agent_task` 默认 200,
  `runtime.py:58`）界**单轮内层工具循环**。二者正交，须在 CLAUDE.md 与文档显式提醒、回归测试守住（§12 DoD）。
- **默认开 + 安全上限 + 一次性提示（决策P4.16-17，§14 #5）**：`goal.enabled` **默认 `true`**（沿用 agentao），`guanlan web
  --goal off` / `goal.enabled=false` 可关；省略 `--for`/`--turns` 时套 `25` 轮 / `120m`。**首次**在某 KB 设 goal 时，前端附
  一句一次性提示「已套用默认安全上限 25 轮 / 120m，可用 `--for`/`--turns`/`--unbounded` 调整」（仅一次，存 localStorage）。

## 13. 决策记录（草案，待拍板见 §14）

- **\[决策P4.16-1]** `/goal` 是 **host-side** 特性，**复用 agentao 已落地原语**（`agentao.cli.goal_state.GoalState`/
  `GoalStatus`、`agentao.cli.duration.parse_duration`、`agentao.tools.goal.UpdateGoalTool`），guanlan 只建 Web 编排层
  （async 续跑循环 + SSE 帧 + 端点 + 前端）。零 agentao 改动。遵 developer-guide part-4/8 host 范式。
- **\[决策P4.16-2]** **只做 time/turn 预算**（继承 agentao；预算判定复用 `goal.budget_tripped()`）。token 预算 out of scope。
  断线 **v1 停为 paused**（非后台续）。
- **\[决策P4.16-3]** 复用 `UpdateGoalTool(goal, on_change)`，经 **loop 作用域** `add_tool(replace=True)`/`remove_tool` 注入
  （**非**构造期 `extra_tools`——非 goal 问答不该见此工具）；`is_read_only=True`（只读会话可用）+ active-only 守卫均 agentao
  内建；duck-type 只需 goal 有 `is_active`/`mark_*`/`status_label`。prior 同名工具快照 + finally 还原（镜像 agentao）。
- **\[决策P4.16-4]** 停止断**整个 goal** **靠现有取消令牌**（纠正初稿臆造的 `_goal_stop_requested`）：轮内 cancel → `arun`
  抛 `AgentCancelledError` → `run_goal` `except AgentCancelledError` `pause()` + break；轮间 `_stop_requested` → 下一轮秒取消
  → break。可选轮顶提前 break 省一空轮。
- **\[决策P4.16-14]（评审 High #1/#2）** `run_goal` 是 `_run_turn` shield 逻辑的**多轮推广**：**每内层轮必 `turn_task =
  ensure_future(self.turn(...))` + `await asyncio.shield(turn_task)`**（绝不裸 `await self.turn`，否则断线 cancel 穿透进
  `turn()`、在 executor 仍跑 `arun` 时释 `conv.lock`、重引并发改 `agent.messages` 的竞态）。`run_goal` **两个 except 都
  pause + 同步落盘 + 传播**：`except AgentCancelledError`（停止/令牌 trip，经 shield 正常上抛）与 `except asyncio.CancelledError`
  （断线 cancel 打到 `await shield`）——保证断线**必落 paused**（修「初稿只在 `except AgentCancelledError` pause、断线走
  `CancelledError` 漏 pause」）。`_run_turn` 的 goal 分支只 `await conv.run_goal`、不另持单轮 `turn_task`。
- **\[决策P4.16-15]（评审 Medium #4 / High #3 / 开放问题 #1·#2）** **单锁 + 不换对象**：① 所有 `GoalState` 读写过
  `_goal_lock`，`UpdateGoalTool` 收 **`_GoalProxy(g, _goal_lock)`**（duck-type 四面各自持锁），残留 execute 内 TOCTOU 良性
  （active-only 守卫，无非法状态）；② **active 期 `clear`/替换一律 409**（先 `/goal pause`）——使 `self._goal` 活动期不换/删、
  `run_goal` 的 `g` 不悬挂（修「clear 删 `_goal` 而循环仍用 live `g`」）；③ **`pause` 端点 = 锁内 `g.pause()` + `request_stop()`**
  （即时打断在飞轮，非等本轮自然结束）。
- **\[决策P4.16-16]（评审 High / Medium #1 / Medium #3，本轮新增）** 三条收尾纪律：① **tool 增删前 inner turn 必 done**——
  断线路径在 pause+落盘后 `await asyncio.gather(turn_task, return_exceptions=True)` **排空** shield 的后台 inner turn，`finally`
  再 `remove/restore`（兜底「未 done 先 gather」）；遵 agentao「工具仅两轮间增删、执行按 live 注册表解析」，杜绝迟到 in-flight
  `update_goal` 落空/竞态（`gather` 一并消费 inner 异常，取代 `add_done_callback`）。② **GoalState 锁外零读**——prompt 与
  `goal_done` 帧字段（objective/turns_used/status/summary）先在 `_goal_lock` 内**快照**再用，prompt 函数只吃 `objective` 串。
  ③ **`_persist_goal` =「`_goal_io_lock` 内：取 `_goal_lock` 快照 `to_dict()` → 释 `_goal_lock` → 临时文件 `os.replace` 原子写盘」**
  （loop/executor 双路调用，专用 IO 锁串行两写者 + 原子替换防半写文件 + 不持 `_goal_lock` 跨 IO，§10、本轮 Medium）。
- **\[决策P4.16-5]** workspace-write goal **逐轮**取/放 write_lock，绝不跨整个 goal 持锁；`_reject_if_writable_active` 纳入
  「活跃可写 goal」。read-only goal 不持写锁。
- **\[决策P4.16-6]** goal **不放宽任何边界**：每内层可写轮照走 P4.5 层①②③ + 写后 check + 撤销；goal 只决定「跑几轮、何时停」。
- **\[决策P4.16-7]** **per-conversation 持久化** `.agentao/goals/<id>.json`（**不**用 agentao 单文件 `goal.json`/`save_goal`），
  复用 `GoalState.to_dict`/`from_dict`。重启后盘上 active 目标**不自动续**，前端显示为「可恢复」、用户 resume 才驱动（对齐
  agentao `_resume_goal` 的 stranded-active 语义）。
- **\[决策P4.16-8]** `/goal <obj>`、`/goal resume`、re-budget 复活 limit_reached **驱动 SSE 续跑流**（受控偏离 P4.4-1）；
  `show`/`pause`/`clear`/`edit`/`budget`(非复活) 瞬时端点 + `noteLines`。命令分类镜像 agentao `_classify`（无参子命令 vs objective）。
- **\[决策P4.16-9]** confirm-on-replace v1 取 **409 + 前端提示先 clear**；`is_stopped` 终态目标直接替换。
- **\[决策P4.16-10]** 轮内异常 → `pause()` active 目标再抛（对齐 agentao，不让异常搁浅 active 目标）。中断（停止/断线）→ pause。
- **\[决策P4.16-11]** 续跑/收尾 prompt **镜像 agentao** `_continuation_prompt`/`_wrap_up_prompt`（私有不可 import）+ 加 wiki
  语境；objective 当不可信数据嵌入。
- **\[决策P4.16-12]** **不走 `force_continue`**（agentao 插件 Stop 路径硬上限 3、注入可见 user 消息，皆不合长任务，
  developer-guide part-4/8.4）；用 host-owned `run_goal`。`--turns`（外层）⊥ `max_iterations`（内层）。
- **\[决策P4.16-13]** **import 复用 `agentao.cli.goal_state`/`cli.duration`**（纯模块、零副作用），接受 cli-namespace 耦合
  （guanlan 本就深耦 agentao 内部）；**定**（§14 #4）：不本地镜像（`UpdateGoalTool` duck-type 让本地镜像也可配，但单一真相源更优）。
- **\[决策P4.16-17]（§14 #5）** `goal.enabled` **默认 `true`** + 默认上限 `25` 轮 / `120m`（沿用 agentao），`guanlan web --goal off`
  可关；**首次设 goal 弹一次性提示**说明默认上限与 `--for`/`--turns`/`--unbounded`（存 localStorage，仅一次，§11）。
- **\[决策P4.16-18]（§14 #10）** `/goal` 首轮**带附件**（`first_images`，来自触发请求体；非 CLI `/image` 暂存），**仅首轮**；
  「成功才消费」靠 `turns_used`（首轮成功 0→1；中断则仍 0、`/goal resume` 首轮再带，前端据 `info().goal.turns_used==0` 重附）；
  附件 base64 **不入 GoalState/sidecar**（§4、§10）。
- **\[决策P4.16-19]（§14 #11）** 设/恢复 goal（驱动 SSE 的端点）在本会话 `_inflight > 0` 时 **409**（先停当前再设/恢复），
  防同会话两条 SSE 流抢 `conv.lock`/串 `begin_turn`/`_inflight`（§8.2）。瞬时端点不受限。
- **\[决策P4.16-20]（§14 #12）** `_reject_if_writable_active` 口径**进程级**：扫全体会话，**进程内至多一个活跃可写 goal**
  （`any(c.in_goal() and c.mode=="workspace-write")`）；只读 goal 不限（§6）。
- **\[决策P4.16-21]（§14 #1/#8）** 续跑循环放**后端 host-owned**（§4，host 拥有 stop 条件是范式灵魂；断线/预算原子）；
  编号 **P4.16**（P4 宿主家族下一顺位）。
